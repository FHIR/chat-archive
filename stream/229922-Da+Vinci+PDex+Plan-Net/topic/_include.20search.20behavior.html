Mark Scrimshire (Jan 27 2022 at 14:54):

@Sean Mahoney I am trying to determine the correct search behavior when the _include=Organization:partof is used in a search on Organization.

Should the _included record be added to the Page of the Bundle if the Organization resource has already been returned in that Page of the bundle.

The FHIR Specification says: When search results are paged, each page of search results should include the matching includes for the resources in each page, so that each page stands alone as a coherent package.

Would this mean that if I returned a search as a series of pages then I would include the referenced Organization.partof record if the Organization was NOT already included in the returned Page.

And If my search returns only a single page of results if the referenced Organization.partof record is already included in the original search criteria then it would NOT be added as an include...

It is not clear from the specification whether _included resources should be added to the bundle or whether the page of the bundle needs to be evaluated to determine if the Organization.partof record is already incorporated into the entries in that page of the bundle.

The reason for this question is that we are trying to determine the correct behavior as part of a certification test for Plan-Net.
