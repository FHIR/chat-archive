Kate Dech (Aug 27 2020 at 14:15):

@Gail Kocher @Robert Dieterle @Robert McClure @Saul Kravitz :

As a PBM, we have distinct contracts for standard retail, but we also have extended-supply retail (up to a 90 day fill, often with better pharmacy or member rates), and preferred retail and preferred ESN. These are often key elements for searching or filtering results by. If we use the OrganizationAffiliation.specialty element for representing the contract, there is a taxonomy code for retail. But it is a required binding and so we are unable to add addition qualifications to the type of retail. Has this been brought up? This is common across PBMs so I would think we would want a consistent element/way of denoting this. If we are using the .specialty element, it feels like it should be an extension directly on one taxonomy code.

Here's the type of contracts/networks that we support in our directories today:

Standard Retail - Taxonomy 3336C0003X Community/Retail Pharmacy
Preferred Retail - Taxonomy 3336C0003X Community/Retail Pharmacy + a Preferred Indicator
Standard Extended Supply/90 Day Retail - Taxonomy 3336C0003X Community/Retail Pharmacy + an Extended Supply Indicator
Preferred Extended Supply/90 Day Retail - Taxonomy 3336C0003X Community/Retail Pharmacy + an Extended Supply Indicator + a Preferred Indicator
Standard Mail Order - Taxonomy 3336M0002X Mail Order Pharmacy
Preferred Mail Order- Taxonomy 3336M0002X Mail Order Pharmacy + a Preferred Indicator
Specialty - Taxonomy 3336S0011X Specialty Pharmacy
Specialty - [Specific Disease/Drug, like Hemophilia] - Taxonomy 3336S0011X Specialty Pharmacy + Disease Specific Indicator + Description
Specialty-Limited Distribution (*likely not needed at this time but may be needed payer to provider) - Taxonomy 3336S0011X Specialty Pharmacy + LD indicator
Long Term Care - Taxonomy 3336L0003X Long Term Care Pharmacy
Home Infusion - Taxonomy 3336H0001X Home Infusion Therapy Pharmacy
