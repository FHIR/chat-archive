Chetana Nayyar (Mar 31 2021 at 08:54):

The plannet Network profile as mentioned in the IG is based on US Core Organization, but there is a line mentioned in the documentation itself ;
The Network profile is based on USCore Organization, since there was no contradiction between the USCore profile and the Plan-Net requirements. However, the NPI and CLIA identifier types, which are Must-Support, are clearly intended for provider organizations only and are not expected to be populated for other organization types.

so how is it feasible to inhert the network profile from US Core Organization where the NPI and CLIA are MUST SUPPORT?

Saul Kravitz (Mar 31 2021 at 13:53):

hi @Chetana Nayyar . Good question. There is no contradiction between the US-Core base for the network profile and the statement in the profile documentation that NPI and CLIA identifiers are not expected for organizations that are not providers.

Must support != required. Basically MS means you have to be capable of producing it (server) and consuming it(client), and must produce it if it is available. If the data is not available, the server need not make excuses -- just don't provide it.
