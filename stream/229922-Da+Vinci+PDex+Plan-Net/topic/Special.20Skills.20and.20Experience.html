For those health plans looking to publish special skills and experience (e.g., trauma, child welfare, substance abuse, physical disabilities, chronic illness, HIV/AIDS, serious mental illness, homelessness, deafness or hard-of hearing, blindness or visual impairment, co-occurring disorders) via API, are you expecting to use Healthcareservice Type ?

Gail Kocher (Apr 07 2021 at 15:27):

@Ron Urwongse Yes, this kind of information goes in HealthcareService.type. The binding here is extensible so if it is not present in the value set, additional coding may be used. There is also HealthcareService.characteristic but there is no value set specified. I would encourage consistent use of HealthcareService.type. Additional values you might look into for use there since it is extensible is the NUCC Provider Characteristics code set. There are a lot of useful values in this code set. https://www.nucc.org/index.php/code-sets-mainmenu-41/provider-characteristics-mainmenu-39/code-lookup-mainmenu-63
