We ran into this question in the case of an Immunization with no identifier, but it seems to be a question either way.

Converting a FHIR Immunization to CDA, should we put the id in CDA identifier?

I think the answer is yes: it can serve as an identifier, and you can even use it to get the original. You just need to use the whole URL, since you can't put the endpoint in the CDA system. Issue encountered with instance with no identifiers, but having an identifier doesn't seem to change the question.

With an OID for URLs in "system": urn:oid:2.16.840.1.113883.4.873.
