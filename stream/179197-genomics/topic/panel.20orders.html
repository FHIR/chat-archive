Andrea Pitkus, PhD, MLS(ASCP)CM, CSM (Mar 07 2019 at 04:29):

As a follow up to call discussions and tracker on panels being orders that result in "groupings" of test results/observations in the genomics report, there are Service Request resource info that can address the issue.

See https://www.hl7.org/fhir/valueset-request-intent.html

The original order is for a single test order/result, or panel order or profile order( a grouping of sub panels) each with their own result items reported as part of Diagnostic Report .

It should resolve the tracker and need for groupings that appear in the diagnostic report for panels. Recommend that CG recognize the context and info provided on the order side of things that end up reported as part of the diagnostic report and structured as other lab tests, panels in other laboratory areas.

Also, Service Catalog has firmed up their resources that are used to generate the Service Catalog and resources/structure used for lab catalogs/menus/CLIA Specimen Collection Manual. Quest shared examples of the 4 types of orders: single test order/result; panel order with one or more results; reflex order (conditional add on order depending on result values), and profile (physician convenience panel/tiered orders/grouping of panels). The same data model/resources would be used for genomics test orders as well. Attaching Service Catalog examples (see slides 11-15) OO_Catalog_resources_discussion_20190305.pptx
