Pieter Edelman (Sep 13 2021 at 09:20):

Hi @Steven van der Vegt
I'm sorry to hear that you have trouble navigating this information, that's something we cannot change in the short term. The main entry point should be the Informatiestandaarden wiki. If you look at the eOverdracht main page on the wiki, you'll see a section "Actuele versie" with links to the relevant information. The main bullets are the Functioneel Ontwerp and Technisch Ontwerp. The Functioneel Ontwerp is based on the Dataset in ART-DECOR, which in turn is based on the Zorginformatiebouwstenen (zibs) 2017 edition. The Technisch Ontwerp is built on the Technisch Ontwerp, and implemented using the zib FHIR profiles which are linked there (which in turn link to Simplifier for visualization). Hope this helps a bit?

The FHIR profile for zib NursingIntervention is a bit of an oddball. If you look at the main comment you'll see that it is a "NursingIntervention as defined by the Dutch Health and Care Information models (Dutch: Zorginformatiebouwsteen or ZIB) version 3.1, release 2017 with an addaption by the 'VNN redactieraad 2018', where a informationmodel is proposed without the NursingProcedure container and attached mandatory Activity concept" [sic]. So the Activity has been removed by request from the V&VN.

Pieter Edelman (Sep 13 2021 at 09:56):

Regarding the code lists: the zib page links to a terminology browser for the SNOMED concept. The NICC codes are not relevant for eOverdracht.
