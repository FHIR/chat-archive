Juha Vuorinen (Dec 22 2017 at 14:07):

Does patient profile "fiphr-patient-stu3" support Finnish Social Identity Number (FSIN)? For me it looks that it supports element patient.id , which basically could store FSIN, but, which according to it's definition, doesn't seem support FSIN.

Anna Korpela (Jan 10 2018 at 08:56):

In Finnish PHR, an uuid assigned for the patient will be stored in the Patient resource instead of a FSIN.
The uuid for the patient (a pseudonym) will be generated by the authorization server (and mapped to a FSIN) when the server creates the Patient resource. This happens when the patient/user authorizes an application and grants the application access to his/her resources. The pseudonym/uuid will be returned to the application with the access token response.
See the Finnish PHR authorization guide for more details about authorization (chapter 5 describes this part).
