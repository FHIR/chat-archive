Topic: [MHD] Suggestion for improving search on contained Resources

Simone Heckmann (Oct 30 2017 at 07:47):

Quote from MHD's "Open Issues and Questions":

MHD_048: We have chosen to define some secondary resources as ‘contained’, which in
FHIR means that they would not standalone, and thus would not be revisable. We chose
this characteristic as the closest to XDS metadata principle. However this means that
225 these secondary resources would not be searchable.

I would suggest to define a profile on the Reference type and make populating the displayelement (e.g. with the author's name) mandatory.
Also, whenever there are identifiers for these resources that are commonly understood within the domain - such as national identifiers for practitioners - Reference.identifier should also be populated. That way, at least a search by name and/or identifier could be supported, which probably covers >80% of the usecases.

John Moehrke (Oct 30 2017 at 13:26):

That might be a good 'best practice', but IHE has existing XD* profiles that don't have that requirement. Thus where the data is stored in XDS, these best-practices can't be filled. A secondary profile that further refines, seems like a useful way to convey these best-practices... Right?
