Dan Gottlieb (Nov 27 2017 at 16:50):

Cross posted from the SMART mailing list:

The SMART team is excited to announce the release of a new web based tool to launch your SMART applications on the SMART sandbox - check it out at https://launch.smarthealthit.org !

Key features include:
- Test and demonstrate SMART apps without needing to create an account or log in.
- Share launch configurations by bookmarking and emailing urls.
- Supports apps that use the SMART EHR launch flow, standalone patient facing apps, standalone provider facing apps, and apps using the backend services authorization API.
- Quickly switch between the SMART DSTU2 and STU3 FHIR servers to test apps that work in both environments.
- Pre-configure contextual information (including the patient, provider, and encounter) before the launch or leave these fields blank to be prompted for the required info during the launch flow.
- Simulate authentication and authorization errors at different points in the launch to give your application a workout.
- Simulate common EHR and patient portal screens for more realistic app demonstrations.
- Integrated patient browser to find the best sample data for your app.
- Simple and straightforward code and architecture to enable experimentation with new FHIR and SMART capabilities.
- More features coming soon!

This is an alpha release (there will be bugs) so please post feedback and any issues you run into here. Also, we wanted to get the project out to the SMART community early, but once the code has been cleaned up a bit, we'll be releasing it as open source on the SMART Github account. The launcher currently only supports JSON FHIR responses.
