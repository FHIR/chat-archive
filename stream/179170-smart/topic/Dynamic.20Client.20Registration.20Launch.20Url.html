In section 1.0.3 - Registering a SMART App with an EHR, there's the requirement to "Register zero or more fixed, fully-specified launch URL with the EHR’s authorization server". Wow, great question. I don't think so, which gives you a sense of how much experience we have with dynamic client registration in EHR Launch scenarios. It would be worth defining an extension parameter on the dynamic registration request like "smart_ehr_launch_uri": array of strings

Alexander Zautke (Sep 29 2021 at 13:41):

Haha, nice. Added FHIR-34031 to create such an extension. Note that RFC7591 requires the extension uri to be an absolute uri or to have it registered in the official registry.
