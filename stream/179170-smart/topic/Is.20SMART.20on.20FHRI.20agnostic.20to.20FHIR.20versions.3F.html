I think that it's always been the intent that the SMART on FHIR spec is agnostic to specific versions of FHIR. Is that stated anywhere authoritatively? I'm trying to counter the idea that SMART can't be used in a SSO integration because it's "based on older versions of FHIR (STU3/DSTU2) and they have no backwards compatibility".

Josh Mandel (May 03 2021 at 15:23):

Section 1 has said, ever since our first publication:

This profile is intended to be used by developers of apps that need to access FHIR resources by requesting access tokens from OAuth 2.0 compliant authorization servers. It is compatible with FHIR DSTU2 and above...
