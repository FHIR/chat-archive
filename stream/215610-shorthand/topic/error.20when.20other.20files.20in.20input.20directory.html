Why is this an error ? 

error Invalid file detected in directory fsh/ig-data/input/examples. Input FHIR definitions must be JSON.

Nick Freiter (Sep 03 2020 at 11:12):

Yeah that is a good point. We do need to alert the user in this situation somehow, because we don't want people to have files that they expect SUSHI to be aware of that SUSHI is not aware of, but I think a warning would be more appropriate. I've made an issue to track it here: https://github.com/FHIR/sushi/issues/594

Nick Freiter (Sep 03 2020 at 11:14):

We are planning on tweaking the way that SUSHI integrates with the IG Publisher, as described here: https://chat.fhir.org/#narrow/stream/215610-shorthand/topic/SUSHI.20.2F.20IG.20Publisher.20Integration.20Proposal/near/205978827. Once we make these tweaks it shouldn't be an issue at all anymore, since we will be looking for BYO-JSON/XML definitions in input/examples directly, instead of fsh/ig-data/input/examples.
