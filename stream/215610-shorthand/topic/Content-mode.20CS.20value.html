Etienne Cantineau (Jul 08 2020 at 13:02):

Hello, if i set a content-mode value for a code system in sushi (* ^content = #fragment) the "count' element disappears , is it a normal behavior?

Chris Moesel (Jul 08 2020 at 13:11):

Yes. count is intended to represent the total number of concepts in the code system. A fragment, by definition, is a subset of the code system -- so SUSHI cannot automatically provide a count since the total count is unknown to SUSHI (i.e., SUSHI doesn't know how many codes exist outside the fragment). In this case, if you want the code system to report a count, you must provide the count yourself via a * ^count = {number} rule.
