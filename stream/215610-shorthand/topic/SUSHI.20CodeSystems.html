Richard Kavanagh (May 31 2020 at 19:53):

Is it possible to set designations, properties or child concepts on concepts within a FSH CodeSystem definition?

I am assuming not. If not, are they on the roadmap?

Nick Freiter (Jun 01 2020 at 12:38):

Unfortunately, your assumption is correct. It is not possible to do this within the context of a FSH CodeSystem. It is possible to create an Instance of a CodeSystem, but this is not really a great solution, since you lose a lot of the shorthand that is the whole point of FHIR shorthand.

Yes, this is a task in our backlog, and multiple uses have run into this issue, so it is something we will prioritize.
