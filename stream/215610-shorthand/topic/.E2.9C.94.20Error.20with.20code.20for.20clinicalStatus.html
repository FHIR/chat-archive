Richard Stanley (Jul 18 2021 at 23:54):

In the fsh below I am using the code #inactive for clinicalStatus in a Condition resource. It is most definitely in http://hl7.org/fhir/ValueSet/condition-clinical but Publisher errors. The error at qa.html is:

"None of the codings provided are in the value set http://hl7.org/fhir/ValueSet/condition-clinical|4.0.1 (http://hl7.org/fhir/ValueSet/condition-clinical), and a coding from this value set is required) (codes = null#active)"

The shorthand:

Instance: Condition-Pregnancy
InstanceOf: Condition
* clinicalStatus = #inactive
* verificationStatus = #confirmed
* subject = Reference(Patient-HIVSimple)
* code = SCT#77386006 "Pregnant (finding)"
* onsetDateTime = "2020-01-01"
* abatementDateTime = "2020-09-30"

Jean Duteau (Jul 19 2021 at 13:39):

The statuses are CodeableConcepts so you need to provide the system with the code. You are just giving the code which is why it’s being flagged as an error.

Nick Freiter (Jul 19 2021 at 13:41):

+1 to @Jean Duteau . In your case, that would look something like this:

* clinicalStatus = http://terminology.hl7.org/CodeSystem/condition-clinical#inactive

SUSHI will then know which value to put at Condition.clinicalStatus.coding[0].system.

Richard Stanley (Jul 19 2021 at 14:04):

Thanks @Nick Freiter and @Jean Duteau I think my confusion was that some codes were happily resolving without declaring the codesystem but those fields were not using codeableconcepts :)
