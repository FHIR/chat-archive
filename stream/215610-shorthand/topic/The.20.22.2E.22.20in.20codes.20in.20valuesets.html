Topic: The "." in codes in valuesets

Halina Labikova (May 15 2020 at 19:52):

Sushi is not happy about the periods (".") included in codes when defining valuesets. The solution seems to be to put the code in quotation marks, though this happened to slip through cracks in the mcode ig.

Chris Moesel (May 18 2020 at 16:11):

Hi @Halina Labikova. Thanks for reporting this. We've just done a few tests with codes containing ., and in our experience, it has worked without needing quotes. For example, I was able to successfully add the following code to a ValueSet definition:

* ICD10#U07.1 "2019-nCoV Acute Respiratory Disease"

We also tested this for codes when defining CodeSystems.

Regarding mCODE, I've let @Mark Kramer know about the missing . in ICD codes in mCODE value sets.
