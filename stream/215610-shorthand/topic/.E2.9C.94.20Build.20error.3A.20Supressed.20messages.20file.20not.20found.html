Eric Torstenson (Sep 10 2021 at 17:06):

I have a Fish bowl that is generating this one error no matter what I do. The IG the error message refers to is being generated by sushi, but it doesn't really seem to reference any problems. The way it shows up in the qa.html report is:

n/a Show Validation Information
ImplementationGuide.definition.parameter[22].value error Supressed messages file not found

Should I do something to generate a file for these types of messages?

David Pyke (Sep 10 2021 at 17:09):

You need an ignoredWarnings.txt file in your input directory.
This file has the following format:

== Suppressed Messages ==

Your Reason Here

error or warning message to be suppressed
error or warning message to be suppressed
...

Lines are grouped by # Reason which explains why the message is considered ok (not an error). Each message line below it contains the message to be suppressed - copy and paste the message from the qa.txt file produced by the ig-publisher. (typically output/qa.txt).
