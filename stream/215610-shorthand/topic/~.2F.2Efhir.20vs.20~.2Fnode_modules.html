I'm not aware of anything that says that FHIR packages can (or should) be stored in a node_modules folder. Nothing in the NPM Package Specification doc or FHIR Package Cache doc says anything about supporting node_modules. But this kind of documentation is spread all over the place. Do you know of doc or spec that references node_modules or are you just assuming it should work like that because it uses NPM package format?
