Chris Moesel (Oct 07 2021 at 12:18):

Hi @Giorgio Cangioli. You're probably going to kick yourself, but... that error message actually tells you exactly what to put in the sushi-config.yaml:

The resource should declare its jurisdiction to match the package id (hl7.fhir.uv.fhir-for-fair, jurisdiction = http://unstats.un.org/unsd/methods/m49/m49.htm#001: World) (for FSH: 'jurisdiction: http://unstats.un.org/unsd/methods/m49/m49.htm#001 "World"')

The config accepts FSH-style codes just like in FSH codes. So this should do it (plus the parameter to apply it, which I think you got right)

jurisdiction: http://unstats.un.org/unsd/methods/m49/m49.htm#001 "World"

For future reference, the SUSHI documentation has an exhaustive example of a config that helps clarify some of these things as well: https://fshschool.org/docs/sushi/configuration/#exhaustive-example
