Topic: ✔ Fixed Value "0" for Quantity isn't applied

Simone Heckmann (Nov 22 2021 at 18:23):

When I create an instance for an Observation profile with a fixed, mandatory low range of "1" , 

* referenceRange 1..1 MS
  * low 1..1 MS
  * low = 1 '{score}' "Punktwert"

the element gets auto-populated as expected.
However, if I change the value from "1" to "0", the value for low is missing from the instance.
..is this a bug, or am I loosing it...?

Simone Heckmann (Nov 22 2021 at 18:29):

..and the same appears to be true, if I assign the "0" value directly in the instance instead of the profile...

Chris Moesel (Nov 22 2021 at 18:48):

This is a known bug and we plan to address it in the next release. See SUSHI#947 for a workaround in instances. The workaround for profiles would look a little different. Stand by.

Chris Moesel (Nov 22 2021 at 18:56):

The workaround in a profile is to use an inline instance to set the Quantity. E.g.:

Profile: MyObservationWorkaround
Parent: Observation
* referenceRange 1..1 MS
  * low 1..1 MS
  * low = ZeroScore

Instance: ZeroScore
InstanceOf: Quantity
Usage: #inline
* value = 0
* code = #{score}
* system = "http://unitsofmeasure.org"
* unit = "Punktwert"

Instance: MyObservationWorkaroundInstance
InstanceOf: MyObservationWorkaround
* status = #final
* code = http://example.org#foo

Example of bug and workaround in FSH Online: https://fshschool.org/FSHOnline/#/share/3cFdPQm
