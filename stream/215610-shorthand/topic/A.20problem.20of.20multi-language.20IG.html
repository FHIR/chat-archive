We build IG in two versions, one is in English, and the other is in Traditional Chinese.

I guess the reason is that we have translated the default description & constraints from English to Chinese. So, the "_genonce.bat" or "publisher.jar" thinks we have modified the default values (even though we only do the translation without redefining cardinality, etc.) and put those translated elements in the differential table. as shown below.

Chris Moesel (Jan 13 2022 at 13:47):

Hi @Li-Hui Lee. I think the differential table shows any element that has changed in any way -- which would include changes to the short description. I understand why you might not want this in cases like yours, where the change really is just a translation of the original text. I think this issue would fit best in the #IG creation stream, so you might want to post it there as well. Maybe someone will have an idea or has done something like this before.

You might notice that a lot of the other data in the elements uses a very light (or partially transparent?) font. This indicates that the data did not actually change. So users can see that the important things (like cardinality and type) did not actually change. But... it is pretty subtle.
