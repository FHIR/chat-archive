Topic: ✔ CodeSystem caseSensitive element

Richard Stanley (Jan 26 2022 at 21:02):

Hi folks. I'd like some help if possible. Publisher complains when CodeSystem resources don't have the caseSensitive boolean element. However, including one in FSH results in the error: "no viable alternative at input '\n* casesensitive". Minimum broken example here below which is reproducible on FSH school. Thoughts?

Alias: $OpenCR = https://intrahealth.github.io/simple-hiv-ig/CodeSystem/OpenCR

CodeSystem: OpenCR
Id: OpenCR
Title: "OpenCR CodeSystem"
Description: "OpenCR CodeSystem"
* ^url = $OpenCR
* #1234 "Golden Record"
// doesn't work
* caseSensitive = false

Jean Duteau (Jan 26 2022 at 22:10):

It's a special attribute:

  • ^caseSensitive = false/true

Chris Moesel (Jan 26 2022 at 22:13):

Right. When you need to break out of the special FSH CodeSystem syntax and get directly at an element in the CodeSystem resource, you reference the element as ^elementName. This is consistent with how you directly reference elements in StructureDefinition when defining a profile.
