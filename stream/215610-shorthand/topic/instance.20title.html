Vadim Peretokin (Jan 04 2022 at 15:08):

If I use Title: in a conformance resource, it comes out in the output correctly. However if I use it in an instance that can also take a title element, like a Questionnaire, it does not come out. Is this known? https://fshschool.org/FSHOnline/#/share/32OCOPB

Chris Moesel (Jan 05 2022 at 17:48):

Hi @Vadim Peretokin -- yes, this is known, and currently by design. For Instance:, the Title: is only used in the IG metadata to give the instance a title when it is listed on the artifact page. We do not apply it to the instance's title element itself because not all instances have a title element -- so it might be confusing to have the behavior differ depending on the instance type. In addition, there is no guarantee that a title element has a particular semantic meaning (although it seems unlikely to differ).

I think that populating title and description in instances is something we can reconsider. It could potentially be a configurable option. I'll add a backlog item for us to consider this.
