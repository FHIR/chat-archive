Eric Haas (Feb 02 2021 at 02:48):

As @Gay Dolin @Brett Marquard and I triage US Core requirements we are seeing some pushback to the search requirements defined in the Guide. I think of most IGs as a bunch of documentation to use a bunch of FHIR Artifacts ( structures ) and a set of interactions to meet a particular use case like patient/provider access. However when reusing arrtifacts from another guide, I think that they are separable . In other words, you don't have to state conformance to the whole IG in the CapabilityStatement ( and dragging along the set of interaction declared in it ) but can pick or choose a structure or two. For example I use SDC Questionnaire in the Argonaut patient list extension as target to an extension, but I don't have any expectation that the SDC guide be supported.

We have formulated these three level of reuse to help frame our discussions and would like some feedback here:

  1. Conform/reuse profile

    • Only asserting to requirements in the StructuredDefinition

  2. Conform to guide

    • asserting to requirements in the StructuredDefinition
    • following the interactions noted in the CapabilityStatement

  3. Certification (ONC Reg)

    • Conform to the guide
    • Whatever else ONC requires.
