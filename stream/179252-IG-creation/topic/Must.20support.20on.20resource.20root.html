Topic: Must support on resource root

Elliot Silver (Nov 17 2021 at 18:13):

I've come across a few IGs that put a must support flag on the root of a profiled resource, e.g. on Patient itself, or on Extension. How should we interpret that? Is it a requirement that your system must support (for some definition of MS) that profile of Patient? Is it a redundant statement?

In the case of an extension with this flag set, anywhere I profile use of the extension, it becomes must support. For example, if I profile use of the IPS Condition abatement extension, on my AllergyIntolerance profile, then I see that slice become must support. Is that intended? Interestingly, if I explicitly add "mustSupport" : false to the slice it then goes away. Is that intended?
