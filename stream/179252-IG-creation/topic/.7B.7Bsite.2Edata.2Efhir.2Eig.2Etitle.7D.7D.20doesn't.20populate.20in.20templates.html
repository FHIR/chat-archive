Chris Moesel (May 15 2019 at 14:09):

Thanks for the tip, @Eric Haas -- but putting _config.yaml in my _includes folder didn't work (and I don't have a layouts folder). That said, I noticed that the title seems to be available at site.data.pages["index.html"].title, so I ended up adding this code into the template wherever I need to access the title:

{% assign title = site.data.fhir.ig.title %}
{% unless title %}{% assign title = site.data.pages["index.html"].title %}{% endunless %}
{% unless title %}{% assign title = site.data.fhir.ig.name %}{% endunless %}

This covers the bases so that when site.data.fhir.ig.title is available, it will use it; otherwise it falls back to the location I discovered it (but is undocumented); otherwise it just falls back to the IG name. Seems to work.

Eric Haas (May 15 2019 at 15:50):

That is indeed a clever hack. Each page has a title and that assumes the index.html = ig title. you are able to define that in the ig.xml file so you have control there. The _config.yml should be in the same directory as the _includes not inside it.

I must admit I could not get it to work forever then tried it again (mostly because Lloyd got it working) and it worked. It may be because I updated my version of Jekyll/Ruby I forget the details.
