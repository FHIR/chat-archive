I'd like to include an external code system, with its own version, into an IG.

The right version is overwritten by the IG one

original <version value="31_12_2019"/>
published <version value="0.2.0"/>

Lloyd McKenzie (Feb 29 2020 at 15:11):

There's an ig parameter that allows you to not have the version propagate everywhere, but it has the consequence that you'll then have to declare (and change) version on everything else. It's cleaner to define your external code system in its own IG and declare a dependency to it
