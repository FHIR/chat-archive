Topic: ✔ Information about extension types that are in FHIR but ...

Richard Braman (FLY.HEALTH) (Nov 04 2021 at 19:08):

When I build , I get an information like this from IG Publisher:

INFORMATION: Patient/T2D-Patient: Patient.extension[2]: This element does not match any known slice defined in the profile http://hl7.org/fhir/us/core/StructureDefinition/us-core-patient

the line that is producing the Information message is :

* extension[+].url = "http://hl7.org/fhir/StructureDefinition/patient-mothersMaidenName"
* extension[=].valueString = "Shirleen968 Barton704"

This extension is defined in FHIR r4; however must not be included in US-Core.
http://hl7.org/fhir/R4/extension-patient-mothersmaidenname.html

The T2D-Patient is profiled to Us-Core:

meta.profile = "http://hl7.org/fhir/us/core/StructureDefinition/us-core-patient"

Is there anyway to use a base R4 extension type while still calling it a US Core profiled resource?

Grahame Grieve (Nov 04 2021 at 19:18):

you can use it, as you have. The information is telling you exactly that - you're using an extension that US core didn't describe, but it's fine for you to use it. The reason the information appears at all is that slicing is hard to get right and sometimes the fact that something isn't recognised slice means that there's something wrong in the profile

Lloyd McKenzie (Nov 04 2021 at 22:16):

Occasionally it also means you accidentally used the wrong extension in your example.
