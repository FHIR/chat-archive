Eric Haas (Jul 03 2020 at 20:53):

I like grahames option or....

  1. in the most recent templates you can add intros before or notes after the default content by creating a markdown file ( or html for masochists) in the folder input/pagecontent and naming it [type]-[id]-intro.md or [type]-[id]-intro.md

for example: for input/resources/CodeSystem-blah-codes.xml id = blah-codes
custom intro would be input/pagecontent/CodeSystem-blah-codes-intro.md
custom notes would be input/pagecontent/CodeSystem-blah-codes-notes.md

this won't be part of the resource but part of the ig.

otherwise could try:

  1. you could copy and paste the generated output and add to your custom narrative.
  2. don't recall if this works but you could try adding liquid to your narrative to {% include [generated fragments] %}
