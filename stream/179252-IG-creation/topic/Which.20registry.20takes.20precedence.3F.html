Topic: Which registry takes precedence?

Bart Decuypere (Dec 02 2021 at 16:04):

Say, someone publishes my.package version 1.2.2 in Simplifier. They also create an IG (and publish it), and the package associated with the IG also is my.package 1.2.2. If another IG depends on my.package 1.2.2, which package would it depend on, the one in Simplifier or the one in the IG registry?

Grahame Grieve (Dec 02 2021 at 17:57):

registry.fhir.org is also simplifier, so that resolution takes place inside simplifier. If they've published their policy on that, I'm not aware of it, but I expect the answer is that the first one wins

Ward Weistra (Dec 28 2021 at 16:04):

@Bart Decuypere @Grahame Grieve Indeed, the ingestion of a duplicate combination of package name + version will be refused when offered by the package feeds to Simplifier/registry.fhir.org. So only the first one will be offered to users of the registry.
