hello folks- Not clear to me how to add an "id" to a dependsOn in an implementation guide... I have

"dependsOn": [
      "uri" : "http://hl7.org/fhir/us/core/ImplementationGuide/hl7.fhir.us.core",
      "packageId": "hl7.fhir.us.core",

]

and the publisher complains Dependency 'hl7.fhir.us.core' has no id, so can't be referred to in markdown in the IG

Lloyd McKenzie (Aug 04 2020 at 17:48):

Just add "id" as another property

Nick George (Aug 04 2020 at 17:49):

is the value meaningful?

Lloyd McKenzie (Aug 04 2020 at 17:56):

The id will manifest as a way to reference the canonical URL and version-specific canonical URL in your IG content:
{{site.data.fhir.someid}} and {{site.data.fhir.ver.someid}} where the id value is 'someid'.

Nick George (Aug 04 2020 at 18:19):

gotcha, thanks. just making sure it doesn't e.g., need to match a value somewhere else (e.g., an id field defined by US Core)

Lloyd McKenzie (Aug 04 2020 at 18:21):

No need to match - but it does need to be unique across all 'id' elements in the IG resource instance

Lloyd McKenzie (Aug 04 2020 at 18:58):

The way to constrain codes is not to fix the system, but to define a value set.

Lloyd McKenzie (Aug 04 2020 at 18:59):

In general, you should never fix the code system. If you want to fix a particular code, use 'pattern' to specify the desired coding.

Nick George (Aug 04 2020 at 19:00):

Lloyd McKenzie (Aug 04 2020 at 19:02):

Pattern is slightly better as it still allows extensions. Are you defining a fixed code? If not, you should be using a value set
