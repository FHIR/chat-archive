In SDC, in amongst all of the template broken links, I'm also seeing these:
The link 'http://hl7.org/fhir/R4/codesystem-v3-maritalstatus.html' for "v3 Code System MaritalStatus" cannot be resolved
The link 'http://hl7.org/fhir/R4/codesystem-v3-rolecode.html' for "http://terminology.hl7.org/CodeSystem/v3-RoleCode" cannot be resolved
The link 'http://hl7.org/fhir/R4/valueset-v3-servicedeliverylocationroletype.html' for "V3 Value SetServiceDeliveryLocationRoleType" cannot be resolved

I think this is being caused by the 'hl7.fhir.r4.examples#4.0.1' package including re-definitions of the code systems already defined in the terminology.hl7.org project, and furthermore being defined with the wrong URLs. I think the solution is either to drop the terminology artifacts that were migrated to THO from the package entirely. If we don't want to do that, then at least the URLs should be corrected.
