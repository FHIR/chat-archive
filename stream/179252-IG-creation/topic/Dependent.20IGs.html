Grahame Grieve (Mar 22 2018 at 07:43):

This is a note for all IG authors:
- There was a bug in the IG publisher that meant the definitions were serialised in whatever version of FHIR the IG publisher was compiled with. (didn't matter for a long time, while the versions were stable).
- That version has now disappeared, and the IG publisher can't read those output files.
- if you are trying to publish an IG that has a dependency on another IG (usually that is either us-core or au-base at the moment), then you cannot reference the IG as published, until they are regenerated.
- You must regenerate the IG locally, and then reference your local copy

If you are working with some dependency that can not be regenerated in this fashion, we need to talk...
