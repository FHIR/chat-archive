Actually, I think the missing cardinality is because when we generate the diff element, we're only putting in things that are truly different... So if only the min changed or only the max changed, you might get what looks like an unbalanced cardinality.

The question is whether we should be intentionally putting non-diff properties into the diff in order to make the doc more readable, or if the doc generator should be doing something about this.
