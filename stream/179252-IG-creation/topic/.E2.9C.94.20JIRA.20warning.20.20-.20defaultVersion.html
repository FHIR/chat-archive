Lloyd McKenzie (Oct 11 2021 at 13:07):

Your CI-build should be defined like this:

{
  "version" : "current",
  "desc" : "Continuous Integration Build (latest in version control)",
  "path" : "http://hl7.org/fhir/ig/HL7/sdc",
  "status" : "ci-build",
  "current" : true
 }

Abderrazek Boufahja (Oct 12 2021 at 08:09):

I inversed the order of the versions in jira in order to set the currentVersion as the first version in the list of versions and this fixed the issue.

However, this does not respect the description in https://confluence.hl7.org/display/FHIR/FHIR+IG+PackageList+doco , which say the ci-build shall be the first in the list on the versions => if we do so, the currentversion will be always the cibuild, based on the publisher tool compilation and generation of the jira-new.xml.

I think we can add this as a check in the buildlocal.bat, that the currentversion shall be the first one of the list of versions in the xml describing the FHIR project.

Lloyd McKenzie (Oct 12 2021 at 13:08):

If the status is "ci-build" and current=true, then it shouldn't be the default version.

Abderrazek Boufahja (Dec 03 2021 at 10:05):

For the record, here are my observations: when the package-list.json contains only versions which have the status "draft" (for my case, they were the versions "current" and "0.1.0"), the generated defaultVersion in the file "jira-new.xml" is always "current" (the ci-build version). When there is one version which is "trial-use", it become automatically the defaultVersion. Adding the "trial-use" version fixed the warning for my case.
