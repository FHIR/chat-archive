Gino Canessa (Sep 21 2020 at 22:49):

For discussion (re: including resources)... right now we have empty, id-only and full-resource defined as options on a subscription. There is a growing need to add the ability to return more than one resource. The main proposal so far has been centered around GraphDefinition (either attached to Subscription or SubscriptionTopic).

I've also been thinking about a syntax using _include / _revInclude (e.g., a topic on Encounter defines that you may request to include patient, practitioner, and diagnosis).

I'll note that there is a general feeling that we shouldn't hold up publication of any 'ready' parts of the spec for this, which ties back into the R4B / R5 discussion.
