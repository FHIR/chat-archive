Jenni Syed (Sep 29 2018 at 19:40):

So far, I'm tracking on a few topics/discussions that we've had:

  • What should servers do if the "authorization" determination is asynch, and the server won't know the subscription is error until later? The doc currently states that the request to create the subscription should return an operationOutcome if there's an error.

Jenni Syed (Sep 29 2018 at 19:41):

  • While security around data is left outside of spec, it should probably be mentioned. EG: considerations about alerts that would allow someone to know there was sensitive data (subscribing to positive HIV results) and what the recommendations/concerns are around that

Jenni Syed (Sep 29 2018 at 19:42):

  • More detail around the use of the Headers param for allowing the Subscriber to "protect" the notification URL with an Authorization token. It's in the examples, but hard to find doc around this as a solid recommendation
    ** In addition, if this is a go forward recommendation, considerations since the headers value can contain cryptographic secrets or other sensitive info that should not be stored in the clear?
    ** Just more doc around this in general

Jenni Syed (Sep 30 2018 at 15:07):

  • Behavior of _list and other query parameters that reference changeable sets of data for a subscription? This is a subscription that changes dynamically.
