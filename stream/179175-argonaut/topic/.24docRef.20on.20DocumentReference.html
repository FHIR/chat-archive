Topic: $docRef on DocumentReference

Peter Bernhardt (Sep 13 2017 at 15:25):

Anyone have some background on the motivation for this operation? I note that it isn't part of the FHIR base profile for DocumentReference and based on my reading of http://www.fhir.org/guides/argonaut/r2/StructureDefinition-argo-documentreference.html it is intended as a not-very-simplified way to execute a query with a CCD type specifier. I don't see how this adds any real value.

UPDATE: Reading further (http://www.fhir.org/guides/argonaut/r2/OperationDefinition-docref.html) I find this qualification: "The server SHOULD return at least all references for documents that it has made available in a document indexing system. This is the same as a simple query (GET [base]/DocumentReference?patient=[id]). This operaton differs from a simple query in that DocumentReferences may be created ‘on-the-fly’ in response to this operation.”

So $docRef exists to support the on-demand use case only?

Jenni Syed (Sep 13 2017 at 17:02):
