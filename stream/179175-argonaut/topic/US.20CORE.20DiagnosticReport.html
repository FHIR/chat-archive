Good Morning,
I am new to Zulip but working with the US Core profiles for the DiagnosticReport.

Speaking specifically to this requirement:
One category in DiagnosticReport.category which must have:
a fixed DiagnosticReport.category.coding.system= “http://hl7.org/fhir/DiagnosticReport-category”
a fixed DiagnosticReport.category.coding.code= “LAB”

The Standard FHIR DiagnosicReport says it can only have 0 to 1 Category. If that one category has to be a lab, would it stand to reason that US core does not support any other Codes within https://www.hl7.org/fhir/valueset-diagnostic-service-sections.html. I am working with Data that has Radiology (RAD) as an example.
Thanks,
Joyce

Lloyd McKenzie (May 11 2018 at 18:31):

It only has 0..1 category, but each category can have 0..* codings. So you're free to convey the concept of "lab" in other code systems too. What you can't do is send a code that conveys a category that's orthogonal to "lab".

Joyce Dunlop (May 11 2018 at 18:34):

Thank you for the Quick Response. I will take another look.

Eric Haas (May 11 2018 at 21:54):

That is correct the USCore DR scope is Lab. We have discussed other scopes in the past but have not created a profile for them.. You can certainly create your own profile that mimics USCore DR for Imaging, but it would not conform to USCore. Or you could submit a tracker to suggest that we have a single USCore DR profile that covers more than lab.
