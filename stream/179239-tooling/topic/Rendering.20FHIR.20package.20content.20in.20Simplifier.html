Topic: Rendering FHIR package content in Simplifier

Morten Ernebjerg (Mar 05 2021 at 15:43):

@Ward Weistra this is probably for you :smile:. I am working on an IG that re-uses certain element of the IPS directly and would like to render the corresponding profiles in my IG. I know that one can render elements of other Simplifier projects using the {{tree:ProjectName/ResourceName}} syntax. However, it seems IPS is not available as a Simplifier project. It is, however, available as a FHIR package. So my question is: Can one render element from a publicly available FHIR package in a Simplifier IG (given that the package registry is "powered by simplifier" :smirk: )?

Ward Weistra (Mar 05 2021 at 19:32):

@Morten Ernebjerg by canonical:
{{tree:http://hl7.org/fhir/uv/ips/StructureDefinition/Patient-uv-ips}}

On bottom right of editor window you can switch the autocomplete from Files to Canonical to get help.

Of course, make sure your project has a dependency on IPS package for the canonicals to be in scope.

(And I see this is undocumented so far, so I'll add that)
