Josh Mandel (Sep 02 2021 at 15:29):

There's no use case for a card that contains both Immunization and lab result records?

There might be. So far, we've erred towards keeping these discrete so applications can individually request the pieces they need; so people aren't coerced into over-sharing; and because we have pretty steep size limits on SMART Health Card contents. Lab results about infection status are the ones we see most widely requested in real-world workflows, and these are relevant for O(1 day), whereas vaccine records are relevant for O(1 year), so combining them into a single card can be challenging for that reason, too.

We don't prohibit this, but we do say:

Additionally, Health Card FHIR Profiles SHOULD only include data that need to be conveyed together. (e.g., immunizations for different diseases should be kept separate. Immunizations and lab results should be kept separate.)
