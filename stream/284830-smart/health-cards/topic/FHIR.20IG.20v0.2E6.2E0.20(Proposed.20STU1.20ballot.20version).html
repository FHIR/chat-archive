Topic: FHIR IG v0.6.0 (Proposed STU1 ballot version)

Max Masnick (Jul 22 2021 at 11:01):

v0.6.0 of the SHC Vaccination & Testing FHIR IG is ready for review (CI build).

From the GitHub pull request for this release:

These are the changes we plan to submit to HL7 for our STU1 (Standard for Trial Use) ballot.

This will be submitted to our sponsoring HL7 work group (Public Health) today (July 22). Content must be locked for ballot by August 8th. The ballot is open from August 13th through September 13th.

The ballot-ready release that comes out before August 8th (which we expect will be very similar to v0.6.0) will likely be the final "stable" release of this IG outside the HL7 process. We expect the next major release to be the official STU1 release after reconciling HL7 ballot comments sometime this fall.

If you notice any issues with v0.6.0 that need to be resolved before balloting, they must be resolved before August 8 so please let us know right away by logging a GitHub issue.

Going forward, the best ways to stay involved with the IG development process are:

  1. To join the vaccination listserv at HL7 (sign up here) for announcements about our meetings
  2. Continue to monitor this stream on Zulip
  3. Attend our meetings which are typically every 2 weeks at 12pm Eastern (search for "SMART Health Card" on this page) -- but will likely be on hiatus until after balloting concludes September 13th

The changelog is included in the pull request for this release. There hopefully aren't any surprises in this release, but if you notice any issues please let us know right away. Thanks!!

Max Masnick (Jul 22 2021 at 17:27):

The design of the IG has been updated to use the default HL7 FHIR template: http://build.fhir.org/ig/HL7/fhir-shc-vaccination-ig/branches/dev/

Max Masnick (Jul 23 2021 at 13:50):

There are no new issues or comments on the v0.6.0 pull request, which we will take as a good sign :smiley:. Unless something is identified today, https://github.com/HL7/fhir-shc-vaccination-ig/pull/148 will be merged into main and become the official v0.6.0 release.

It's possible the Public Health work group will request QA changes during the next 2 weeks, which will likely be released as v0.6.1 -- this will be the ballot version of the IG. We would then anticipate v1.0.0 to be released sometime after balloting closes mid-September -- this will include changes to address ballot comments.

The one additional enhancement that we will try to get in before balloting is method of specimen collection (https://github.com/HL7/fhir-shc-vaccination-ig/issues/165). Otherwise we expect the spec to be stable through v1.0.0, apart from QA changes and responses to ballot comments.
