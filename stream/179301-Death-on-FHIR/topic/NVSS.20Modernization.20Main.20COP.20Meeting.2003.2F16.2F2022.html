1. What is the difference between Raven and Canary?

Canary was built specifically for testing with the VRDR IG. Its .net based. Canary is primarily for learning FHIR and validating if your system can produce valid FHIR records. It also provides context and examples of data in the IJE format compared to FHIR. It was built with the EDRS to NCHS data exchange in mind

Raven is a reference implementation of a medical examiner/coroner case management system. Its not a vendor system but a testing and training system. It is used by those without a Medical Examiner/Coroner Case Management System for getting FHIR into their workflow. But not intended to replace a case management system. It will help with the lift when testing the IG quite easily. You can deploy to your environment with help from GTRI or with your IT
Raven happens to be more expansive for handing case information using VRDR IG and MDI IG (also to be in similar structure to a case management system which helps with testing and training). GTRI is building additional features that will have similar testing components in Raven as Canary does (ie. A validator). Raven is Free. It’s open source.

2. Does canary testing validate all FHRI standards including cross validations?

It does not validate the content against the standard in the IG but checks the format. There are test files to validate the content but it is limited to those particular test cases.
