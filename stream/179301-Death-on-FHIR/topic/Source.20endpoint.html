Pete Krautscheid (Aug 25 2021 at 18:09):

Hi @Velan, that source endpoint should be set to a value that's specific to the jurisdiction you're supporting. At this time it's just a string (in URI form) that is used to identify the jurisdiction for return routing at NCHS and through STEVE; it is not expected to represent an actual system endpoint that listens for FHIR messages. Each jurisdiction should simply select a URI-shaped string, and for the purposes of the upcoming connectathon provide that string to NCHS and to STEVE.
