Several comments:
In address fields, the state should use the standard 2 letter postal encoding. So, New York is NY, not YC.
State is state, and jurisdiction is jurisdiction. Only in the case of jurisdiction is YC applicable. They key missing aspect of your example is the lack of a juridictionId extension in the Death Location resource.

Saul Kravitz (Aug 24 2021 at 21:43):

Hi @Velan,
The Jurisdiction Value Set is used only in the jurisdictionID extension to DeathLocation.
For BirthRecordIdentifier.component:state the representation is ISO 3266-2 (e.g., US-NY), using states, not jurisdictions (NY, not YC).
All other state fields should use the USPS 2-letter codes (as per USCore).

I'll try to look at your record tomorrow. The new version of the VRDR library and tools (not yet Canary) has been released (V3.2.0-preview1 nuget github) so you could try to roundtrip conversion to IJE on a do-it-yourself basis.

Regards, Saul
