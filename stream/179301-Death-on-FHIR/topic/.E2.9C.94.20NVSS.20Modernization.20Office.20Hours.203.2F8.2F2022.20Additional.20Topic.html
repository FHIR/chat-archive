Bontu Fitta (Mar 12 2022 at 03:10):

  1. What is the message endpoint going to look like? What protocol is going to be used? What is/are the basic message formats?

Endpoints:
The FHIR API is a REST API over HTTPS. There are 3 API calls for users to be aware of.
1.The SAMs API gateway authentication endpoint: https://apigw.cdc.gov/auth/oauth/v2/token
2.The bundles endpoint: https://apigw.cdc.gov/OSELS/NCHS/NVSSFHIRAPI/%3cjurisdiction-ID%3e/Bundles where jurisdiction id is the two-letter jurisdiction abbreviation
a. Users POST a FHIR message to the endpoint to submit the death record to NCHS
b. Users make a GET request to the endpoint to retrieve FHIR messages containing the coded responses from NCHS

Format:
The messages are all FHIR json. Submissions must be structured as a FHIR Message and responses will be structured as a FHIR Bundle.

Documentation:

The API endpoints are defined here
https://github.com/nightingaleproject/Reference-NCHS-API/blob/main/README.md#the-nvss-api

Examples of authenticating, POST of a FHIR message, and a GET request for FHIR message responses can be found here
https://github.com/nightingaleproject/Reference-NCHS-API/blob/main/README.md#interacting-with-the-api

There are also links in that README to definitions of a FHIR Message and a FHIR Bundle.
