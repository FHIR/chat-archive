Topic: Basisprofil Medication - PZN Validierungsfehler

Morten Ernebjerg (Jan 22 2019 at 13:34):

Ich bekomme einen Validierungsfehler, wenn ich das Medikationsbeispiel für die Basisprofile (Example-Medication-de-basis-1.xml) mit dem Standard-Java-JAR-Validator validiere. Genauer gesagt beklagt er sich über den PZN-Kode:

Error @ Medication.code.coding (line 6, col11) : Error null validating Coding http://fhir.de/CodeSystem/ifa/pzn#87776663 [null}

Ich habe auch diese Variante gesehen in einem ähnlichen Kontext:

Medication.code.coding (line 6, col17) : The code system "http://fhir.de/CodeSystem/ifa/pzn" is not known; The code provided (http://fhir.de/CodeSystem/ifa/pzn#12345678) is not  valid in the value set All codes known to the system

Zumal es ja keine konkrete Auflistung von den PZN-Codes gibt (und das CodeSystem entsprechend <content value="not-present"/> hat), verstehe ich nicht ganz, wieso er überhaupt versucht den Kode zu validieren.

Ich wollte mal Fragen, ob das ein bekanntes Problem ist, bevor ich das als Issue beim Validator einbringe (der Simplifer-Validator scheint ja zufrieden zu sein).

Patrick Werner (Jan 22 2019 at 15:48):

Hi Morten,
das Problem wurde Grahame beim letzten WGM durch mich mündlich mitgeteilt und gezeigt. Die Ursache ist, dass der Java Validator tx.fhir.org als Terminologieserver verwendet und dort/oder im Validator ein BUG besteht wenn er ein System nicht auflösen kann. Er meinte er merkt sich das, aber ein unterstützendes gForge item von einem weiteren Betroffenen hilft bestimmt.
