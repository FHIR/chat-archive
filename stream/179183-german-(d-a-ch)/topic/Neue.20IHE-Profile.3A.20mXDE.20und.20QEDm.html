Simone Heckmann (Oct 09 2017 at 12:56):

Zur Allgemeinen Info hier der Inhalt des IHE-Newletters, der mich soeben erreicht hat:

Two new IHE Profiles are introduced and have wide-reaching implications not only for mHealth:
Mobile Cross-Enterprise Document Data Element Extraction – mXDE Profile and
Query for Existing Data for Mobile – QEDm Profile.

The mXDE Profile combines:
A/ Either the “coarse grained” IHE XDS Profile - in use today in more than 20 countries or the emerging IHE MHD Profile (Mobile access to Health Documents), which leverages the HL7® trial use standard FHIR® (Fast Healthcare Interoperability Resources).
PLUS
B/ The “fine grained data elements” accessed with the new QEDm Profile, which also leverages FHIR that is appropriate for granular queries of clinical data elements.
1.- Specific data elements are extracted from the structured documents per mXDE Profile.
2.- Data elements (e.g. allergies) queried using the FHIR based QEDm Profile.
3.- Each data element is referenced to the document(s) from which it was extracted per mXDE Profile.
4.- Clinician accesses context of any data element of interest using source documents (XDS, MHD Profiles) providing the clinical context in which the observation was recorded.

To find out further details please visit mXDE Profile web page.
