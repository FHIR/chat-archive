Dan Miller (Dec 17 2020 at 14:23):

Hi All,

We have some existing CDS products which have a use case on administration.
Would I be right in saying that there currently isn't a hook for administration and if not, we are ok to suggest support for one? I would imagine if we did it would require extra implementation on the client side.

The basic flow of events would be something like:
When a patient is about to have a medication administered
The clinician starts the administration process
Once the administration details are entered the application triggers the hook to process CDS
The hook services check for issues with the administration e.g. overdose/underdose and sends cards back to the client.

Bryn Rhodes (Dec 17 2020 at 16:46):

Hi @Dan miller , yes, there's not currently a hook defined or proposed for medication administration. The Hook Maturity Model in the CDS Hooks spec describes the process for proposing and maturing hook definitions, I'd suggest writing up a hook definition with the proposed workflow there and submitting that as a PR for folks to comment on.
