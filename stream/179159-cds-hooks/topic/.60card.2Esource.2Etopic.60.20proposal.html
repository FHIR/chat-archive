Isaac Vetter (Dec 11 2019 at 16:36):

Hey @Bryn Rhodes During a conversation this morning, @Dennis Patterson asked some really good questions about PR 505.

Can you weight in and help us understand:

  • Why hasn't Clinical Reasoning already solved this? (or has it?)
  • Can you elaborate on the intent of PlanDefinition.topic?
  • And contrast it with 'PlanDefinition.purpose and PlanDefinition.Goal.Category ?

Bryn Rhodes (Dec 17 2019 at 14:57):

PlanDefinition.topic provides the kind of categorization that would be useful for indexing the PlanDefinition in a repository. Purpose provides narrative guidance about why the PlanDefinition was created and what it's for. goal.category provides a categorization for that specific Goal. There is no GoalDefinition resource, so the goal elements provide a way to define goals within a PlanDefinition.

Bryn Rhodes (Dec 17 2019 at 15:03):

As far as whether Clinical Reasoning has solved this, there are 3 main use cases related to identification/categorization that the metadata resources all support. The first is unique identification, which uses the canonical url and version elements in the same way as the conformance resources. The second is topical organization (using the topic element) to provide high-level categorization, mostly used for indexing the artifact in a repository. The third is useContext, which provides structured representation of the contexts in which it is appropriate to use the artifact (e.g. in-patient setting, males, aged 45-65, diagnosis of colon cancer, etc.).

Bryn Rhodes (Dec 17 2019 at 15:07):

I think it makes good sense for cards to have a way to indicate what guidance was followed when producing the cards, and I think that's an instance of the first use case (canonical url and version). That's why I suggested a uri for that type of element. It may correspond to a FHIR PlanDefinition, but it could just as easily reference a guideline or a web page providing further documentation about the guidance that was followed.

Bryn Rhodes (Dec 17 2019 at 15:09):

I also think it makes good sense for cards to have topics describing what the guidance is about, the second use case above.

Bryn Rhodes (Dec 17 2019 at 15:10):

But I also think we should be careful about mixing those use cases. We should be clear about supporting them both.
