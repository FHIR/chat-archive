Topic: .NET 4 support

Ewout Kramer (Dec 06 2016 at 10:33):

In our effort to keep the .NET codebase and project manageable (and to prepare for the migration to .NET Core), we are looking at dropping support for .NET 4. Microsoft dropped support for this platform in january this year, and we are wondering whether to follow Microsoft's lead and drop support for it too. What do you think?

Bryn Rhodes (Dec 06 2016 at 20:08):

We still use it, but only because we haven't yet upgraded some of our projects to use Core. We are on a track to do so, so if you dropped support for it, we would be fine, we'd just have to get to Core sooner.

Brian Postlethwaite (Dec 06 2016 at 21:22):

Maybe more explicitly, dropping support for .NET 4.0 and 4.5 in favour of the supported 4.5.2 as the base version

Peter Jordan (Dec 06 2016 at 22:34):

I agree with the proposal to target only the currently-supported 4.x versions of the .NET Framework and, when feasible, add support for .NET Core.

Brian Postlethwaite (Dec 06 2016 at 22:38):

But want to ensure that we get STU3 out first (main issue preventing progress was the code generation).

Ewout Kramer (Dec 07 2016 at 12:39):

Sorry, I meant just to drop 4.0, but certainly keep 4.5(.2) for now, so no hurry for Core....
