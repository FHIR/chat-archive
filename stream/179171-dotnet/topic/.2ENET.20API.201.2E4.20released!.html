Marco Visser (Oct 14 2019 at 21:25):

We use submodules now in our source code. Common is such a submodule. So when you want to clone our source code, you should also update the submodule common. Some Git clients do that automatically, but some don't.

When you are using the Git CLI use can do the following:

  git clone --recurse-submodules https://github.com/FirelyTeam/fhir-net-api.git .

Then everything is cloned correctly.

I will update this information on our Wiki page. Thanks for noticing it.

Michel Rutten (Oct 15 2019 at 08:50):

Git submodules are fairly new. Tooling support still needs to mature a bit. Submodules simplify (no pun intended) the management of our FHIR API repos for STU3 & R4 on Git, but they also introduce some new (tooling) challenges. After consultation with e.g. Microsoft, we decided that the technology was sufficiently stable to adopt. I expect tooling to follow suite.
