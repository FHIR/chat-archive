Bob Milius (Nov 15 2018 at 15:33):

re Glossary Tracker: (GF#16513) The resolution says we agree, but need a formal proposal, which we don't have yet. I suggest we defer it. We have definitions for each element in the SDs, but they need to be more robust and unambiguous. Each SD should have a paragraph describing what it is and how it's used. eg. http://build.fhir.org/ig/HL7/genomics-reporting/obs-haplotype.html doesn't have anything describing haplotype. I think the CG group should have a glossary that's agnostic to the standard, so perhaps should be separate from IG but pointed to by the IG? Maybe developed by IM? Each term could be pointing to SO, or VMC, or some external definition when appropriate, with added considerations for CG, and the IG would describe considerations for FHIR . Maybe it starts out as part the IG, eg a separate artifact/static page, just labeled 'glossary', and later de-couple it from FHIR. But at this stage, maybe we can create a page with some of these comments on it and looking for input.
