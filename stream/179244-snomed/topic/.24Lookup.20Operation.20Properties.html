Topic: $Lookup Operation Properties

Peter Jordan (Oct 30 2018 at 22:20):

Today's SNOMED on FHIR call ended with a discussion on the nature of the Necessary Normal Form expression that the Using SNOMED CT With FHIR page specifies should be returned when a normalForm, or normalFormTerse, property is requested by a $lookup operation. Leaving aside the active debate on how to generate these expressions, an overarching question was put to the group - how are clients expected to use these expressions? Aside from purely decorative display purposes, the Group had no further use cases, so we are seeking input from the broader FHIR implementation community. Prior to the July 2018 release of SNOMED CT, normal form expressions could be used for equivalence testing but, due to structural changes in the terminology, that is no longer the case (a Classifier is required). Are there any other requirements for using these expressions?

Grahame Grieve (Oct 30 2018 at 23:05):

The only use I can think of is for display, as you say

Michael Lawley (Nov 04 2018 at 03:18):

I believe there are people who extract 'property values' from them (ie the relationships) when they're not exposed as explicit properties via $lookup. Use cases for this might include the site of a finding/procedure or ingredient/strength from a drug

Grahame Grieve (Nov 06 2018 at 06:00):

so shouldn't they be explicit properties in $lookup then?

Michael Lawley (Nov 06 2018 at 22:32):

Yes, I believe they should be
