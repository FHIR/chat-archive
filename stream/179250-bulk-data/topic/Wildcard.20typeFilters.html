Topic: Wildcard typeFilters

Mohammad Jafari (Apr 22 2019 at 19:03):

In the bulk export demo I'm working on, we are trying to demo how a client's granted OAuth scopes can be mapped to security-label typeFitlers to ensure that the exported results will only include those resources that the client is authorized to see.
In this use-case it is very desirable to have support for wildcard typeFilter parameters which orthogonally apply to all the exported resources, regardless of resource type or other filter parameters.
I have added this to Hotaru Swarm and the format is simply to replace the resource type with *, for example: *?_security=N. This can nicely accommodate the _since parameter as well because the _since parameter can be mapped to a wildcard filter: *?_lastUpdated=gt2019-04-22.
I'd be interested in hearing others' thoughts on this.
