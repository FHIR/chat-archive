Peter Jordan (Jul 11 2019 at 03:29):

For Code Systems, I believe that should be determined by their owner/custodian. If it's the Ministry then we already have a base Naming System Identifier URL...
https://standards.digital.health.nz/codesystem
an example being...
https://standards.digital.health.nz/codesystem/ethnic-group-level-4
For Value Sets, it should be the canonical URL that represents the literal address of the authoritative definition of the Value Set. This does not prevent other FHIR servers from holding any given Value Set and, indeed, I'd be happy to place them on Terminz until such time as the MoH can host them (I already have the NZ Ethnicity ones).
