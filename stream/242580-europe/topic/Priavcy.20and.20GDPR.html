Jose Costa Teixeira (Nov 03 2020 at 13:30):

From the Security on FHIR call yesterday:
We want to see how we can use the resources (Consent, Permission, Provenance) to support a decent/good implementation of GDPR and others.
We will bring some use cases from Belgium, around

  1. Declaring "Processing Activities" à la GDPR, but using FHIR
  2. Maintaining a system's Permissions (which professionals can access/process which information for which purposes)
  3. Enabling the patient to control the data (e.g. right to be informed/right of access/ right of erasure ma non troppo)
  4. studying how this works on broad use cases e.g. "what if a patient gets a CarePlan, do the sub-plans inherit permissions?"
