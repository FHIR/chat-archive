The message-events ValueSet current has no members. I'm sure there are some example events and actions we could define that would be minimally useful, at least for demonstrating how $process-message would work. One might be "extract" which when $process-message is called would basically do the same as POSTing a transaction bundle with POST requests for each entry. Thoughs?

Lloyd McKenzie (May 04 2019 at 20:38):

We yanked all codes because the ones we had had little/no thought behind them and even less consistency and we didn't want implementers to start using them on the premise that they were legitimate. As well, almost all messaging interfaces that were being/have been implemented have been in closed communities that defined their own event codes.

Lloyd McKenzie (May 04 2019 at 20:38):

That doesn't mean that HL7 International couldn't actually define event codes (and corresponding MessageDefinitions) if there's a genuine desire to standardize messages at the international level.
