Github Notifications (FHIRcast) (Nov 20 2018 at 22:15):

isaacvetter opened Issue #47

We should update the FHIR examples to use FHIR r4 resources. Although the FHIRcast specification itself is independent of any given version of FHIR, the examples in the specification should either be multi-version or kept reasonably up to date. Since we're looking at balloting next year, r4 will be the most recently released version of FHIR.

Of particular note is that r4's ImagingStudy resource removed the uid and accession elements by collapsing that information into the identifier element.

Isaac

Credit to @gkustas and @ElliotSilver for questioning/pointing this out in issue #46 and zulip.
