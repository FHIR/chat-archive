It would be helpful to suggest that "sandboxes" (or possibly production) implementations provide a way to trigger a subscription for testing. We could actually do this by suggesting implementers provide a "test" button or UI to allow a consuming application to test their subscription endpoint.

However, a more important tool may be something closer to what CDS Hooks has: https://sandbox.cds-hooks.org/

In speaking with the CDS group (@kpshek), it sounds like having that website has been one of the most important tools in helping people adopt CDS hooks. This wouldn't solve production testing use cases, but does help subscribed applications test their implementations and see what code they may need to write.
