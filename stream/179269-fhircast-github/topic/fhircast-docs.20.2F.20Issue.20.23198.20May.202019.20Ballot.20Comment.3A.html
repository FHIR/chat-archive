isaacvetter commented on Issue #198

NOTE on ANSI modal verbs: The usage of ANSI modal verbs consistently provides a vehicle for quickly discerning the conformance criteria.
HL7 adheres to ISO/IEC directive, Appendix G: ANSI modal verbs ("SHALL", "SHOULD", "MAY", "SHALL NOT", "SHOULD NOT", "MAY NOT") SHALL be capitalized. See FHIR conformance rules, section 2.1.0.1 "Conformance Language". http://hl7.org/implement/standards/fhir/conformance-rules.html

Github Notifications (FHIRcast) (May 06 2019 at 14:14):

isaacvetter commented on Issue #198

Tony,

This isn't intended to be a mandatory statement, rather a statement of fact. The required behavior is that the Hub must no send notifications to a subscriber once lease_seconds has expired.

Existing Wording:

The Hub-determined number of seconds that the subscription will stay active before expiring, measured from the time the verification request was made from the Hub to the subscriber. Subscribers must renew their subscription before the lease seconds period is over to avoid interruption.

Proposed wording:

The Hub-determined number of seconds that the subscription will stay active before expiring, measured from the time the verification request was made from the Hub to the subscriber. If provided to the client, the Hub SHALL unsubscribe the client once lease_seconds has expired. Subscribers must renew their subscription before the lease seconds period is over to avoid interruption.
