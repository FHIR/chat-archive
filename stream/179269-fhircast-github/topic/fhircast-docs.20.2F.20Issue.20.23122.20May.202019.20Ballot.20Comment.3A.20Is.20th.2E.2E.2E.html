Hey Ricardo,

Is the user's session the topic (hub.topic)? Is hub.topic always associated to a user's session?

In the case where there's a user, yes, it is.

Isaac

Github Notifications (FHIRcast) (Jun 07 2019 at 10:56):

RicardoQuintano commented on Issue #122:

Hi Isaac,

Here is a suggestion:
"Accompanying this scope grant, the authorization server SHALL supply the cast-hub and hub.topic.
These parameters identify the Hub's base URL and a unique, opaque identifier of the current topic.
When there is only a single user involved in the synchronization of disparate systems, the user's session ID can be used to identify the topic."

Ricardo

Github Notifications (FHIRcast) (Jun 27 2019 at 14:46):

isaacvetter commented on Issue #122:

## :telephone_receiver: II Working Group Vote (6-27-2019)

Meeting notes: https://confluence.hl7.org/display/IMIN/Teleconferences

@RicardoQuintano moved the following disposition, seconded by @wmaethner

Disposition: Persuasive with Mod
Disposition Comment: There are potential multi-user use-cases in which it would be useful to share a single session among multiple users, such as multi-disciplinary team meeting, perhaps remote telemedicine. These use-cases are not the primary FHIRcast use-case, but also seem relatively straightforward to implement with FHIRcast. We will change this sentence:

These parameters identify the Hub's base url, and a unique, opaque identifier of the current user's session

to

These parameters identify the Hub's base url, and a unique, opaque identifier of the current session

:+1: For: 8
:expressionless: Abstain: 0
:-1: Against: 0

:tada: The motion passed! :tada:
