## May 2019 Ballot Comment: Request for Websocket to be officially supported, to enable Single Page Applications to use FHIRcast.

Submitted by Ron Parker on behalf of Kenn Sinn
HL7 Canada (ken.sinn@ehealthontario.on.ca )
Chapter/section: Security Considerations
Url: http://fhircast.org/security-considerations/#experimental-websockets-support
Type: A-S Enhancement
In Person requested: Yes :bust_in_silhouette:

Summary: Request for Websocket to be officially supported, to enable Single Page Applications to use FHIRcast.

Comment: The prescribed model for intent verification only works for server based architecture due to the fact that you cannot post to a single page app. In order to support single page app FHIRcast should support websocket as it is supported in the sandbox project. Can this be an officially supported feature of FHIRcast?
