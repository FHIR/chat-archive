## May 2019 Ballot Comment:

Submitted by Anthony Julian
Chapter/section: Event Notification Request Details
Url: https://fhircast.hl7.org/specification/May2019Ballot/index.html
Type: NEG :exclamation: Clarification
In Person requested: Yes :bust_in_silhouette:

Summary:

Comment: The descripton of context says there is an array of named fhir objects, then goes on to list resources by example, and restricts the return to FHIR resources.

Existing wording: An array of named FHIR objects corresponding to the user's context after the given event has occurred. Common FHIR resources are: Patient, Encounter, ImagingStudy and List. The Hub MUST only return FHIR resources that are authorized to be accessed with the existing OAuth2 access_token.
Proposed wording: An array of named FHIR objects corresponding to the user's context after the given event has occurred. Common FHIR resources are: Patient, Encounter, ImagingStudy and List. The Hub MUST only return FHIR resources that are authorized to be accessed with the existing OAuth2 access_token.
