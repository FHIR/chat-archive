Github Notifications (FHIRcast) (May 21 2019 at 14:13):

isaacvetter commented on Issue #161:

@bvdh, how about:

Proposed wording: Hubs SHALL send one or more json FHIR resources in the context

The intent is that a hub could perform a RESTful GET / FHIR read on it's FHIR server, shove the resulting json object into the context and then send the notification.

Github Notifications (FHIRcast) (May 22 2019 at 11:23):

bvdh commented on Issue #161:

@isaacvetter
The FHIR resources and fields are defined in the events. The source of the data could be either the EMR context or the FHIR server.
The data to be included is at least the elements mentioned, potentially more.

Proposed wording:
Hubs SHALL include the elements of the FHIR resources defined in the context part of the event definition. Hubs MAY include other elements of these resources.
The source of these resources is the EMR context or the FHIR server. The hub SHALL return FHIR resources from the EMR context. If the resource is not part of the EMR context, it SHALL read them from the FHIR server.
