Topic: Requiring "id" on stratifier component in CQFM Measure

Miriam (Jul 02 2020 at 14:07):

The CQFM profile requires Measure.group.stratifier.id, but not Measure.group.stratifier.component.id. Is there a reason for that? I'm trying to understand how to reference the component ids during evaluation. http://build.fhir.org/ig/HL7/cqf-measures/StructureDefinition-measure-cqfm.html

I also seem to remember a discussion about replacing use of id with linkId - if someone can point me to the ticket that would be great - thank you!

Bryn Rhodes (Jul 02 2020 at 22:22):

Components are coordinated by code, since the stratifier values in that case are literally constructed from the components. The stratifier example in the base spec illustrates how that works: http://hl7.org/fhir/measurereport-hiv-indicators.xml.html (see the Option 3 - component section at the end of the example)

Bryn Rhodes (Jul 02 2020 at 22:27):

I looked for an issue discussing linkId, but can't find anything. There is this one though to clarify guidance about the use of the id element to coordinate: https://jira.hl7.org/browse/FHIR-25718
