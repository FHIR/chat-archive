Stream: cql

Topic: `timezone from` vs. `timezoneoffset from`

Zach Smith (Dec 20 2019 at 19:00):

The docs say the syntax for extracting the timezone offset from a DateTime value is timezoneoffset from MyDateTime. However, when looking through the CMS eCQMS (example: MATGlobalCommonFunctions), they use the syntax timezone from MyDateTime. Are one or both of these correct?

Chris Moesel (Dec 20 2019 at 20:17):

In CQL 1.3, it was timezone from. In CQL 1.4, it changed to timezoneoffset from. This is most visible in the formal grammars:

  • https://cql.hl7.org/STU3/grammar.html
  • https://cql.hl7.org/STU4/grammar.html

Zach Smith (Dec 20 2019 at 20:55):

Gotcha, thanks for the clarification

Zach Smith (Dec 20 2019 at 20:55):

Not sure if this is the appropriate place to ask questions about eCQMs, but do you know if there are plans to upgrade eCQMs to 1.4?

Chris Moesel (Dec 20 2019 at 23:28):

@Bryn Rhodes

Bryn Rhodes (Dec 21 2019 at 01:25):

The next round of eCQMs will use 1.4, yes. We are still using 1.3 for FHIR-based eCQMs, but expect to have that updated soon as well. For a list of breaking changes between 1.3 and 1.4, refer to the history list. The change from timezone to timezoneoffset was made as part of the ballot changes.
