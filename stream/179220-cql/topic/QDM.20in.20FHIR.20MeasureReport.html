Topic: QDM in FHIR MeasureReport

Patrick Sexton (Aug 29 2018 at 17:26):

We are wanting to put the results of the eCQM measures into the FHIR MeasureReport resource. I want to be able to report on which QDM elements were relevant in the calculation of each measure population (numerator, ipp, etc...). I am looking at putting these elements into the contained array of the MeasureReport, and using an extension on the population items to reference those contained elements. One issue is how do we put QDM elements into a FHIR resource? Has anyone done this before, or seen an approach for this?

Bryn Rhodes (Aug 29 2018 at 21:57):

Hi @Patrick Sexton , is this for a population or individual report?

Patrick Sexton (Aug 29 2018 at 22:53):

For an individual report, though I imagine there would be a similar question if it was a population.

Bryn Rhodes (Aug 30 2018 at 17:01):

So are you looking to include the actual data, or just an indication that there is data for a particular data criteria in the HQMF?

Bryn Rhodes (Aug 30 2018 at 17:09):

For the former, tricky because the data elements would be in a QDM-like format. For the latter, there would need to be a transformation from the QDM to the FHIR resources. There is a skeletal mapping for this: http://hl7.org/fhir/us/qicore/qdm-to-qicore.html and it is the focus of a scenario in the Clinical Reasoning connectathon track: http://wiki.hl7.org/index.php?title=201809_Clinical_Reasoning#QDM_Mapping
