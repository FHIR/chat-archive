Topic: Null argument semantics in includes vs. contains operators

Zach Smith (Dec 16 2019 at 19:14):

The docs for the includes operator say that for the point overload, the operator is equivalent to the contains operator.

However, these two operators define different semantics when the first argument is null. includes declares that the result is null, while contains declares the result to be false.

A similar discrepancy can be seen in included in vs. in.
