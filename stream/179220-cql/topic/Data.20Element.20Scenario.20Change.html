We're seeking feedback on a proposed change to the Data Element Submission Scenario diagram. Specifically, we're proposing adding a Data Repository role to separate the functions of a "native" FHIR server and the "Submitting System" that processes measure specifications to gather the data and calculate measures/care gaps, etc. There is a tracker for the proposed change here: https://jira.hl7.org/browse/FHIR-34882 and a preview build of the change applied to the IG here: http://build.fhir.org/ig/HL7/cqf-measures/branches/fhir-34882-data-repository-role/capabilities.html
