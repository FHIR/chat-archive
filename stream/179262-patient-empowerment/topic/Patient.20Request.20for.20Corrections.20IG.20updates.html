Topic: Patient Request for Corrections IG updates

John Keyes (Aug 04 2021 at 19:47):

I have updated the Patient Request for Corrections draft IG (v0.3.0) with the following changes, which we (@Debi Willis , @Virginia Lorenzi , @John Moehrke , @John Keyes ) discussed in our 2021-08-03 call:

  1. communication.about is now a reference to the original communication associated with the correction request (unless this is the original communication, in which case this will be empty).
  2. task.reasonReference (for a correction request, medRecCxReq) now is a reference to the original communication associated with the correction request.
  3. task.reasonReference (for a disagreement, medRecCxDenialDisagree) now is a reference to the original communication associated with the correction request (it is not a reference to the first communication of the disagreement).
  4. A new search parameter has been added for communication.about, allowing a client to find all communications associated with a corrections request.
  5. A new search parameter has been added for task.reasonReference, allowing a client to find the task(s) associated with a correction request. (There should be a Task created as a result of the original correction request, and also a Task created as a result of any disagreement.)

Comments welcome.
