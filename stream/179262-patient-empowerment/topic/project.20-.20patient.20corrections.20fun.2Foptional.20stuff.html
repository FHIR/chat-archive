The other thing I'd like to explore in connectathon is "structured requests" -- it's a feature Michele will rightly push back against as beyond-the-bare-minimum, but I think it'd be good to experiment. Basically: can I include in my request a bundle of FHIR Patches (or similar) that specifies exactly how I'd like things to change. An administrator would still review and accept/reject this, and wouldn't ever need to apply my patches verbatim... but I think it opens the door to more precise conversations and (over time) more automation.

Lloyd McKenzie (Dec 03 2020 at 00:16):

My general leaning is that that's a cool idea for a phase II. Not sure we necessarily want to bite it off initially. First, reality is that much of the data patients will want to change won't be exposed via a REST API at all. Also, actually making the change is (typically) a tiny part of the cost, so expending lots of effort in automating it isn't necessarily a huge value.

Josh Mandel (Dec 03 2020 at 00:42):

I agree with the assessment here; still think it's worth some early exploration.
