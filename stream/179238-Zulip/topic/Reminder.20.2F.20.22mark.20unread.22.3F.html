Tim Abbott (Jan 21 2021 at 23:28):

The ability to mark things unread is definitely something we'll build eventually. But it's unlikely to be in the next few months, just because it'd be a bit of a complex coordination to change, as we'd need to audit a lot of logic for anywhere where we take advantage of the current invariant that messages only go from unread to read.

The starred approach that folks mention above (combined with the "Display setting" to show starred messages counts) should definitely help, as will upcoming changes to make it easier to avoid marking things read in some common flows.
