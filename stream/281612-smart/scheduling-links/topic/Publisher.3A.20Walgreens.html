Josh Mandel (May 13 2021 at 20:27):

I've got an updated version of Walgreens data being published now to https://wbaschedulinglinks.blob.core.windows.net/fhir/$bulk-publish -- a few notes:

  • Added NPI for each location -- these are available for each pharmacy location and presumably are good cross-organizational identifiers. Do others have/use NPIs for locations? (FHIR defines an Identifier System of http://hl7.org/fhir/sid/us-npi for these, which is what I'm using.)
  • Added lat/lon for each location
  • Added county for each location
  • Latency of slot data is now ~0-5 min, rather than the 0-20min I had previously
