Ward Weistra (Dec 28 2021 at 16:32):

@Spencer Crosswy Hi! The FHIR Package Registry is indeed really meant to find the FHIR package you need based on a bunch of filters. Like, packages for the US realm containing a ValueSet related to Covid.
And just to be sure, the resource filter 'Patient' will currently give you (packages containing) instances of Patient, not profiles on Patient. For that you will need the filter on instances of StructureDefinition (which profiles are) and there currently is no way to explicitly filter on the base type.

What's the use case behind your question? Are you looking to find specific patient profiles, or do you really need all Patient profiles ever published in a FHIR package to run statistics on them?
