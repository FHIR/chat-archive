Is Testscript.metadata supposed to be called metadata? I find it a strange name for what it means

Jose Costa Teixeira (Sep 13 2021 at 07:53):

looking at the description of the data elements, I'd think a name like "expectations" or "requirements" would be more suitable

Richard Ettema (Sep 13 2021 at 20:37):

The name 'metadata' was originally meant to convey that the information within the metadata structure would carry different types of high-level information about the conformance requirements that were being tested. The current description of 'metadata' does convey this:

Required capability that is assumed to function correctly on the FHIR server being tested

That being said, yes, I agree the name 'requirements' or something similar would likely be better.
