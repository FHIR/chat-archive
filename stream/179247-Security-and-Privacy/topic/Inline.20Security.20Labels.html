The FHIR DS4P IG proposal for inline security labels (for labeling parts of a resource) is up in a PR and you can read the write-up here:
https://github.com/HL7/fhir-security-label-ds4p/blob/feature/inline-labels/input/pagecontent/inline.md

This addressed some of the use-cases that called for sub-resource security labeling in this stream including the use-case raised by @Josh Mandel in this thread:
https://chat.fhir.org/#narrow/stream/179247-Security-and.20Privacy/topic/Representing.20IAL.20in.20FHIR

I have included an example that directly shows how this can help with the use-case raised in the above thread:
https://github.com/HL7/fhir-security-label-ds4p/blob/feature/inline-labels/input/examples/extension-inline-sec-label-immunization.json

This hasn't been merged so feedback and comments are appreciated!
