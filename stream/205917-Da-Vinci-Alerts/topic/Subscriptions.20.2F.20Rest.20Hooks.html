Patrick Haren (May 12 2020 at 19:31):

Good question Matt. There is a view that FHIR Subscriptions can supercede the need for a separately structured specification like Da Vinci Alerts. I'm thinking that there might be a need for both. If we attempt to map the concepts in DV Alerts into the subscriptions infrastructure, this might make it apparent. For example, how would the source systems produce alerts for different sets of member/patient panels (by payer, ACO, etc.), or would this still be an intermediary function (an Event Notification Service). Assuming that it is still an intermediary function, should that service transmit alerts using fhir subscriptions as well? Does it still fit the specification?
