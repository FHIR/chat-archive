Reminder that the url fields of many of the MeasureReport resources in the examples.json.zip file are currently incorrect. All of the "FHIR-*" ones.

Incorrect: http://hl7.org/fhir/us/saner/FEMADailyHospitalCOVID19Reporting

Anyway on the NextGen/Mirth server I've already updated those resources, so if you search on MeasureReport by the measure search parameter, make sure to use the correct URL, like so:
http://ec2-34-210-212-20.us-west-2.compute.amazonaws.com:9001/MeasureReport?measure=http://hl7.org/fhir/us/saner/Measure/FEMADailyHospitalCOVID19Reporting

Hopefully this spares some headaches, cheers
