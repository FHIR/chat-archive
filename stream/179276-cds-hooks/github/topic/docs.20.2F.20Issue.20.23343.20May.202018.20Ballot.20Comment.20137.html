## May 2018 Ballot Comment 137

Submitted by Nell Lapres on behalf of @isaacvetter from Epic

Chapter:
Section:
Type: A-S :bulb:
In Person Requested? No

Comment:
In the Example prefretch templates section, the prefetch template keys would be more readable if they were named something like "patient", "hemoglobin-a1c" and "user"; rather than "p", "a1c" and "u". While the documentation's goal to illustrate that the prefetch keys can be named whatever is laudable, it's actually not that important. In practice, it would be a benefit to the community if "patient", "user", etc were commonly used by CDS services to describe the current patient and user in prefretch templates.
