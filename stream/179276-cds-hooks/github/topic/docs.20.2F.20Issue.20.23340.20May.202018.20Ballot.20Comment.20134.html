## May 2018 Ballot Comment 134

Submitted by @kensaku-kawamoto on behalf of @gdelfiol from University of Utah

Chapter:
Section: https://cds-hooks.org/specification/1.0/#card-attributes
Type: NEG :exclamation:
In Person Requested? Yes :bust_in_silhouette:

Existing Wording:

Allowed values, in order of increasing urgency, are: info, warning, hard-stop.

Comment:
This value set is mixing urgency indicator with user experience. If the goal is to convey urgency, it would be better to use values that do not bind to any particular user experience, such as low, medium, and high, and let the EHR decide how to render each level in the UI.

## :de: Köln May 2018 Working Group Vote

@jmandel moved the following disposition, seconded by @brynrhodes.

Disposition: Persuasive with Mod
Disposition Comment:
Rename hard-stop to critical for the 1.0 specification.

:+1: For: 25
:expressionless: Abstain: 0
:-1: Against: 0

:tada: The motion passed! :tada:
