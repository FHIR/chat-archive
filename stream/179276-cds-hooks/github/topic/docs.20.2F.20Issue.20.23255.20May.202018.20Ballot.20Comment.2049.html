## May 2018 Ballot Comment 49

Submitted by @bvdh from Philips Healthcare

Chapter: CDS Services
Section: Passing the Access Token to the CDS Service
Type: A-S :bulb:
In Person Requested? Yes :bust_in_silhouette:

Existing Wording:

If your CDS Service has no decision support for the user, your service should SHALL return a 200 HTTP response with an empty array of cards.

Proposed Wording:

If your CDS Service has no decision support for the user, your service SHALL an empty array of cards.

Comment:
Empty is not optional.

## Triage Information

Proposed Disposition Comment:
