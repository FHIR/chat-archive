## May 2018 Ballot Comment 128

Submitted by @kensaku-kawamoto on behalf of @gdelfiol from University of Utah

Chapter:
Section: https://cds-hooks.org/hooks/
Type: NEG :exclamation:
In Person Requested? Yes :bust_in_silhouette:

Comment:
The distinction between context and prefecth is not clear and the line between what goes in the context and prefetch is fuzzy. For example, based on the description provided, the medication-prescribe hook should include in the context all medications the patient is taking, not just the ones being prescribed, since most medication CDS (e.g., DDI checks, duplicate meds) requires looking at current meds. One option is to say that context is for data that is being collected in the current session and therefore is not available through prefetch or calls to the FHIR server.

## :de: Köln May 2018 Working Group Vote

@isaacvetter moved the following disposition, seconded by @olbrich.

Disposition: Persuasive with Mod
Disposition Comment:
We will improve the documentation of prefetch and context to make it more clear for implementers as the ballot comment shows a misunderstanding of context.

We will also look at referring to context everywhere as "hook context", though the field will remain named simply "context".

:+1: For: 11
:expressionless: Abstain: 0
:-1: Against: 0

:tada: The motion passed! :tada:
