## May 2018 Ballot Comment 75

Submitted by @cmoesel from MITRE

Chapter: Providing FHIR Resources to a CDS Service
Section:
Type: A-S :bulb:
In Person Requested? No

Existing Wording:

FHIR resources requested in the CDS Service desciption are passed as key-value pairs, with each key matching a key described in the CDS Service description, and each value being a FHIR resource.

Proposed Wording:

FHIR resources requested in the CDS Service desciption are passed as key-value pairs, with each key matching a key described in the CDS Service description, and each value being a FHIR resource (in the case of searches, a "searchset" Bundle containing a set of FHIR resources).

Comment:
On first read, the expectation that a prefetch value is a single resource can be confusing until you realize that search results come back as a single bundle containing multiple resources. I suggest the spec notes this to avoid confusion. My proposed wording is an example -- modify as you wish.

## :de: Köln May 2018 Working Group Vote

@kpshek moved the following disposition, seconded by @olbrich.

Disposition: Persuasive with Mod
Disposition Comment:
Agreed. We will add something like "For example, in the case where the prefetch template is a search, the prefetch query will return a Bundle"
