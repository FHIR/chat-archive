kpshek opened Issue #194

In an offline discussion with @brynrhodes, @isaacvetter, and @brettmarquard, we came to the conclusion that we really need a distinct repository for the specification content. This will help us be crisp on what content is part of the specification and what content is just supporting documentation, implementation guides, etc.

This will also help with the HL7 specification publication process. This will also affect the current website structure.

A straw man proposal:

  • Our existing docs repository will store all content BUT the specification.
  • Create a new specification repository to store the specification content:
    -- https://github.com/cds-hooks/docs/tree/master/docs/specification/*
    -- https://github.com/cds-hooks/docs/blob/master/docs/hooks/index.md
  • Move the specification content from http://cds-hooks.org to http://specification.cds-hooks.org
