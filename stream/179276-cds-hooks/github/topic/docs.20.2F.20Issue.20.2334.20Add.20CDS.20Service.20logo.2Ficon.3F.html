Github Notifications (May 10 2017 at 11:28):

kpshek commented on Issue #34

At the Madrid Connectathon this past weekend (2017-05-07), a large group of us participating in the CDS Hooks track discussed this issue in an offline discussion. This group of ~12 represented a broad set of stakeholders, from multiple EHR vendors, several CDS Service providers, and a healthcare organization. What follows is the summary of our discussion and the consensus of the group.

We agreed that we should add an optional field on each card for this image. Additionally, we agreed that should be an icon, not a logo. Icons have specific dimensions and other requirements (see for reference Apple's iOS app icon requirements).

We did not discuss concrete requirements for the icon. That will need to be proposed here.

Github Notifications (May 11 2017 at 14:05):

isaacvetter commented on Issue #34

Hey Everybody - a couple of points:

1) This is a nice feature to support. I don't think it's vital to a 1.0 release.
2) I read Apple's App Icon UX guidelines (per @kpshek's link). They're solid UX guidelines, but don't help us figure out what should be in the spec.

I think that the primary questions are:
* What size and shape should the icon be?
* Do we need to support multiple icons?

Continuing the analogy to Apple, here's the list of sizes that they require. (Note that they're all square).

@gotdan pointed to slack's card format as an analogy in #33, but I don't think they're supporting dynamic icons.

I'm going to propose that cards can define a small square icon of 40x40px (apple spotlight size) and a large icon of 120x120px. The card should point to a url.

Thoughts?
