## May 2018 Ballot Comment 17

Submitted by @bvdh from Philips Healthcare

Chapter: CDS Services
Section: Providing FHIR Resources to a CDS Service
Type: NEG :exclamation:
In Person Requested? Yes :bust_in_silhouette:

Existing Wording:

The second enhancement enables the CDS Service to retrieve FHIR resources for itself, but to do so more efficiently than if it were required to request and obtain its own authorization. If the EHR decides to have the CDS Service fetch its own FHIR resources, the EHR obtains and passes directly to the CDS Service a bearer token.

Comment:
Is passing an authorization optional? According the CDS http request it is. I understand that not all CDS Services will require such access. But this allows a call that can not be handled as access is required. Maybe we should add a field in de CDS Service description where a CDS Service can state its requirements (pre-fetch, fhir-server, none).

## Triage Information

Triage Notes:
Reviewed with Bas

Proposed Disposition: Persuasive with Mod
Proposed Disposition Comment:
We agree this would be useful functionality and is something we will explore in a future release.
