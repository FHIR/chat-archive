Topic: ValueSet OID not available in HL7 OID registry

Michelle (Moseman) Miller (Mar 06 2020 at 14:00):

There are a bunch of JIRAs logged (across multiple work groups) stating that various value set OIDs are not available in HL7 OID registry. Are we expected to be adding those value set OIDs to the registry?

Lloyd McKenzie (Mar 06 2020 at 15:07):

Individual work groups definitely are not. It's not clear if it's something we'll bother doing in bulk. @Grahame Grieve ?

Grahame Grieve (Mar 06 2020 at 20:06):

I don't know. would HL7 have to pay itself? @Ted Klein do we have a policy on this?

Lloyd McKenzie (Mar 06 2020 at 21:51):

We auto-import new v3 code systems and value sets. Presumably that would be possible for FHIR content too. My hope/expectation is that the OID registry will get sucked into UTG using the same change request approach as everything else

Ted Klein (Mar 06 2020 at 22:50):

Note sure about the OID registry itself Lloyd. But we do have a policy at HL7 that OIDs created by HL7 WGs for objects in balloted HL7 published IGs and Standards (those in work that will become this when finished) are no charge whoever registers them. As far as OIDs for FHIR project items, for any that fall under the above policy, we can import them in bulk (a simple matter of a wee bit of data manipulation); we have done this in the past for things like batches of Affiliate OIDs. For FHIR connectathon projects done by non-HL7 groups, then that would have to be ruled on by HQ.
