The http://wiki.hl7.org/?title=FHIR_Spreadsheet_Authoring#Events_Tab says "The use of the Events tab (for defining messaging events) is currently incomplete. If you need to use it, please discuss with the FHIR tooling team (post to the Committers list)."
Patient Care just approved GF#8048 to add a new communication request event. Is there any guidance on how to fill out that tab (i.e. what each column means)?

Grahame Grieve (Nov 04 2016 at 18:47):

Event Code - the actual code that goes in the message - lowercase, no spaces
Category - Consequence | Currency | Notification - see messaging page
Description - text description of the event
Notes - more text description that goes in the last column - advice to implementers
Request Resources - comma delimited list of resources that go in request message
Response Resources - comma delimited list of resources in the response
Request Aggregations - leave this blank
Response Aggregations - leave this blank
Follow Ups - list of events (event code) that would be expected to follow this message exchange
