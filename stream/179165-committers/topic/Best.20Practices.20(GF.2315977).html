Topic: Best Practices (GF#15977)

Grahame Grieve (Jul 07 2018 at 01:43):

I have just done the build support for GF#15977, which now allows for committees to define 'best practices'. These are documented as things that are best practices for implementers, along with a reason why it is a best practice. Implementers can choose to ask the validator to enforce best practices, but it won't enforce them by default

Grahame Grieve (Jul 07 2018 at 01:43):

this is particularly relevant for CIMI, which seeks to define best practices.

Grahame Grieve (Jul 07 2018 at 01:44):

http://build.fhir.org/best-practices.html will document these

Grahame Grieve (Jul 07 2018 at 01:44):

committees are encouraged to define their known best practices (things that implementers should do, but can't be made mandatory because of legacy records, for instance)
