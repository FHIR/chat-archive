Lloyd McKenzie (Mar 30 2017 at 19:05):

A profile can be referenced by multiple resource instances. A profile can only apply to a single resource or data type though - you can't define a profile on Observation and then apply it to Procedure, for example.

Paul Knapp (Apr 06 2017 at 06:21):

1. A profile is a 'profile on a resource' and other resources may refer to a resource of the type addressed by a profile. For example, Latvia may create a nation profile on Patient, then in their implementation guides on other resources wherever the Patient resource is referred to they indicate to use the Latvian profile on Patient.
2. While some extensions are resource-specific, others such as 'race' or 'birthplace' or a 'non-Gregorian calendar' may be appropriate to use on many resources. Essentially it would be appropriate to use an extension wherever the context in the resource matches the defined context-of-use of the extension.
