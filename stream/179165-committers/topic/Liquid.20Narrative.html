Topic: Liquid Narrative

Grahame Grieve (Apr 13 2019 at 21:11):

If you want to use liquid to control the narrative generation, you add a "liquid" property to the paths object in the json configuration file. That folder that contains a set of liquid templates (see http://wiki.hl7.org/index.php?title=FHIR_Liquid_Profile) where the file name is [resource-type].liquid. When narrative is built for a resource, the nominated liquid template will be used rather than the default narrative generation

Lloyd McKenzie (Apr 14 2019 at 02:07):

To confirm, should the root node of the [resource-type].liquid file be a <div/> element or should it be the content of the div? (The former would be better for those of us who hope to do some degree of schema validation, though I suspect the location of the liquid commands will mess that up some of the time.)

Grahame Grieve (Apr 14 2019 at 07:01):

yes must be a single root which is the div
