Topic: Consequences of renaming an HL7 GitHub repository

Rick Geimer (Jan 31 2019 at 00:17):

Hi all,

Wanted to know people's thoughts on renaming HL7 GitHub repositories. The use case I have involves the Da Vinci group, which recently decided to rename some of their IGs going to a comment-only ballot in May, and so I would like to rename the repositories to match the new acronyms. For example: davinci-ehrx will become davinci-hrex.

Would like to hear people's thoughts around the consequences, if any, of such a renaming.

Grahame Grieve (Jan 31 2019 at 00:25):

my thoughts:
- we have no insisted that they be the same, and lots of IGs aren't - though I prefer it when they are
- what happens to any existing local repositories? do they get orphaned?
- it leaves behind the old build on the current build - but I think I'm ok with that?

Rick Geimer (Feb 01 2019 at 17:04):

Perhaps we should suggest that implementers who do this make one last commit to the index.html page before renaming, noting at the top that this version of the IG is no longer current/valid, and include a pointer to the current build location with the new name?

Rick Geimer (Mar 17 2019 at 07:56):

@Grahame Grieve Do we have a final decision on this?

Grahame Grieve (Mar 17 2019 at 11:02):

well no one objects, so go ahead
