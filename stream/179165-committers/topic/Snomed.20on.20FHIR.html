Donna Truran (Aug 13 2018 at 05:59):

in principle it's not wrong to bind to multiple heirarchies but in particular cases it could be a very bad idea. I don't have an opinion on this particular value set

It's not wrong to use content from multiple hierarchies, though caution is needed. It will be necessary to use multi-hierarchy concepts to fulfil real world use cases. The ValueSet, as it stands, doesn't include any concepts from different (numerous) hierarchies because it defines is-A 242990004 and there are no child concepts of 242990004 (and event) anyway.
