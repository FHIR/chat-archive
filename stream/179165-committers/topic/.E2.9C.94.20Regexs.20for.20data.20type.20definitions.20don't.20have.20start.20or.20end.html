Topic: ✔ Regexs for data type definitions don't have start or end

Richard Townley-O'Neill (Dec 09 2021 at 05:19):

The Regex definitions at http://hl7.org/fhir/R4/datatypes.html do not include string starts and ends. For example integer Regex is
[0]|[-+]?[1-9][0-9]*
This will treat 23.45 as two matching strings. Is this intended?
A Regex of
^0$|^[-+]?[1-9][0-9]*$
will treat this a a non-matching string.

Why not have ^ and $ in the various Regexs?

Lloyd McKenzie (Dec 09 2021 at 06:31):

Richard Townley-O'Neill (Dec 09 2021 at 06:41):

FHIR-34436

Chris Moesel (Dec 09 2021 at 13:52):

The spec does actually account for this. There is a section below all the primitive descriptions and regexes that indicates the regexes have some issues. One of the bullets it calls out says:

  • The regexes should be qualified with start of string and end of string anchors based on the regex implementation used (e.g. caret '^' and dollar-sign '$' for JavaScript, POSIX, XML and XPath; '\A' and '\Z' for .NET, Java, Python and others; please verify these definitions with the regex implementation used).

So it sounds like they were intentionally left out because some regex flavors apparently use other anchors.
