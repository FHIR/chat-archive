Corey Spears (Jun 29 2021 at 13:56):

Sorry for a delayed response here. This is something I am still trying to unpack. It seems like it may be related to another issue of this IG regarding _include brought up here: https://chat.fhir.org/#narrow/stream/204607-CARIN-IG.20for.20Blue.20Button.C2.AE/topic/_include.3A*
_include parameters are supposed to based on search parameters (thus the additional search parameters you found in an automatically generated portion of the IG). The CARIN BB IG did not include those search parameters in the CapabilityStatement. These search parameters are really not all that helpful in the context of the use case of the IG. The _includes were added as a matter of convenience for client applications so additional logic and API calls were not needed. This has caused a bit of an incongruence in the spec that we will look to fix in STU2.
