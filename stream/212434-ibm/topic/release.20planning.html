Topic: release planning

Lee Surprenant (Jan 25 2022 at 19:40):

The IBM FHIR Server team is making plans for our next release of the open source project. There are some big changes we’d like to make, including:

  • support for the forthcoming FHIR 4.3.0 (R4B) release
  • dropping support for Java 8
  • dropping the experimental “export to parquet” feature
  • changing the default behavior of whole-system history (already in main)

Because of these changes, we’re thinking to bump the major version on all modules, so this release would be tagged as 5.0.0.
To avoid confusion with the next major release of the FHIR Specification (R5), we’re thinking to break the tie between our “marketing version” and our module versions, so the next release would be called “IBM FHIR Server 2022-1” instead of “IBM FHIR Server 5.0.0" at https://github.com/IBM/FHIR/releases and in any announcement / communications.

If anyone has questions/concerns with this plan, please raise here or otherwise get in touch with us.

Lee Surprenant (Apr 05 2022 at 18:14):

FHIR R4B is still not published. Therefore, the IBM FHIR Server team has decided to revise our previously-communicated plan and release what we have in our main branch as version 4.11.0.
Of the 4 bullets above, this release will include everything except the FHIR 4.3.0 support. Please let us know if you have any concerns as we plan to tag the release soon.
