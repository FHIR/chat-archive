Grahame Grieve (Nov 17 2021 at 23:54):

Grahame Grieve: I'm just about to release a new IG publisher that adds two new warnings for HL7 published code systems

  • CodeSystems shall declare case sensitivity
  • code systems with a hierarchy SHALL have a hierarchyMeaning

UTG has 100s of examples of resources that fail one of these tests.

  • all the v3 code systems: "HL7 Defined CodeSystems SHOULD have a stated value for the caseSensitive element so that users know the status and meaning of the code system clearly"

  • 100s of FHIR code systems: "HL7 Defined CodeSystems SHOULD have a stated value for the hierarchyMeaning element so that users know the status and meaning of the code system clearly"

As far as I know, all the v3 code systems are case sensitive - and we already agreed that we'd label the accordingly. And I quickly reviewed the FHIR ones, and they're all is-a hierarchy

So we have 4 choices:

  • publish with the old IG publisher and keep our heads in the sand
  • publish with the warnings
  • let me fix the resources automatically, and then publish
  • have a big long discussion about what to do

I'm figuring that we'll go with the last option ;-)
