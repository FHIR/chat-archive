Topic: Metadata Datatype Resource Proposal

Brian Alper (Feb 24 2021 at 21:41):

In the HL7 CDS Work Group meeting March 3 we will have a deep discussion on March 3 to review the Metadata Datatype Resource Proposal.

The brief discussion of a MetadataResource Resource Proposal (changing the pattern behind many resources) noted a significant concern for backward compatibility with multiple normative resources.

A modified approach that does not have such severe backward compatibility concerns is to “ignore” the required metadata elements for basic FHIR functionality, and for the many ‘optional’ metadata elements provide a new Metadata Datatype that can support a common metadata framework but not require it.

The reworked Metadata Datatype Resource Proposal can now be found at https://docs.google.com/document/d/1KkHWpQxaqiUpY1mC4X00id10z4mtxOg4/
