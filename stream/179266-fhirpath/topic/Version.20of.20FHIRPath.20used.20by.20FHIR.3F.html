Topic: Version of FHIRPath used by FHIR?

Paul Lynch (Oct 16 2018 at 18:24):

When there is more than one version of FHIRPath, will each version of FHIR require a particular version of FHIRPath?

Bryn Rhodes (Oct 16 2018 at 19:36):

I don't think so, the intent was that all changes to the FHIRPath spec are aligned with the FHIR specification such that the normative specification should be able to be used by all the versions.

Grahame Grieve (Oct 16 2018 at 19:39):

R4 will specify and use the latest versions. The older specs use an older version, but I think it doesn't matter (I didn't use any of the type features were things changed)

Paul Lynch (Oct 16 2018 at 19:41):

That sounds safer to me. I was thinking that someone might send an R4 Questionnaire containing 1.1 of FHIRPath to system that understands R4 but only 1.0 of FHIRPath, but if R4 specifies 1.1 (or whatever), then it should be fine.

Paul Lynch (Dec 24 2018 at 14:05):

@Grahame Grieve I might have misunderstood your comment, but I was thinking that the page http://build.fhir.org/fhirpath.html would (by R4) contain a mention of a particular version of the FHIRPath spec.

Grahame Grieve (Dec 25 2018 at 09:30):

well, there's a challenge there. FHIR R4 depends on the fhirpath spec that isn't yet published. We're still considering what to do about that
