Topic: ✔ New Validation Rules coming your way

Grahame Grieve (Nov 17 2021 at 05:19):

Advance notice: the next release of the IG publisher will introduce some new validation rules:

For CodeSystem supplements:

  • they SHALL not have content, caseSensitive, and heirarchyMeaning elements

For other Code Systems:

  • they should have content, caseSensitive, and heirarchyMeaning elements
  • if they are published by HL7 they SHALL have a content element (see http://hl7.org/fhir/codesystem-definitions.html#CodeSystem.content)

The IG publisher will start validating these rules as of the next release. The fix is simple: define these elements in your code systems:

  • content = complete or example (for HL7 IGs, anything else will require approval by HTA)
  • caseSensitive=true (for HL7 IGs, anything else will require approval by ?TSMG)
  • hierarchyMeaning=is-a - If and only if you have a hierarchy (concepts that contain concepts). If you don't, don't populate the hierarchyMeaning element and you won't get a warning if you don't. (for HL7 IGs, If you want a value other than 'is-a', you need vocabulary approval)

Grahame Grieve (Nov 17 2021 at 05:21):

these have been forced on me by hunting bugs in the tools introduced by fragments and examples in UTG. Things that used to work are breaking now if these things are not populated.

For non-HL7 guides, you can ignore the warnings, but I advise you strongly not to - the fix ix easy and the consequences downstream are nasty. For HL7 guides, obviously, FMG will not approve exceptions for these warnings (well, I don't think it will, but FMG hasn't discussed it yet)
