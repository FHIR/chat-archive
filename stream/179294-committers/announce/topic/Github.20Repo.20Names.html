Topic: Github Repo Names

Grahame Grieve (May 17 2019 at 01:12):

New naming standard for git hub repositories - the name should start with [xxx]- where [xxx] is the product family. e.g. cda- or fhir-

Grahame Grieve (May 17 2019 at 01:14):

(this is in the HL7 organization). Any new github repos must follow this pattern. talk to me if you don't know the product family. CDA product management group will mange the naming policy for cda- repos.

Grahame Grieve (May 17 2019 at 01:15):

Lot's of repos don't follow this pattern; as re-naming opportunities arise, I will look to standardize on the pattern
fhir-[realm]-[code] where realm and code are the designated realm and code for the IG the repo holds

Grahame Grieve (May 17 2019 at 01:15):

repos that don't hold IGs - we'll consider on a case by case basis

Grahame Grieve (May 17 2019 at 01:16):

any of you with the rights can rename repos you are working on, but please check with the working team before doing so.

Also, please commit an index.html that redirects to the new name before you do (so ci-build leaves a redirect behind on the old name).
