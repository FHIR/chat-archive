Amnon Shvo (May 13 2018 at 08:11):

That's a bug I came across in SImplifier validator and that was the answer from Firely:

The same validation error was noted in an issue on the .net API repo at: https://github.com/ewoutkramer/fhir-net-api/issues/424. This turns out to be an error in the FHIR spec itself. The constrain in the spec is not formulated correctly. A GForge ticket was already filed by Michel, which has been approved and which will be addressed in R4 of FHIR.

This is something our validator may need to skip. At the moment, you can also get past this validation error by putting another constraint before the slicing.
