Brett Esler (Jul 24 2018 at 22:58):

Medications working group is going around on PBS manufacturer code a couple of discussion topics
a) PBS manufacturer is not necessarily the factory source rather can be factory OR supplier i.e. a sponsor - we are discussing not using Medication.manufacturer and using an extension instead to represent this concept (Medication.manufacturer is used to represent the responsible physical product source only) - objections?
b) Reference to PBS manufacturer is via two letter manufacturer code - assertion is this is a logical identifier to an organisation that has a name, address, contact details etc. - would like a suitable URL to use as identifier system; we have a suggestion under pbs.gov.au but would like this ratified to include. This overlaps with NCTS concept of adding properties to PBS Code system the complexity of PBS definition is troubling me as seem more like rule based eligibility criteria than coding alone - any comments on this?
