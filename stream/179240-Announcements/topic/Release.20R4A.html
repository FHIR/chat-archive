Topic: Release R4A

Grahame Grieve (Jun 03 2020 at 22:58):

FMG is considering an interim release of FHIR R4, that introduces limited changes to R4, since it appears that the R5 timeline is slipping (due to the virus and other reasons, maybe early 2022?). The intent is that it will be published by start of Q2 2021, and that all implementers will be able to update their implementation to the new version simply by changing the version they claim to implement, except for some clearly defined areas where change is to be made - that is, introducing the new subscription framework, and updating the Medication Definition resources (MedicinalProductDefinition etc) and probably EBM resources (Evidence etc).

This raises several questions:

  • is this a good idea? Please discuss here: https://chat.fhir.org/#narrow/stream/179166-implementers/topic/R4A.20Release
  • how would versioning work? - whatever we do will break our existing infrastructure. Discussion on this from tool smiths etc at https://chat.fhir.org/#narrow/stream/179239-tooling/topic/Approach.204A.20Versioning.20Policy
  • what would the committers editorial policy be? discussion here: https://chat.fhir.org/#narrow/stream/179165-committers/topic/R4A.20Editorial.20Process
