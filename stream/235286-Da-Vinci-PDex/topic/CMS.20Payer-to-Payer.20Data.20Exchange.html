Milinda Perera (Sep 03 2021 at 09:10):

Hi team,

Following are some concerns that we have in discussion

If we consider data sharing, In FHIR perspective there are multiple ways to share data between payers:

  1. Via member mediated SMART-on-FHIR Application : For this approach, everything is fine until you hit the jackpot that the member needs is an account that works in the old payer portal. That is hard to come by, when you try to load records from a very old health plan because it is reasonable even in payer perspective to disable the account.

[Possible solutions to consider: 1. CMS rule to mandate member login in read-only mode for a time period which the payer needs to maintain records. 2. Impose and Introduce temporary account activation (read-only) mechanism]

  1. Payer push / pull - in this method upon member request (or without) new payer can pull or old payer can push records. Record pull case works fine. But in the push case there is no defined endpoint where the old payer can push data.

[Possible solutions to consider: Introduce bulk data import operation so old payer can perform member-match and do bulk data import or POST support for patient-everything]

Even with all above approaches, it won’t really solve interoperability among payers unless those aspects are at least added to an IG and best it is best if mandate in CMS rule.

I personally think the rule or IG should be strict on communication and data sharing protocol at first so all payers follow the same. That will even reduce organizational overhead that payers must face with manual workflows etc. when usage rises.
