Is there a shortcut to accessing the type for an element in the FHIR Client model.

e.g.:

import fhirclient.models.questionnaireresponse as QR

new_qr = QR.QuestionnaireResponse()

# instead of this
print(new_qr.elementProperties()[0][2])

# something like this

print( new_qr.id.type)    # which does not work BTW

Pascal Pfiffner (May 21 2018 at 16:21):

Properties are only accessible if they're assigned, otherwise you have to go via elementProperties() indeed.

Geoff Low (Jun 19 2018 at 20:53):

As a hack you might be able to do something like new_qr.id.__class__.__name__

Geoff Low (Jun 19 2018 at 21:13):

but the better way is to use type(new_qr.id)
