Topic: NLM SDC Questionnaire Tools - Update

Paul Lynch (Feb 20 2019 at 20:16):

The NLM SDC Questionnaire SMART app (https://apps.smarthealthit.org/app/lforms-questionnaire-app) has been updated with the latest version of the LHC-Forms to include the following changes:

  • Fixes for extensions used to represent units
  • QuestionnaireResponses now include the "subject" field, specifying the selected patient.

Work toward observationLinkPeriod support is still in progress.

Paul Lynch (Mar 14 2019 at 15:43):

The NLM Form Builder for building Questionnaires now includes an option for previewing and exporting the result in R4.

Paul Lynch (Mar 15 2019 at 21:09):

For anyone who actually tried the above link for the NLM Form Builder and couldn't get through, it turns out there was a firewall configuration issue blocking access. That is now fixed. Many thanks to @Oliver Egger for noticing and contacting me.

Paul Lynch (Mar 26 2019 at 22:40):

One of the feature requests that came from the last connectathon was for the NLM Form Builder to be able to import sections of LOINC panels in addition to complete panels. That feature is now live.

Paul Lynch (Mar 28 2019 at 19:17):

Prepopulation support via the observationLinkPeriod extension is now available in the NLM SDC SMART app, through the LHC-Forms package.

Paul Lynch (Mar 28 2019 at 19:17):

If anyone has an R4 SMART on FHIR server on which the app could be tested, please let me know.

Paul Lynch (Apr 24 2019 at 18:33):

The NLM SMART SDC Questionnaire App (which probably needs a shorter name) now allows you to enter your own FHIR server URL, for cases where you don't have a SMART interface for the FHIR server you want to use. If you go directly to the app's page without launching through something like the SMART app gallery, it will prompt you to enter a FHIR server URL and then pick a patient.

Paul Lynch (May 04 2019 at 20:02):

For the NLM SMART SDC Questionnaire App (and the LHC-Forms rendering widget)I just got observation-based extraction working with the R4 HAPI server, from one test Questionnaire. This is just in my development area at the moment, and is likely still buggy, but if anyone at the connectathon is particularly eager to try it, I could put push out an update to the public web app. Otherwise, it will probably be live within a couple of weeks.

Paul Lynch (Jul 03 2019 at 19:18):

I just updated the NLM SMART SDC Questionnaire App with some changes:

  • Observation-based extraction is working (which was there before, but I forgot to announce it)
  • New launch URL at https://lhcforms.nlm.nih.gov/sdc, which will launch with a SMART connection to an R4 (HAPI) server
  • Fix for Observation-based prepop so that prepopulation does not take place when rendered saved QuestionnaireResponses.
  • Updated its copy of LHC-Forms, which includes:
    • A fix to enable prepopulation of fields with answer lists
    • A fix for missing saved units for answers with unit lists

Paul Lynch (Sep 11 2019 at 22:12):

The Clinical Table Search Service can now be searched via FHIR $expand operations, as documented here: https://clinicaltables.nlm.nih.gov/valueSetExpand.html. This means that the data sets at the Clinical Table Search Service can be used in Questionnaires (answerValueSet) for autocompletion.
