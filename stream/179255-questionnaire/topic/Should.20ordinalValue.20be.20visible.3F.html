Topic: Should ordinalValue be visible?

Paul Lynch (Jan 29 2020 at 19:11):

When the ordinalValue (a.k.a. "score") extension is used on answerOption, should those score values be visible as a part of answer? LHC-Forms currently appends them to the answer text, e.g.: "Yes - 1", "No - 0". I think we did that because someone wanted the scores visible, but now we have a user requesting that they not be visible. Should the decision to show or not show answer scores be configurable at the Questionnaire level, or is that an application level decision?

Lloyd McKenzie (Jan 29 2020 at 19:24):

My leaning is that if you want the ordinal visible, you put it in the display value for the code. In general, ordinals shouldn't be visible because you don't necessarily want the user completing the form to be influenced by the weights.

Paul Lynch (Jan 29 2020 at 19:31):

What if list is from LOINC (or some other code system)? "If the 'display' element is populated, the string used in display SHALL be one of the display strings defined for that code by the code system".

Lloyd McKenzie (Jan 29 2020 at 19:52):

Then LOINC get to decide if the score should be visible or not I guess...
