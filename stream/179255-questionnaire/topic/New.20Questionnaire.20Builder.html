RP (Nov 02 2020 at 17:14):

Hello we've been working on a Questionnaire Builder these past 2 months. It's early
for us but we were hoping to preview it to get some feedback and see how we can improve going forward.
If you're interested in checking it out see https://augolit.dev. We have a WYSIWYG editor
that supports publishing your Questionnaires with a shareable URL. Thanks and let me know
if this is something you would use and if theirs an essential feature you would otherwise need.

Lloyd McKenzie (Nov 02 2020 at 17:44):

In terms of scope, are you just tackling the base FHIR spec or are you also including some of the Structured Data Capture functionality - and if so, what? :)

RP (Nov 02 2020 at 17:46):

Yep we want to add as a next step SDC capture for the next phase.

RP (Nov 02 2020 at 17:47):

We have a few ideas around SDC from a UI,UX standpoint (considering a table view to point from fields to StructureDefinition).

RP (Nov 02 2020 at 21:08):

More specifically what we want to support from SDC is essentially everything populate, extract, various other questionnaire extensions. We support a few of these pieces right now (calculatedExpression and a few validators) but adding more is a focus for us going forward. For populate and extract we would also like some way of unifying the two (doesn't seem to be part of the spec but for many cases I can imagine what you populate you would also want to extract). Our real end goal in all of this is to have a low code platform that makes it easy to create FHIR applications.
