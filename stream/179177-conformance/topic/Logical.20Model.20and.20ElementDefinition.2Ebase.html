Michel Rutten (May 20 2019 at 10:06):

Looks like you're trying to implement derived logical models, which Forge does not support. You may be able to hack it, but I've never tested this nor spent any time thinking about what this means. This probably explains why ElementDefinition.base component remains empty. Forge assumes that all element definitions in a logical model are derived from Element, repeating this everywhere would be redundant.
