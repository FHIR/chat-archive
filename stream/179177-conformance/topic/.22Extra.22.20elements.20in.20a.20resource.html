Lloyd McKenzie (Feb 25 2021 at 15:31):

Servers are allowed to ignore unexpected elements if they're trying to be future-version friendly (though that's really only 'safe' with normative resources, because with non-normative, the new element could be a modifier.)

Lloyd McKenzie (Feb 25 2021 at 15:32):

James Agnew (Feb 25 2021 at 22:10):

This depends on which ParserErrorHandler you have configured.. The default is the Lenient one that will log a warning, but there is also a Strict one that will flat out reject.

This applies to a handful of other possible parse errors too - E.g. using "valueBoolean": "true" instead of "valueBoolean": true and null values
