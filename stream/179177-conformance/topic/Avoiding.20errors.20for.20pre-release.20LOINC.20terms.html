Topic: Avoiding errors for pre-release LOINC terms

Morten Ernebjerg (Jun 08 2020 at 16:07):

Hi :wave: I'm working on validating Questionnaires related to COVID-19 and hence use some of the prerelease LOINC codes. At the moment, the standard Java validator complains that these are not valid codes. I tried adding them as a CodeSystem supplement for LOINC and load that via the -ig parameter, but that did not work. Is there a way to avoid errors being generated for these codes (without using a custom terminology server)?

Lloyd McKenzie (Jun 08 2020 at 16:19):

In short, no. Code system supplements can't define new codes and there's no way to introduce additional codes to an existing system via an IG artifact.

Morten Ernebjerg (Jun 08 2020 at 16:32):

OK, thanks! - I guess the next LOINC release is close anyway.
