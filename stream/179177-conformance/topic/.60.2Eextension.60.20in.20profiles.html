Josh Mandel (May 11 2017 at 17:34):

I'm reading profiles for BP and trying to re-familiarize myself with the representation. Is there a good way for me to understand when the .extension property is listed (vs. omitted) in a StructureDefinition? For example in http://hl7.org/fhir/bp.profile.json.html I see

{
  "id": "Observation.valueQuantity.extension",
  "path": "Observation.valueQuantity.extension",
  "slicing": {
    "discriminator": [
      {
        "type": "value",
        "path": "url"
      }
    ],
}

isn't this implicit from the datatype (Quantity)?

Michel Rutten (May 11 2017 at 19:43):

In the .NET API snapshot generator, we include elements in the diff if they have constraints. Otherwise they are implied and can be excluded. This applies to all elements, including "*.extension".

Michel Rutten (May 11 2017 at 19:44):

Note that this does not cause tooling issues, but it is just redundant
