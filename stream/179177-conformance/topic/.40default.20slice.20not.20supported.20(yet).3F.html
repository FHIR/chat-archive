Rob Hausam (Jan 20 2020 at 04:08):

I've considered using the @default slice in the past, but found that it wasn't supported. I thought I would try it again now, following the recent validator and publishing updates. I modified the Immunization-uv-ips profile by changing the slicing rule to closed and adding a new '@default' slice. There are no slicing-related build errors or warnings for the profile, but when validating the Immunization/75680 example it gives this error: 

ERROR: Immunization/75680: Immunization.vaccineCode: Could not match discriminator ([$this]) for slice Immunization.vaccineCode:@default in profile http://hl7.org/fhir/uv/ips/StructureDefinition/Immunization-uv-ips - the discriminator [$this] does not have fixed value, binding or existence assertions

It's complaining that there is no fixed value, binding or existence assertion for the @default slice discriminator, but according to the documentation those constraints are prohibited ("Default slices must not fix the value of the discriminator elements"). So I presume this means that it still isn't supported?
