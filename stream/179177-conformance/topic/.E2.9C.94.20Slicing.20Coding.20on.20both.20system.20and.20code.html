Topic: ✔ Slicing Coding on both system and code

Morten Ernebjerg (Aug 13 2021 at 07:37):

I would like to slice a Coding element to indicate the preferred use of certain codes for certain scenarios. This is similar to what the is done with the slicing directly on Condition.code in the IPS Condition profile. However, as I understood from this long discussion, slicing of non-repeating, non-polymorphic fields like Condition.code will not be (generally) supported, so I am trying use slicing directly on CodeableConcept.Coding instead. The slices I want are defined by single codes, but from different code systems. I thought this might work with pattern-slicing but there I am forced to pick out a single element inside Coding (at least Forge wants me to do that) so I cannot provide a pattern involving both system and code. I suppose I could also bind each slice to a different single-code ValueSet & discriminate on that, though that somehow seems a little heavy-handed. Would that be the recommended approach or is there another way to achieve this?

Alexander Zautke (Aug 13 2021 at 07:56):

Forge doesn’t offer you „$this“ when selecting the discriminator in the drop-down box. However, you can simply enter it directly. Then on each slice you can add a pattern with code and system.
