Michel Rutten (Aug 01 2018 at 13:21):

@John Moehrke In most computer languages, the dot . is a reserved character / has special meaning; usually to navigate to a specific object member. User-assigned identifiers cannot contain reserved characters, as this would break parsing. All code generators have to take this into account. In FHIR, code generators operate on conformance resources. Therefore, conformance resources require special invariants on relevant property values to ensure that generated code is syntactically valid.
