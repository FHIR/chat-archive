Michel Rutten (May 12 2018 at 10:39):

This message indicates an error in the STU3 SimpleQuantity datatype definition, that specifies a slicename on the root element. This is not allowed.
When you open or create any profile on a base resource that (directly or indirectly) references SimpleQuantity, then Forge will first generate the snapshot for SimpleQuantity and that will (always) trigger this error message.
You can safely ignore the error message. This is out of control of profilers and should actually be fixed in the FHIR standard (technical correction). Fortunately, as far as I know, this little bug has not caused any issues downstream, further than the Forge reporting the error.
