Topic: PractitionerRole in the Practitioner compartment instance

Stefan Lang (Jun 13 2018 at 18:55):

http://build.fhir.org/compartmentdefinition.html#bnr states:
"The practitioner compartment includes any resources where the resource is explicitly linked to a Practitioner (usually as author, but other kinds of linkage exist)"
For Practitioner, unlike the other compartments defined, there is PractitionerRole as an indirect way to link a Resource instance to a certain Practitioner. So, logically, every Resource instance that refers to a PractitionerRole of Practitioner X would belong to the Practitioner compartment instance X. This becomes more important, since PractionerRole itself becomes more important from R4.

So either indirect references/links via PractitionerRole should be included in the above text and in the Practitioner CompartmentDefinition.

Or, if there are reasons to not include that type of linkage, it should at least be explicitly stated that these indirect links are not included, maybe also on http://build.fhir.org/compartmentdefinition-practitioner.html .

I don't know if that has been discussed before, just came across that and didn't find any info.
