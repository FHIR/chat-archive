Topic: "Guess who" ?

René Spronk (Dec 07 2021 at 07:10):

For our FHIR training course it may be fun to create a kind of "guess who" (game) type decision tree: if one player were to randomly choose a resource type, how could the other player, using the lowest number of questions possible, find out (using simple yes and no questions) what resource they had in mind? (to show you a more serious side of this game: in essence this is also the question: if you have some conceptual/legacy data model in mind, what would be the fastest decision tree to find out which resource type it maps to?)

My off the bat starter questions:

  1. Does the resource type contain data for individual patients ? would be a good starter question, as it does split the number of resource types roughly in half.
    1.1 If yes: Does the resource type contain clinical data for individual patients ?
    1.2 If no: Is the resource type used to express conformance requirements ?

What would your starter questions be ?

Indeed - the above suggestions already illustrate there are multiple ways / 'dimensions' one could use to 'define sub categories' of resources. One aspect is however relevant: such questions would need to be largely release-agnostic, so 'maturity level' is not a very good bases for defining sub-categories, a resource having required elements is a shaky criterium (given that we have resources with low maturity levels). Questions will probably need to be more about functionality than about the technical artefacts that may be contained in a resources. But even then there are multiple ways to slice and dice resource categories.
