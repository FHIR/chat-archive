Tim Abbott (Apr 28 2017 at 17:22):

I think there's a recent version of Chrome that just leaks memory; we've had some reports of Zulip using a lot more memory than it used to in Chrome (even in servers running a fixed Zulip version). Typically, those users also found GMail was consuming several GB as well. Apparently, it can help to close and make a new tab (i.e. it appears Chrome is leaking memory across reloads of the browser window, which is definitely a bug). In theory, Zulip should not be a particularly memory-intensive application (and it certainly wasn't a few months ago).
