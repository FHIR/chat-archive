Talk 2: SMART on FHIR overview - @Adam Culbertson

The big thing about SMART on FHIR is that it provides a common interface for apps to talk to, which will work on any supporting EHR: Epic, Cerner, etc etc. In health IT this is huge news. Most ordinary people expect any competent website to work regardless of what computer you're talking to, but in healthcare that's never been true. SMART makes it possible, they say.

Next topic: The Bulk Data API

While a lot of health data transactions are for a single patient, a lot of other work (e.g. analyzing population health) requires telling an EHR to send tons of data - "Give me all your XYZ data about men between ages 25 and 55," etc. This is a whole different API.
