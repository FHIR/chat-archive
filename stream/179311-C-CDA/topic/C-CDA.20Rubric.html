Christof Gessner (Jan 28 2019 at 19:30):

The CDA XML Implementation Technology Specification - Data Types 2.28.2 Unit of Measure states that "The default unit of measure is 1 (the "unity")." So if unit is not present in a PQ, it takes the value 1.

C-CDA CONF:1198-31484 states that "If Observation/value is a physical quantity (xsi:type="PQ"), the unit of measure SHALL be selected from ValueSet UnitsOfMeasureCaseSensitive 2.16.840.1.113883.1.11.12839 DYNAMIC"

The value set contains "All units of measure". So it clearly includes "1", which is a valid UCUM code.

My conclusion: The unit attribute may be omitted in PQ, if and only if it assumes the value "1", the default value.
