Saul Kravitz (Feb 10 2021 at 04:35):

hi @David Clowers
You are correct. If there are instances of FHIR List resources that are not connected to the Formulary, they would be returned by a query for GET [base]/List.
1) It would be a reasonable request by a client IMHO to request ALL CoveragePlans, and I don't see any way to easily get only CoveragePlans and not all Lists.
2) It appears there is a missing search Parameter on CoveragePlan to allow query of a CoveragePlan by PlanID. I've added a tracker for these two cases. https://jira.hl7.org/browse/FHIR-31038
3) FormularyDrug has similar, but not quite as serious issues. It is possible to query all FormularyDrugs by PlanID, but not easy to query all MedicationKnowledge instances that are FormularyDrugs. (I haven't created a tracker for this). Querying MedicationKnowledge resources by code (RxNorm code) might return MedicationKnowledge resources that aren't FormularyDrugs. Querying by drugName, DrugTier, or DrugPlan leverage content in extensions that will only be present in FormularyDrugs.

I think we need to consider how best to address these issues.
@Dave Hill
