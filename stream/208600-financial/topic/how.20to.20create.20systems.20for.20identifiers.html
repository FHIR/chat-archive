Topic: how to create systems for identifiers

Varvara (Dec 11 2019 at 15:28):

@Paul Knapp Hi, Paul! Is there a rule how to create a system for identifiers, which does not have an existing system in FHIR terminology? For example, DEA provider id, Medicaid provider id, etc

Lloyd McKenzie (Dec 11 2019 at 15:56):

If they are systems that others are likely to need, submit a change request to have HL7 assign a common URL for everyone to use.

Varvara (Dec 11 2019 at 16:22):

@Lloyd McKenzie thank you, I will collect a list. Those identifiers are common to US, so probably they should be listed in US core? which working group should I mention in change request?

Lloyd McKenzie (Dec 11 2019 at 16:37):

Right now, a lot of US identifiers are included in the FHIR core spec. You're right though that moving them to US Core might make better sense. However, maintenance of these sorts of things - as well as external terminologies and certain value sets will be moving to a continuous maintenance process that is shared across all FHIR product families (FHIR, v2, CDA, etc.) on the grounds that metadata about identifier types, code systems, etc. is typically needed regardless of what standard is being used.
