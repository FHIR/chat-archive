The Coverage resource defines the 'class' field for 'Additional coverage classifications'. There is an issue with the field name as in many programming languages 'class' cannot be used as variable name because is a special keyword. Should this field's name be changed in the future?

Lloyd McKenzie (Sep 29 2020 at 13:28):

We can't necessarily avoid all key-words in all software languages or technologies. We typically name things in resources based on what the business term is. The reference implementations adopt naming conventions that ensure that generated class and property names don't violate naming conventions in that language.
