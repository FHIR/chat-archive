Cora Drenkhahn (Feb 03 2021 at 11:44):

Hallo zusammen,
bei der openEHR-Modellierung des Laborauftrags ist uns aufgefallen, dass im ServiceRequest-Profile für den code kein spezifisches ValueSet hinterlegt ist und lediglich auf ProcedureCodes(SNOMEDCT) verwiesen wird. Für die Beauftragung von Laboruntersuchungen könnten aber gut die "Order"-Codes aus LOINC verwendet werden (mit Wert "Order" oder "Both" für den Parameter "ORDER_OBS"). Ein ValueSet ließe sich mit folgendem Filter definieren:

"filter": [
  {
     "property": "ORDER_OBS",
      op": "regex",
      "value": "Order|Both"
   }
],
"system": "http://loinc.org"

(Von HL7 gab es das auch schon.)
Was haltet ihr davon, dieses VS zentral festzulegen? @Andreas Bietenbeck @Alexander Zautke @Danny Ammon
