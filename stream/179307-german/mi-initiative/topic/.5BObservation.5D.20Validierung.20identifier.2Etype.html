Alexander Zautke (Feb 18 2021 at 15:33):

Hi @David Meyers einige Leute würden sagen das es ein Feature des Java Validators ist, andere würden gerne die Warning abschalten. Das entsprechende Element hat ein Extensible-Binding was uns erlaubt den korrekten Code (hier OBI) zu hinterlegen. Der Java Validator möchte darauf aufmerksam machen, dass er nicht in der Lage ist ein Extensible Binding zu validieren, da alle weitere Codes erlaubt sind und somit keine geschlossene Menge an erlaubten Codes existiert. Man soll durch die Warning halt nochmal darüber nachdenken, ob man wirklich den korrekten Code gewählt hat. Nervig wenn es halt jedes Mal ist.

Alexander Zautke (Feb 20 2021 at 16:46):

Nicht ganz. Identifier.type ist gebunden an ein ValueSet: https://www.hl7.org/fhir/valueset-identifier-type.html Dieses besteht aus einer Auswahl an Codes aus dem besagten v2 CodeSystem. Jedoch wurden hier nur eine handvoll Identifier ausgewählt. Die genauen Regeln für ein Extensible Binding finden sich hier: https://www.hl7.org/fhir/terminologies.html#extensible Generell ist die Idee jedoch, dass man mit einem Extensible ValueSet einen Vorschlag macht der in vielen Fällen zutreffen sein wird für das Element. Wenn es jedoch mal nicht passt, kann ein eigner Code verwenden werden (in unserem Fall zufällig aus dem gleichen CodeSystem). Somit ist es per Definition nicht validierbar.
