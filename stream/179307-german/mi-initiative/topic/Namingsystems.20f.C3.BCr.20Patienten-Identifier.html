Topic: Namingsystems für Patienten-Identifier

Simone Heckmann (Nov 27 2019 at 11:12):

Noch ein Nachgedanke zu dem auf dem Hackathon identifizierten Problem der Vergabe von NamingSystem-URLs für die Patienten-Identifier der einzelnen Häuser:
Im Prinzip kann ja in der Tat jedes Haus selbst entscheiden, ob es eine URL aus der eigenen Domain vergeben möchte, ideralerweise gemäß den Best-Practice-Empfehlungen im Basisleitfaden, oder eine URL aus der Domain der MI-I verwendet, oder eine OID ( :cry: ), so lange es eine zentrale Stelle gibt, an der die jeweiligen NamingSystem-Ressourcen publiziert werden, so dass man dort danach suchen kann...

Vorteile:

  • OIDs können als alternative Identifier mitgeführt werden (wichtig, wenn man CDA->FHIR mappen will)
  • Die URLs können anhand von NamingSystem.publisher zu einer menschen-lesbaren Repräsentation aufgelöst werden
  • Man kann mit wenigen Zeilen Code eine webbasiertes Auskunftssystem bauen, an dem man die NamingSystems recherchieren kann
