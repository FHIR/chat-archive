Topic: /metadata endpoint change in STU3

James Agnew (Nov 07 2016 at 21:16):

Hello all, a quick announcement: One change that has been made in FHIR in STU3 is that the /metadata endpoint will now return a resource called CapabilityStatement instead of a resource called Conformance.

This breaks any client that is expecting the previous resource to be returned (including but probably not limited to the current versions of HAPI's client, the .NET client, and the Python client).

Grahame's server has been updated to return the new type, but HAPI's has not. I'm planning on making that change this week.

HAPI 2.1 has been updated so that its client will accept both a CapabilityStatement or a Conformance. There is a 2.1-SNAPSHOT build up on the global Maven repos with this fix now, and HAPI 2.1 will be release this week.
