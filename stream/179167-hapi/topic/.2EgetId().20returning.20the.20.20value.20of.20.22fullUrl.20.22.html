Topic: .getId() returning the value of "fullUrl "

andrew (Oct 11 2018 at 13:05):

I am trying to get the id field of a resource with the .getId() method. For some reason it is not returning the id, which is "carrier-23976451019", but instead returning what is in the "fullURL" field, which is <fqdn>/carrier-23976451019. Any reason as to why and how I can get the value of the id field instead?

As a clarification the "fullUrl" field isn't actually present in the resource, I have a bundle with a number of resources, and I'm doing the .getid() on a resource received from a BundleEntryComponent.getResource(). That BundleEntryComponent has a "fullUrl" field that matches the value I am seeing from .getId(). However, that value is not seen in the actual resource which the .getId() method is called on.

Andrea Essenwanger (Oct 19 2018 at 12:08):

You may need to use .getId().getIdPart() instead of just .getId() or .getId().getValue()

String getValue()
Returns the value of this ID. Note that this value may be a fully qualified URL, a relative/partial URL, or a simple ID. Use getIdPart() to get just the ID portion.
Specified by:
getValue in interface IPrimitiveType<String>
See Also:
getIdPart()

James Agnew (Oct 30 2018 at 14:16):

You need to call resource.getIdElement().getIdPart(). For the DSTU3+ structures, getId() returns the whole qualified ID as a String, whereas getIdElement() returns the IdType which you can query for the individual parts.
