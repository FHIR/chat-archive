Morten Ernebjerg (May 25 2021 at 08:56):

We were looking at how one is supposed to resolve version conflicts in FHIR packages. That is, how to handle the case where distinct packages A & B both have the same package C as a dependency, but use different versions of C so that there are, potentially, two different version of the same profiles from C available. As far as I can tell from this general thread about the theme from April, there does not seem to be an agreed FHIR-wide strategy for handling this. But does HAPI have anything in this direction implemented in the package-loading modules (or is there HAPI-specific guidance on how to handle this)?
