Topic: Binary being converted to externalized-binary-id extension

Sean McIlvenna (Feb 10 2020 at 20:00):

Any Binary resources (including contained Binary resources) are having the "content" (STU3) and "data" (R4) properties converted to externalized storage extensions using hapi-fhir-jpaserver-starter@rel_4_2_0. According to documentation, the default is to behave the same as it always has and include the Binary data/content directly in the resource. Any idea why externalized storage is being used by default?
