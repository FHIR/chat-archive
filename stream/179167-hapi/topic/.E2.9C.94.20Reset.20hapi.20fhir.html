Mohsen Jafari (Feb 12 2022 at 11:49):

I'm using the HAPI FHIR server by the following docker:

docker pull hapiproject/hapi:latest
docker run -p 8080:8080 hapiproject/hapi:latest

Is there any way to reset the server to its first state without any resources?

John Silva (Feb 12 2022 at 18:55):

I think you could stop and delete the running container. Then, when you restart it, it should create a new container from the image you've already downloaded. (May not be the fastest but it's probably the easiest thing to do.)

docker ps -a |grep hapi docker stop <containerId from previous step> docker rm <containerId>
Of course this is for Linux/Mac but you can do something similar on Windows using PowerShell or 'findstr' (the Windows, grep-like tool)
Also, there are ways to reduce the number of steps by using the -f (force) option; check out the docker help pages for more info.
