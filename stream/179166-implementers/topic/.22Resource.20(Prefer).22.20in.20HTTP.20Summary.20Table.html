Ken Sinn (Feb 11 2020 at 02:52):

On the HTTP page, in the Response Summary table (http://www.hl7.org/fhir/http.html#summary), there are a few instances of Response Body: "Resource (Prefer)". Does "Resource" mean the Resource Type for the Create? Could the response body (for a Successful Create) be:
1) a Bundle resource, with the created Resource and an OperationOutcome?
2) a Bundle resource with multiple OperationOutcome entries?
3) an OperationOutcome (severity=information)?

Recognizing this would be unusual behaviour, and implementations may not be capable of handling this response, but curious about its conformance. I've looked at the rules around Prefer headers, and the conformance language regarding returning an OperationOutcome (information) for success is only "should" and not "shall".

The use case is for a FHIR Create endpoint A Facade, with an underlying distributed system, which in turn initiates FHIR Create on Server X and FHIR Create on Server Y. We want the ability for the requester calling endpoint (A) to know that the results from Server X and Server Y, whether successful or failure. In the case that (X) is successful but (Y) fails, (A) is still considered successful as it has done its job to call the underlying systems (similar to FHIR batch rules).
