Andy Stechishin (May 16 2019 at 05:36):

@Ravindra Kondiparthi I am guessing you mean that the binding for the valueset-coverage-type is preferred from the Coverage Resource.

Within that context, that means that you should use one of the values in the current valueset if it matches your needs but you can supply additional values if the current valueset does not (which seems to be the situation here). To get additional values added, you will need to submit a tracker which would then be considered by the Financial Management work group. If you submit a request to add the value, it always helps if you bring your use case so that everyone can understand the need.
