Topic: $validate-code for abstract ValueSet members

Michael Lawley (May 29 2018 at 00:28):

The semantics of $validate-code are defined as

Validate that a coded value is in the set of codes allowed by a value set.

and

True if the concept details supplied are valid

However, this does not make clear what should happen when the code is a member of the ValueSet.expansion.contains but has expansion.contains[].abstract = true

Grahame Grieve (May 29 2018 at 20:01):

this is why there is a parameter 'abstract' on the API - so the client can tell the server whether this is a use where abstract codes are ok

Yunwei Wang (May 30 2018 at 13:31):

There is abstract input parameter in $valudate-code operation. My understanding is that if $valudate-code?abstract=true, then the abstract codes in the expansion are used in $validate-code operation. Otherwise, the operation should not consider abstract codes in expansion.

Michael Lawley (May 30 2018 at 23:10):

Thanks, I should have checked further down in the spec.
