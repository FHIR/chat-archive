Topic: Address on PractitionerRole

Samantha Mater (Feb 03 2017 at 16:51):

In the spec, practitionerRole has a reference to a location resource, which has an address element. However, in some cases, this address might differ from the address we want to display for a given PractitionerRole. If this is the case, we were thinking of adding a contained location resource with only the address element populated.
Does it make sense to have both populated on a PractitionerRole?

Brian Postlethwaite (Feb 04 2017 at 01:04):

I'd recommend just creating another location with that address. In most cases there is likely more than 1 practitioner or role for the location, and would be nicer to share the location.
The other consideration is searching, which is less supported on contained resources.
