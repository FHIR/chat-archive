Nik Klassen (Nov 05 2018 at 21:04):

When adding a request.method property for history bundles (v3.0.1) we noticed that PATCH is not a supported option in the HTTP verb value set. However, it has been added by v3.5.0. But the documentation for history says that "The interactions create, update, and delete create history entries." Should PATCH create a history entry with method PATCH, or should it be represented as an update with the method as PUT?

Brian Postlethwaite (Nov 05 2018 at 22:18):

Nik Klassen (Nov 05 2018 at 22:58):

Sorry, to clarify, did you mean logically an update is occurring so it makes sense to keep it as a PUT?

Lloyd McKenzie (Nov 05 2018 at 23:12):

It makes sense to represent it as PUT - but please submit a change request for us to make this clarification in a future version of the spec.
