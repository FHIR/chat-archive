Anand Mohan Tumuluri (May 14 2018 at 20:04):

For the operation [Resource]/[id]/$validate, the spec says that the resource parameter is mandatory unless the mode is delete. So there is no way to validate an existing resource instance? HAPI seems to be validating the current resource content when the mode is unspecified and the behavior is different from explicitly specifying the mode as delete. I couldn't test with Grahame's test server at http://test.fhir.org/r3 as it returns an OO for any validation.

No reason, that was more of a spec interpretation question. The only case that I can imagine/invent is when the profiles are developed/enhanced some time after you get data into the system. In that case, it would be useful to find out the resources which are not conformant to the profile.
