Mikael Rinnetmäki (Sep 03 2020 at 10:47):

Some work in the past haas resulted in this: https://simplifier.net/finnishphr/fiphr-sd-bloodglucose-stu3. In Finland using mmol/l as the primary (required) unit, but supporting mg/dl too on the side, as a 'translation'.

Mikael Rinnetmäki (Sep 03 2020 at 11:19):

My own view is that implementations all implementations should support both mmol/l and mg/dl and implementations visualizing the data should be able to present the data in either units, based on user's selection.

Mikael Rinnetmäki (Sep 03 2020 at 11:21):

Also, experience has shown that for patient-generated data, keeping data from glucometers and continuous glucose separate from each other is worthwhile. A CGM produces around 288 measurements per day.

Mikael Rinnetmäki (Sep 03 2020 at 11:24):

So we have three categories in #finnish PHR. One for CGM values, one for glucometer data, and one generic (mainly lab data, and for apps that don't know better). And we tag glucometer data with both the generic code and the specific code, so that generic apps find the data, and smart apps can make the distinction. But we keep CGM data separate, as generic apps might not be able to handle it well.
