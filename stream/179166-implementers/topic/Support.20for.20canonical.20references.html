Topic: Support for canonical references

Rik Smithies (Jul 26 2021 at 20:40):

Do servers support transactions where the temporary links between resources are not the usual type (reference to an id) but links between canonical references (reference to a url, perhaps with a version appended)?

And how about chained searches/reverse chain, includes and revinclude across canonical references?

@James Agnew @Ward Weistra @Josh Mandel @Paul Church @Lee Surprenant

Paul Church (Jul 26 2021 at 20:49):

No, the Google implementation does not do any of those. We only handle canonicals in the context of looking up artifacts like profiles, valuesets, etc.

Lloyd McKenzie (Jul 26 2021 at 22:46):

@Paul Church - so I can't search for QuestionnaireResponses based on the characteristics of the referenced Questionnaire? Or include a Questionnaire when I retrieve a QuestionnaireResponse?

Paul Church (Jul 26 2021 at 23:26):

Nope. It's part of the R4 functionality that we haven't got around to implementing. It's called out in the docs as unsupported but no one has asked about it yet - same with reference :identifier and token :of-type.

Ward Weistra (Jul 27 2021 at 09:39):

@Rik Smithies Same seems to be true for Firely Server. @Alexander Zautke says there's no support for that.

Lee Surprenant (Aug 02 2021 at 20:27):

Sorry for the late reply (I'm just back from a vacation), but the IBM FHIR Server has yet to implement handling canonicals like reference search as well. That said, it looks like we've recently started on that feature.
