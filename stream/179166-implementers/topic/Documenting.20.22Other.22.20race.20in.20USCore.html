Topic: Documenting "Other" race in USCore

Kenny Blanchette (Jul 05 2018 at 16:57):

Our system allows a provider to document "Other" for a patient's race. In reviewing the QICore / USCore extension for us-core-race (http://hl7.org/fhir/us/core/StructureDefinition-us-core-race.html) and the associated value set (https://www.hl7.org/fhir/us/core/ValueSet-omb-race-category.html), there doesn't appear to be a value we can correctly use.

The UNK value looked promising, but it explicitly states that if the race cannot be expressed using the values in the value set, implementers should not use UNK and should rather use OTH or UNC. However, OTH and UNC are not valid options for omb-race-category.

How have other implementers represented "Other" for race?
