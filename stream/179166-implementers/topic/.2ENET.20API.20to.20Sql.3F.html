Hi, is there anything in the FHIR .NET API which can help me to create SQL queries?
My understanding is that "SearchParams" and methods associated to this are for making calls against fhir clients, or am i wrong?

Lloyd McKenzie (Sep 06 2018 at 14:22):

SQL queries will be specific to how you choose to organize your persistence layer - and there are lots of strategies driven by many factors. Some throw the entire resource into a blob and have a separate table of index terms. Some store json fragments for non-queriable bits. Some are driven by legacy. Some break things down based on a profile. Because of that variance, a general purpose tool like the reference implementation can't generate queries - it won't know what the data model is. To get that, you'd actually need to use a full open source server - which has the persistence layer defined.
