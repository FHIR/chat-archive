Christopher Vitale (Aug 17 2020 at 03:49):

Hi. Quick question re: Condition.bodySite. US Core spec (https://www.hl7.org/fhir/us/core/StructureDefinition-us-core-condition-definitions.html#Condition.bodySite) says: "Codes describing anatomical locations. May include laterality"

Does this allow for only laterality (i.e., "right")? Or is the intent to only allow a body site with or without laterality (i.e., "ear", "right ear")?

Christopher Vitale (Aug 17 2020 at 18:13):

Hi. Wondering if anyone has experience with this that can share. Thanks!

Robert McClure (Aug 17 2020 at 23:22):

The intention is that the code should fully describe a site. In general that would mean including laterality. An instance with only "Right" would not make any sense, but if you are using SNOMED CT and ECL, you can include a post-coordinated code.
