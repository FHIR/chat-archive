Simone Heckmann (Nov 02 2016 at 21:18):

According to the spec, the type level _history-interaction
GET [base]/[type]/_history{?[parameters]&_format=[mime-type]}
supports the parameters
_count
_at
_since

I can see UseCases, where a system would want to poll a server for "new Labresults" or "any Patient updates where serviceProvider = Organisation 123"
But adding any criteria to the interaction would be illegal, if I read the spec correctly.

GET [base]/Patient/_history?_since=[timestamp]&organization=123
is not allowed?
