Giorgio Cangioli (Jul 11 2018 at 16:44):

We have discussed this point today in the IPS call.
The fourth solution (add an optional translation element) is the preferred option for the multi-language support for narratives.
We recognize that this is probably not in the 80% for current systems, but possibly it is still a reasonable consideration for enabling global interoperability. But if this preferred solution will not be applied, we hope that at least xhtml extensions will be allowed for Narratives, so that a universal Narrative profile could be specified to have a uniform way to handle this need
