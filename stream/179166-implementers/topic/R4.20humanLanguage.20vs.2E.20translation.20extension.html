Robin Bosman (Jul 18 2019 at 12:44):

This topic just to get some feedback how others might approach this: as official extension in FHIR we have
https://hl7.org/fhir/R4/extension-language.html and we have
https://hl7.org/fhir/R4/extension-translation.html.
As the first one is defined "is used where an item may repeat to indicate the language of each repeat" -> it would make sense to use this extension to e.g. when needed give adresses in the Belgian Brussels region that is a bilingual region. Right? AddressBrusselsRegion_SameAddressInTwoLanguages.txt

Lloyd McKenzie (Jul 18 2019 at 15:14):

It depends on how you want to handle maintenance. Do you want a single address with translations of the content or do you want distinct address instances for each language? (The former is likely to be more manageable and avoid issues when it comes to address updates.)
