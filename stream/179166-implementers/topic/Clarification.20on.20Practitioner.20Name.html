Topic: Clarification on Practitioner Name

Lucio Moraes (Jan 16 2018 at 02:21):

We are following this http://fhir.hl7.org.au/fhir/pd2017Dec/StructureDefinition-au-pd-practitioner.html, there apears to be a discrepancy between the differential and snapshot tables for the specification. In the differential table it states that a name should be of type HumanName and contain the fields use of type code, and family and given of type string. This is not compatible with the current definition of HumanName which has supplementary fields of type string; prefix, suffix, text and period of type Period. In the snapshot table these extra fields are included along with another field of type string, the id. Which table should we be fitting our specification to?

Lloyd McKenzie (Jan 16 2018 at 04:58):

Why is it not compatible? The differential table indicates what you're expected to support and/or extra guidance. The other elements are still there inherited from the core specification. The snapshot reflects overall what is possible. The differential just indicates where there are "differences".
