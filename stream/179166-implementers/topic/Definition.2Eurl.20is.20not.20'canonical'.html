Topic: Definition.url is not 'canonical'

Eric Haas (Jun 25 2018 at 17:38):

Why is [Definition.url(http://build.fhir.org/definition-definitions.html#Definition.url) not 'canonical' it actually references other definitions as canonical?

Bryn Rhodes (Jun 25 2018 at 19:29):

Because canonical is a reference type, i.e. wherever it is used, it is a reference to the canonical url. The url element on Definition is actually defining the canonical url for the resource.
