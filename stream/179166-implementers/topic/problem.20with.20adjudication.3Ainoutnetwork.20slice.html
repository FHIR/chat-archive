Topic: problem with adjudication:inoutnetwork slice

Raikol Amaro (Sep 23 2021 at 12:47):

Hello,

I've got a problem with the adjudication:inoutnetwork slice in the C4BB ExplanationOfBenefit Professional NonClinician profile (Version 1.0.0).
The binding for the category field indicates I should use a value from the C4BBPayerBenefitPaymentStatus code system which contains values such as innetworkand outofnetwork. However, when I use the innetwork code, the validator tool fails with the following error:

"None of the codings provided are in the value set http://hl7.org/fhir/us/carin-bb/ValueSet/C4BBAdjudicationCategoryDiscriminator (http://hl7.org/fhir/us/carin-bb/ValueSet/C4BBAdjudicationCategoryDiscriminator), and a coding from this value set is required) (codes = http://hl7.org/fhir/us/carin-bb/ValueSet/C4BBPayerBenefitPaymentStatus#innetwork)"

Indeed, when I inspect the C4BBAdjudicationCategoryDiscriminator value set (version 1.0.0) the innetwork and outofnetwork codes are missing BUT they are present in the 1.1.0 version of the same document which makes me wonder if this is a bug with the 1.0 IG which was fixed in 1.1 and there's nothing I can do to make the validator work for this slice.

Has someone seen this before? Is there any way to get rid of this error?

Grahame Grieve (Sep 23 2021 at 13:00):

you could ask on #CARIN IG for Blue Button®

Corey Spears (Sep 23 2021 at 13:14):

@Raikol Amaro , Yes, this was a bug found in STU 1.0 and was fixed in STU 1.1. Unfortunately, there is nothing that can be done to get a 1.0 instance to pass validators for the Professional & Non clinician EoB except to not include EoB.item. Otherwise you will have to move to 1.1.
(The ticket for reference: https://jira.hl7.org/browse/FHIR-32285)
