Brian Reinhold (Apr 20 2017 at 19:52):

After losing a profile that I started in an implementation guide I am concerned about the correct way to do this (and not lose my work). On a re-do I created and edited the profile outside of the ImplementationGuide Package. Then I added it to the package. When I closed Forge and reopened the implementation guide I see the profile in the Package. However, in order to edit that profile should I load the profile separately and NOT do it from the Package? I can mechanically do either but is the proper way to edit the profile outside the Package?

Michel Rutten (Apr 21 2017 at 09:25):

Hi @Brian Reinhold profiles and implementation guides are separate resources and files on disk. If you create/add a profile as a child of an IG package, then Forge adds a reference to the (external) profile to the package. However you must always save your profile(s) separately. If you open an IG, Forge tries to load all the referenced external profiles (assuming they are available).
