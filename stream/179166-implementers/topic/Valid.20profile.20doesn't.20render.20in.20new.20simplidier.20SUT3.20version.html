Chris Munro (May 12 2017 at 11:49):

I just tried putting a profile (that is valid in forge) into a new simplifier STU3 project and it doesn't render, with not a very helpful error message. Is this functionality working yet, or do we need to wait for it to be completed?

https://simplifier.net/ELabNetwork/Organization

Michel Rutten (May 12 2017 at 12:41):

Hi Chris, I noticed that the StructureDefinition does not have any actual element constraints, both differential and snapshot are missing. If you publish to Simplifier from within Forge, the tool will automatically generate the snapshot before uploading. However if you upload it manually through the web interface, you have to make sure that the StructureDefinition contains a snapshot in order to get a complete rendering of the structure. Check the Forge Options menu, there is a setting to enable snapshots when saving to disk (disabled by default, as the snapshot is quite large).

Michel Rutten (May 12 2017 at 12:42):

FYI In the future, Simplifier will be able to perform the snapshot expansion itself. The underlying FHIR server already has the capability (by calling out to the .NET FHIR API), but we still have to integrate that into Simplifier. Won't take long.
