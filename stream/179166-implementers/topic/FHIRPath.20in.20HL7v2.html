Ewout Kramer (Sep 14 2018 at 12:23):

Hi Aaron - in the .NET API the FhirPath engine is not written against the FHIR model, but against an abstract interface called IElementNavigator. This interface represents the data according to the abstract model sketched in the intro of the FHIRPath specification. I have not implemented this interface for V2, but anyone should be able to do so.
