Topic: Repeating primitive elements having extension but no value

Anand Mohan Tumuluri (Apr 20 2020 at 09:52):

The STU3 version of RI/HAPI is unable to parse null values within arrays in JSON and throws a NullPointerException. This occurs when reading back resources that have been written using RI/HAPI when the resource contains a repeating primitive element with no value but has an extension. For example, timingTiming.event in http://hl7.org/fhir/stu3/activitydefinition-example.json.html. This is fixed in R5 parser but is not back ported to R3/R4. It would be really appreciated if it could be back ported to STU3 (and also R4).
