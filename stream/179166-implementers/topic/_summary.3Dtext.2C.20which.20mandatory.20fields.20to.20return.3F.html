Topic: _summary=text, which mandatory fields to return?

Marco Visser (Jun 22 2021 at 14:39):

The definition of the search parameter _summary=text is like this:

Return only the "text" element, the 'id' element, the 'meta' element, and only top-level mandatory elements

But what should happen in the case of Observation? The top-level elementcode is mandatory (1..1), so it should be included in the search result. But code is of type CodeableConcept. Does that mean that all the members, like coding and text (which are not mandatory btw) should be included as well? And also their members, etc. And also their extensions?

Grahame Grieve (Jun 22 2021 at 14:40):

I believe that the answer to that is server discretion, but you have put something in there

Gino Canessa (Jun 22 2021 at 17:27):

If I remember correctly (though I can't find any discussion here, so maybe on a call?), last time I asked about this the general consensus was that once something is included, it is probably better to include everything from that point down in the tree.

It makes sense when you start looking at BackboneElement and what a user would expect - e.g., if someone uses _elements to filter for Bundle.entry, the expected behavior would be to include the filled-out contents of the entries.
