Topic: Use of Languages Values Set in US Core

Penny Probst (Nov 08 2021 at 15:52):

I am new to FHIR and it's use for the CMS Interoperability rules. I was reviewing the FHIR 4.0.1 standard online and it appears to indicate the languages value set (https://www.hl7.org/fhir/valueset-languages.html) uses a subset of the ISO 639-1 data set rather than the entire set. Is this correct or am I missing something? Thank you.

René Spronk (Nov 09 2021 at 07:38):

The intent is not to exclude any languages, but to provide a list of languages commonly used by FHIR projects. Mostly there'll be an extensible/example binding of this value set, so you could use whatever codes [languages] you want. Remember that sometimes FHIR can't reference/use a particular terminology because of licensing restrictions - one may very well need a license to use ISO 639-1, whilst the codes in the RFC mentioned in the value set definition can be used for free.

Penny Probst (Nov 09 2021 at 12:57):

I was hoping that was the case. Thank you for confirming it.
