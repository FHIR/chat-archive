Richard Townley-O'Neill (Aug 06 2019 at 03:08):

@Dongtu
Extensions are defined with the StructureDefinition resource as profiles of Extension. If the structure definition is known to the FHIR server, the server can use the extension in validation, the same as any other profile.
The URL of the extension is its unique identifier. It has to be a valid URL and be unique.

Lloyd McKenzie (Aug 06 2019 at 03:29):

If there's a "standard" extension (defined by HL7, IHE, your national organization or anyone else that likely has significant adoption, you should use that existing extension (which means using the existing URL). However, if you define your own extension for something new, you would define the URL

Richard Townley-O'Neill (Aug 06 2019 at 03:29):

You identify that profile by its own URI.
If you create your own profile, give it a URI from a name space which you control or one in which you can assign URIs.
Your FHIR server will know about your extension the same way that it knows about any other profile that you define.
