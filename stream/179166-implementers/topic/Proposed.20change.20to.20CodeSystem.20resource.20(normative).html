Topic: Proposed change to CodeSystem resource (normative)

Rob Hausam (Feb 13 2020 at 23:41):

There is a proposal in J#25983 to add invariants to ConceptMap that will require the hierarchyMeaning element to be populated whenever a hierarchy is being represented in a code system (using either nested concepts or parent/child properties). This was always the intent, but we haven't been enforcing it and therefore some of the existing instances in the spec (and likely elsewhere) haven't populated it when they should. If you have implemented CodeSystem (or have an opinion about it), we would appreciate it if you would provide your thoughts on the proposed change here or in the tracker. Thanks!
