Topic: Planned destination

Hans Buitendijk (Jun 26 2018 at 18:02):

I'd like to ask for some opinions on the use of the address for a temporary "future" destination visa vis using Q/QR. The use case is a review of a patient's vaccination requirements for a specific country for upcoming travel. One approach is to use Patient.address.use = "temp", Patient.address.country for the destination the patient intends to travel, and perhaps using Patient.address.period for a future period. The other approach is to use Observation through Q/QR using some agreed to Observation.Code to represent the destination. Both have challenges, particularly having a very vague address on a patient that may never materialize, unless a new use of "planned destination" would be added. Q/QR may be more appealing as there would be a form in play, so it could be done as part of the questions rather than the patient's demographics. Any pros/cons for either approach, or alternative are welcome. Thank you!
