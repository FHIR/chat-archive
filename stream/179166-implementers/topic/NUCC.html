Rob Hausam (Apr 08 2021 at 18:22):

All that matters is if that is the canonical url that has been specified for the code system and that the terminology server (tx.fhir.org or whichever one you are using) recognizes it. As Michele said, it doesn't matter if the url actually resolves to anything.

Rob Hausam (Apr 08 2021 at 18:25):

This is the HTA page for NUCC, which shows that is the correct canonical url.

Robert McClure (Apr 12 2021 at 14:44):

@Jennifer Grill Urls as identifiers are confusing that way. Us terminologists hate the use of meaningful identifiers for these things (it breaks all sorts of rules) and confuses folks like you. To be clear, there is no expectation that a code system or a value set canonical url will resolve to anything. The whole it's nice when it does result makes you think that you can find the content at the canonical endpoint, and that is rare in the real world but (unfortunately for training all users on this point) is very common here inside the FHIR publisher space.
