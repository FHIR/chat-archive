Gabriel Bezerra (Nov 05 2018 at 15:47):

Hi all,

The definition of the _history interaction of the RESTful API says that: http://build.fhir.org/http.html#history

If the server does not populate response.status, it should ensure that only successful interactions appear in the history logs

The text describing the status field in http://build.fhir.org/bundle.html says

Status response code (text optional)

However, the cardinality of the status field is 1..1: http://build.fhir.org/bundle-definitions.html#Bundle.entry.response.status

How is this supposed to be after all? Is status optional or not? If so, how should it be represented in the empty case?
