Topic: XML Schema for Profiles

Richard Kavanagh (Mar 01 2019 at 15:52):

A recurring question that implementers ask us is whether we can supply an XML schema for the FHIR based messages that we develop. Developers new to FHIR expect to have schemas when working with XML.

Even though we point out the datatype and resource schemas from the FHIR downloads section and explain how to validate in FHIR - there is still a desire for schema. It would appear that some middleware products can ingest schema so as to make processing easier.

So, that in mind is anyone aware of any tools that can generate XML schema from profiles?

Lloyd McKenzie (Mar 01 2019 at 15:54):

(And that's consistent with how CDA does things too.)

Lloyd McKenzie (Mar 01 2019 at 18:04):

To my knowledge, no one's developed one because in most cases, it wouldn't be much use. The most important things done in profiles are slicing and vocabulary constraints - and schema can't handle either of those things.
