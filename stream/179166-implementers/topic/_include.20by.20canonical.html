François Macary (May 05 2020 at 08:48):

The _include search parameter instructs the server to include resources related to the search results. The examples provided in the specification show resources related through a Reference link, like:
GET [base]/MedicationRequest?_include=MedicationRequest:patient.
My assumption is that this parameter also works with resources related through a canonical link, like for example PlanDefiniton.action.definitionCanonical.
Am I correctly interpreting?
So, GET [base]/PlanDefinition?_id=s123&_include:iterate=* should bring back the PlanDefinition s123 together with all resources pointed by it, either by Reference or by canonical. Correct?

Michele Mottini (May 05 2020 at 13:19):

I'd say yes but only if there is a corresponding reference search parameter (that appears to be the case for PlanDefinition.action.definition)
