Topic: ✔ Name:text search implementation

Brian Beatty (Aug 09 2021 at 14:53):

I'm trying to implement a name with the text modifier and have a question about the implementation
If I have a querystring like name:text=Bob%20Smith, would I return entries like "Bob Smith", "Bob X Smith", "Bobby Smith" or only "Bob Smith" if there is no given[1]?
Where if effectively combines all names into one string, then performs a contains? or takes the first word as given[0]name and second name as a family name and search them as an OR, or an AND?

Also any suggestions on how to implement name:text=Bob AND Smith and _text=(bone OR liver) and metastases?

Lloyd McKenzie (Aug 09 2021 at 15:17):

Based on the descriptions, you would get a 'contains' search looking for "Bob Smith" on the given, family, text and other elements of the name. So, in general, you're not going to get what you want. It would be better to do name:text=Bob&name:text=Smith.

Brian Beatty (Aug 09 2021 at 15:23):

are each elements, searched on independently?, ex: name:text:Bob%20Smith would only match is family name contains "Bob Smith" or given[0]="Bob Smith" or given[1] = "Bob Smith"
Or would it be given[0] + " " + family = "Bob Smith" ?

Lloyd McKenzie (Aug 09 2021 at 15:26):

The default expectation would be independent search. That doesn't mean that systems couldn't implement differently, but I wouldn't expect that to be the norm.
