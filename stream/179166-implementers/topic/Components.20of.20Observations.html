Topic: Components of Observations

Bryn Rhodes (Aug 05 2016 at 16:01):

The Observation resource has a value[x] element, as well as a component element that can be used to represent multiple results. Do any of the current FHIR server implementations support the use of the component element of the Observation resource to report things like Lab Panels?

Lloyd McKenzie (Aug 05 2016 at 17:52):

I believe the intention is that panels generally would not be captured using Component. But I'll let someone who was part of those discussions confirm. @Eric Haas ?

Eric Haas (Aug 05 2016 at 19:48):

lab panels are typical reported out as Diagnostic reports. Components are for component observations that need to be interpreted together. there is also related for nested lab reports like a micro and susceptiblity.

Bryn Rhodes (Aug 15 2016 at 17:42):

Thanks for the responses, that makes sense. Are there any implementers then that currently support reporting lab panels using DiagnosticReport?

Eric Haas (Aug 15 2016 at 22:47):

last time I spoke with Jenni (Cerner) and Danielle (Epic) they were doing it this way.
