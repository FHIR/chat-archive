Mikhail Lapshin (Oct 29 2018 at 12:49):

@nisha ks Hi Nisha. Unfortunatelly Fhirbase 3.0 is not compatible with Fhirbase 2.0, so you can't easily migrate between them. The best thing you can do is to export all the resources from Fhirbase 2.0 database into NDJSON file(s) and then load them with fhirbase load command into the new Fhirbase 3.0 database. And because you want to migrate from DSTU2 to DSTU3 you'll need to write code to migrate resources from DSTU2 to DSTU3, because Fhirbase won't preform such migration automatically.

I'm not sure which mapping tables you mention in your question, could you provide exact table names? And regarding to FHIR Search, its not going to be implemented in the new Fhirbase. New Fhirbase only supports basic CURD functionality.
