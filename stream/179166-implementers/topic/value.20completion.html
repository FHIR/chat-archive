Topic: value completion

Katarina (Feb 20 2020 at 08:59):

Hi, is it possible, that I define a value for an element in a profile (e.g. Composition.date should be "2020-02-20") and then when I create a new instance of this profile without this element that the fhir server (e.g. vonk) completes the instance with the element and the defined value?

Lloyd McKenzie (Feb 20 2020 at 15:03):

Is it possible, yes, but certainly not expected and if the element is minOccurs=1 in the base resource, it would be non-conformant. The expectation is that fixed values will be present in the instance. Consumers of the instance won't necessarily have any knowledge of the profile - or any capability of filling data into the instance.
