Topic: Use cases for contact info on the Location resource

Cooper Thompson (Jul 08 2020 at 19:53):

The Patient Admin WG is soliciting input from folks about what uses cases exist to track contact information for a location. Location already supports a telecom and address property, but there isn't a way to associate an individual (i.e. HumanName) or purpose. We're considering replicating the structure on Organization, but are interested in learning about what use case folks have.

Peter Bomberg (Sep 22 2020 at 12:57):

Depending on the location we track billing contacts, admin contacts, security contacts, shipping and receiving contacts, emergency access contacts, access contacts, etc. We track some of these at both org and location level as an org can have many locations each with different contacts for the same role.
