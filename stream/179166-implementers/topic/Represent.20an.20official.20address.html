Topic: Represent an official address

Thomas Tveit Rosenlund (Jun 14 2018 at 05:48):

We are looking into representing Norwegian addresses in FHIR by profiling the Address Datatype. In our HL7 Norway workshop one result was the need to represent official address for a person /organization, typically this will be the address that person/organization reports to some official registry. The problem here is that there is no way in the Valueset to represent an official address in address.use(or address.type). using the required ValueSets. In the Netherlands this was solved by defining an extension to represent official, a boolean value. We will have to do something similar if the valuesets can not be extended with an official use.

https://simplifier.net/HL7Norway/no-basis-Address
https://simplifier.net/NictizSTU3/nl-core-address

What is the general consensus, should we just use an extension for this or is this a more common use-case that should be supported in the required ValueSet used by the Address DataType?

Would love to have some input on this from other countries.
