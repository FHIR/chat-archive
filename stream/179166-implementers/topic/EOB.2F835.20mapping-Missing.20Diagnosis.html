Can you please provide more information. What version are you seeing this on (STU1.0.0)? What profile and path are you getting the error on.
If it is the inpatient Institutional ExplanationOfBenefit.item.produceOrService, this is a known issue with STU1. The issue was captured here: https://jira.hl7.org/browse/FHIR-31683. It was addressed by adding the code http://terminology.hl7.org/2.1.0/CodeSystem-data-absent-reason.html|**not-applicable** to the codeset for that particular profile.
This has been addressed in the STU Update, currently available in the CI Build - http://build.fhir.org/ig/HL7/carin-bb/. This update STU1.1.0 is currently going through a publication request process and should be published soon.

If that is not the issue, please provide more information so that I can assist further.
