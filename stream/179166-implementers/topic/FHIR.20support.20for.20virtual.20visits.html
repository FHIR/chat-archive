Stream: implementers

Topic: FHIR support for virtual visits

frank cao (Aug 05 2020 at 19:26):

Hi, does FHIR have plans in the future to support virtual visits? Virtual visit information may require new elements to capture things like modality (video, audio, secure messaging), software platform, and etc. Any recommendation on how to expand the current encounter to support virtual visit scenarios?

Vassil Peytchev (Aug 05 2020 at 21:44):

FHIR has an extensibility mechanism. You can propose extensions to the Encounter resource that you think may be useful by creating a Jira change request from the Propose a Change link at the bottom of any page of the specification.

Having a discussion about your needs in this stream is also a good starting point.

Lloyd McKenzie (Aug 05 2020 at 22:42):

The value set for Encounter.class already supports the notion of 'virtual' encounters. If you want to capture the mode of each participant (they might not all be the same), that would be an extension on Encounter.participant.

Grahame Grieve (Aug 06 2020 at 01:08):

there's this: http://www.healthintersections.com.au/?p=3017

frank cao (Aug 06 2020 at 14:03):

Thanks! It sounds like FHIR have no immediate plans to make core changes to support virtual visits, and we should stick to extensions, is that correct?

Lloyd McKenzie (Aug 06 2020 at 14:59):

You're certainly free to use a change proposal to suggest changes to core. These will be evaluated based on degree of support by existing systems.

Richard Townley-O'Neill (Aug 07 2020 at 05:00):

For R4, extensions are the way to go. Any changes to core will appear in R5 (or later). Experience with extensions will influence any changes to core.

René Spronk (Aug 07 2020 at 14:51):
