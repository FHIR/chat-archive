Hi all,

I'm looking at the syntax for _revinclude parameters on searches, and had a quick question for the group:

The include/revinclude sections mentions that there are three potential parts to a _revinclude parameter value:

Parameter values for both _include and _revinclude have three parts, separated by a : character:

The name of the source resource from which the join comes
The name of the search parameter which must be of type reference
(Optional) A specific of type of target resource (for when the search parameter refers to multiple possible target types)

The first two parts (name of source resource + search parameter) make perfect sense - /MedicationRequest?_revinclude=Provenance:target means I'm looking for Provenance where the target search parameter resolves to MedRequests found. I'm a little unsure about the third part (type of target resource) though - for non :iterate _revinclude searches, should type be MedicationRequest as that's what I want the Provenance.target search parameter to be resolving to, or Provenance as that's the type of resource that I'm requesting to be included in the search bundle?

I believe MedicationRequest is the right answer based on the spec, but wanted to confirm with the group since it seems a little redundant on a MedicationRequest search. (I definitely see how it would be useful in an _revinclude:iterate scenario)

Thanks in advance!

René Spronk (Nov 05 2021 at 07:45):

In your specific context, without :iterate, and with just one focal resource of the search (MedicationRequest) the use of the third component of _revinclude doesn't make much sense. MedicationRequest would be the only useful value for the third component.
