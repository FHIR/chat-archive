Topic: ✔ Wording on use of Codesystem supplements

Jens Villadsen (Aug 26 2021 at 08:13):

The spec says the following for using CodeSystem supplements (https://www.hl7.org/fhir/codesystem.html#supplements) -

The CodeSystem.url for a supplement must be under the control of the authority creating or publishing the supplement (e.g. not in the same space as the code system being supplemented, unless the supplement is being issued by the same authority as the original code system

Does that mean that if I make a supplement that contains eg. translations for another language for eg. http://hl7.org/fhir/address-use - should that Codesystem supplement then be under the authority of HL7 (and not eg. the affiliate that provided the codesystem supplement)?

(Tagging you @Lloyd McKenzie as you usually have an oppinion on these matters)

Grahame Grieve (Aug 26 2021 at 11:08):

no it means you have to use a URL you have control over if you publish the supplement. If you do it through an HL7 Committee, then yes, ues an HL7 namespace. If you do it through the affiliate, use their name space
