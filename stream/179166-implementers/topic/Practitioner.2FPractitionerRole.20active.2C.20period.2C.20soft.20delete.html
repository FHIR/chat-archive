Topic: Practitioner/PractitionerRole active, period, soft delete

Yarek Tyshchenko (Aug 13 2018 at 15:32):

What is the purpose of the active field in Practitioner and PractitionerRole. The description in the docs for both of them is identical. If the field is really intended as "resource created in error" then isn't it doing a similar function as a soft delete?

The docs are similarly vague on other resources' active fields too. Could anyone please weigh in on the typical usage behind them?

Brian Postlethwaite (Aug 13 2018 at 23:39):

Its also intended to be used where the practitioner leaves the organization and should no longer be actively used. But we still want them in the system as there is lots of data that is attributed to them, so don't want to delete the record.

Brian Postlethwaite (Aug 13 2018 at 23:39):

Stops them appearing in default searching/filters etc.

Brian Postlethwaite (Aug 13 2018 at 23:40):

(as well as the entered in error status when you aren't sure if its been used or not yet, or the system doesn't support delete)

Yarek Tyshchenko (Aug 14 2018 at 15:03):

That makes sense for Practitioner, but for Role... If a Practitioner is inactive should all their attached PractitionerRoles be also retroactively marked inactive, even if they have their period set in the past?

John Silva (Aug 14 2018 at 20:18):

A quick guess -- a Practitioner might have had different roles over time, e.g. from an Intern to a Resident to an Attending, etc. and if this info needs to be there for historical purposes then I suppose Role would need to be maintained as well

Yarek Tyshchenko (Aug 15 2018 at 10:48):

Thank you that clears it up. From my understanding Roles that are incorrect and can't be 'corrected' with a period should be set as inactive, so they don't appear in normal views. For all other cases period field should suffice.
