Is there a specific reason why CareTeam.status is not of type codebut CodeableConcept, unlike many other similar resources? Is it because there is not yet a (required) CareTeamStatus value set in place, although there are some suggested codes?

Michelle (Moseman) Miller (Jan 04 2017 at 15:40):

Once Patient Care WG finishes up QA on this resource, the data type could indeed change from CodeableConcept to code. First, we need a discussion on what the status value set should be and whether it's required binding. I had previously logged GF#10725 as a ballot comment to address the missing value sets within this resource, so it's on our radar to resolve before STU3.
