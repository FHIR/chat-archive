Topic: ✔ Setting Reference to Elements of Another Resource

Kyle Thompson (Oct 27 2021 at 23:29):

Suppose I have the following

RequestGroup = {"resourceType": "RequestGroup", "id": "123", .., "action": [{"id": "ABC", ..}]}

I would like to create a Task referencing the action within the RequestGroup with id ABC. What is the appropriate way to do this? I was thinking something along the lines of

Task = {"resourceType": "Task", "basedOn": [{"reference": "RequestGroup/123/action/ABC"}], ..}

A pointer to the appropriate reference within the docs would also be much appreciated.

Lloyd McKenzie (Oct 28 2021 at 00:46):

That's not possible. References point to resources. It's not possible to point to sub-elements. However, you can point to the Resource that is pointed to by the RequestGroup.action. (Keep in mind though, that you can't manipulate the resources within a RequestGroup separately. You can't suspend or cancel or start one request - you suspend, cancel or start the entire RequestGroup.
