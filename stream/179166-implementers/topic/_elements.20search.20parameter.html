Hi,
I need some clarification regarding the _elements search parameter. Is there a way to specify the elements on a complex type through the search query.
Example says, GET [base]/Patient?_elements=identifier,active,link
Is there a way to specify elements within an identifier for example ,
GET [base]/Patient?_elements=identifier { system, value }

Thanks
-Sanjaai

Igor Sirkovich (Apr 17 2018 at 19:40):

My understanding is that this is not allowed. The spec says "The _elements parameter consists of a comma separated list of base element names such as, elements defined at the root level in the resource"

Grahame Grieve (Apr 17 2018 at 22:41):

right. if you want more than that, there's graphql, but that's not widely supported at this time

I had exactly this question at our connectathon yesterday and gave the same answer :)
