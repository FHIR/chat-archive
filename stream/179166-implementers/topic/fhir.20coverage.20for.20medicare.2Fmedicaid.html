Paul Knapp (Dec 21 2017 at 16:12):

@Tim Zallin This would be the topic of a US implementation guide, but essentially: the type would be 'HIP', the payor would reference an organization which has in its identifier the 5 digit NIIP for the specific Medicare/Medicaid/MedicareAdvantage plan, and Group, Plan etc. elements would only be included if the plan provided group and plan numbers. In R3 extensions are used to convey RX BIN and other components, in R4 we expect these can be expressed in a native element.
