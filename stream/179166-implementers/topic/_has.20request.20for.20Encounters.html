Topic: _has request for Encounters

Doug Morand (Jan 28 2022 at 03:04):

I'm trying to write a FHIR request to pull all Encounters for a Patient if they have a ServiceRequest (code=12345) but I can't seem to wrap my head around how to construct the _has query (assuming this is the right path).

This is what I'm using to retrieve Patients that have a particular ServiceRequest

Patient?identifier=99999999&_has:ServiceRequest:patient:code=12345

Lloyd McKenzie (Jan 28 2022 at 03:40):

If the ServiceRequest is issued as part of the encounter, then it'd be:
Encounter?patient=123&_has:ServiceRequest:encounter:code=12345
However, if you're interested in ServiceRequests issued outside that encounter, I don't know that you can. @Grahame Grieve?

Doug Morand (Jan 28 2022 at 14:05):

Yeah that SR is associate to an Encounter but then it only returns the Encounter associated to that SR. I was hoping to be able to get ALL Encounter resources if the patient had that SR entered. I might just need to do two requests

René Spronk (Jan 28 2022 at 14:33):

Patient?identifier=99999999&_has:ServiceRequest:patient:code=12345&_revinclude=Encounter:patient includes all encounters (for the patient with indentifier 99etc with the specified _has) in the response bundle..

René Spronk (Jan 28 2022 at 15:12):

If the SR doesn't have a reference to the patient, but only via the encounter, Encounter?patient=123&_has:ServiceRequest:encounter:code=12345&_include=Encounter:patient&_revinclude:iterate=Encounter:patient (however, not all servers will support the :iterate modifier)
