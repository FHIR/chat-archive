Topic: :in modifier for token

Shamil Nizamov (Jan 17 2022 at 15:46):

What does the :in modifier do for token? Is it the same as :contains for string?

Michele Mottini (Jan 17 2022 at 15:49):

No, it looks into a value set

Shamil Nizamov (Jan 17 2022 at 16:09):

If the token is the Identifier?

Michele Mottini (Jan 17 2022 at 16:27):

No, it really applies only to searching by code / coding / codeable concept

Michele Mottini (Jan 17 2022 at 16:27):

'The search parameter is a URI (relative or absolute) that identifies a value set, and the search parameter tests whether the coding is in the specified value set. The reference may be literal (to an address where the value set can be found) or logical (a reference to ValueSet.url). If the server can treat the reference as a literal URL, it does, else it tries to match known logical ValueSet.url values'

Shamil Nizamov (Jan 17 2022 at 16:30):

I've seen several requests to make a partial search on Identifier. Should we extend the :in behavior to allow this? Or include :contains to tokens?

Michele Mottini (Jan 17 2022 at 16:48):

There is :text for that already

Michele Mottini (Jan 17 2022 at 16:48):

And no, we should not modify the specs

Shamil Nizamov (Jan 17 2022 at 16:49):

The description - the match does a partial searches on the text portion of a CodeableConcept or the display portion of a Coding - does not give the sense that this also applies to Identifier. This needs to be clarified then. Do you like me to send a change request?

Michele Mottini (Jan 17 2022 at 16:52):

Good point, I don't know actually if it is _supposed_ to apply to identifiers

Michele Mottini (Jan 17 2022 at 16:52):

Sure, a change request for a clarification seems good
