Topic: ✔ Identifier vs custom extension

James Dinsmore (Jul 14 2021 at 13:31):

We need to add a value to the Appointment resource that is an internal identifier. This identifier is not unique per Appointment record. Is it acceptable to use Appointment.identifier for this use case or should it be a custom extension?

Daniel Venton (Jul 14 2021 at 14:35):

If it's not a unique value, then how do you use it as an identifier now, just internally?

James Dinsmore (Jul 14 2021 at 14:39):

Yes. It is a code that we use internally to signify an organization.

Daniel Venton (Jul 14 2021 at 14:42):

If it's an identifier of an organization then it belongs on an Organization resource. Perhaps your appointment resource will reference the organization resource via the supportingInfo or indirectly via the participant.actor.

Mario Villace (Jul 14 2021 at 14:46):

Hi! IMHO, identifiers must be unique in a domain/context by nature or they are not an identifier at all (maybe an attribute or characteristic of the modelled element); as described https://www.hl7.org/fhir/datatypes.html#Identifier , if this "Identifier" is not unique per appointment record, don't use an Identifier type. "The value SHALL be unique within the defined system and have a consistent meaning wherever it appears."
