Topic: Questionnaire Secondary Text

Pascal Pfiffner (Dec 12 2016 at 08:56):

In DSTU-2, we (ab)used the http://hl7.org/fhir/extension-questionnaire-instruction.html extension for secondary text on a question or group/display items. For example in our SF-36 implementation, the text would simply say “Vigorous Activities”. Then we'd have the instruction extension to say “such as running, lifting heavy objects, participating in strenuous sports” (and render this in smaller font right under the question).

This extension has gone away in STU-3 and it seems the idea is to nest another display type item to achieve this behavior, correct? This could be workable, but it seems that having a primary and a secondary text element on questionnaire items is rather common – the text element is already labeled “primary text”. I'm wondering if there should be a core element so that an item can have primary and secondary text. Could be title and text, which would demote text to secondary, or an additional element like details. Thoughts?

Lloyd McKenzie (Dec 12 2016 at 10:41):

There's a whole bunch of different types of nested display possible - secondary text, instructions, units, etc. I don't think any one of those is "more" core than the others, nor do I think that any of them fall within the 80%.
