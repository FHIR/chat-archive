I thought no one else would answer and haven't seen this topic in a long time.
In our understanding, the laboratory devices managment system is an independent application that:

  • accepts orders for hardware laboratory tests (hereinafter "order")
  • selects the devices to execute the order
  • manages order execution job queues (hereinafter referred to as "task")
  • sends tasks to devices
  • gets results from devices
  • returns order results to the customer

Planned integration business processes between laboratory information system and laboratory devices managment system:

  • synchronize integration references
  • send order
  • get order results
