Lloyd McKenzie (Feb 03 2022 at 16:40):

You can render content as you like. And you can render "allowed answers" with XHTML using an extension. However, the answer itself won't be XHTML. The answer will be a code or an integer or a date or some text typed by the user - and if text, the text typed isn't going to be XHTML or markdown. The most we support right is "text" which is essentially a string where carriage returns and line breaks are supported. If you want to allow an answer to actually be text containing embedded markup, you could submit a change request.
