Topic: "fixed" internal resource id for contained resources

Ken Sinn (Dec 04 2019 at 22:42):

Might be obvious to some folks, but it wasn't entirely obvious to me, until a developer asked -- when assigning and referencing resource IDs inside a contained resource, is there any need to dynamically generate the contained resource IDs, if we know that there will only ever be 1 resource of that type? e.g. For a Consent resource, if it will ever only contain one Patient resource and contain one Practitioner resource, could we simply use a fixed value of "practitioner-internal-id" and "patient-internal-id" instead of generating internal IDs on the fly? This seems to adhere to the rules in https://www.hl7.org/fhir/element.html#id

Lloyd McKenzie (Dec 05 2019 at 03:52):

Sure. But if you're lazy and prefer to save key-strokes, "pat" and "pract" are more traditional :)
