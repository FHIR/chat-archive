Chetana Nayyar (Apr 15 2021 at 18:35):

Currently US Core defines smoking status profile; in case there is a drinking status reported won't this information be available through a US Core endpoint since there isnt any specific profile which helps showing this data?

Daniel Venton (Apr 15 2021 at 18:57):

Just because USCore doesn't mandate use of or "tips" on how a drinking observation should be populated. Your API can certainly return an observation with a code (how much do you drink) and a value (answer). USCore tells you these are the things you must be able to do, nowhere does it say you can't do more.
