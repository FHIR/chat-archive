Topic: _query versus OperationDefinition

Dave Barnet (Mar 11 2022 at 09:31):

The documentation around _query seems pretty scant (http://hl7.org/fhir/R4/search.html#query). I'm a bit confused on the difference between POSTing an Operation to a server and using GET [base]/thing?_query as a Named Query? I see from the spec there is a distiction in that Operations are executed using the Operations Framework (http://hl7.org/fhir/R4/operationdefinition.html#exec) and Named Queries are executed using a search (http://hl7.org/fhir/R4/operationdefinition.html#equeries) - but when would you use one over the other?

Lloyd McKenzie (Mar 11 2022 at 15:27):

_query can be combined with other search parameters and involves hitting the standard 'search' endpoint, rather than a $someOperation endpoint. On the other hand OperationDefinition can be used to define all sorts of operations that don't fit into the query mode.
