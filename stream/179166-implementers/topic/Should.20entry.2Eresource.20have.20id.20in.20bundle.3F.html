Topic: Should entry.resource have id in bundle?

Yunwei Wang (Oct 31 2018 at 14:36):

When I read Composition example, http://hl7.org/fhir/2018Sep/document-example-dischargesummary.xml.html, I noticed that some entry resources have entry.resource.id and some don't. And all entries on the first group (resource with id) have http typed url in entry.fullUrl and all entries on the 2nd group (resource w/o id) have uuid in entry.fullUrl. Is there any reason/preference behind this?
Example for resource with id:

  <entry>
    <fullUrl value="http://fhir.healthintersections.com.au/open/Encounter/doc-example"/>
    <resource>
      <Encounter>
        <id value="doc-example"/>

Example for resource w/o id:

  <entry>
    <fullUrl value="urn:uuid:541a72a8-df75-4484-ac89-ac4923f03b81"/>
    <resource>
      <Observation>
        // no <id> element
