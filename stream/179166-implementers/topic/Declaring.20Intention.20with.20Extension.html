Topic: Declaring Intention with Extension

Martin Grundberg (Apr 05 2019 at 12:22):

Is there a way of declaring the intent of an Extension when creating one? Some will be created just to solve some local or system specific business requirement, but many are really needed to support common requirements. A sign of this is of course all the official extensions.

As a part of Swedish profiling, we see that we have the same requirement as our Dutch friends in stating that an address is "official" (http://fhir.nl/fhir/StructureDefinition/nl-core-address-official). We are having a discussion on how to handle our common requirement, but we agree that a common/official extension would be good.

Question
Is there a structured way of declaring the intent of an extension (e.g. Local/National/International, Local/Official etc)? This could make it easier to reuse extensions instead of creating new ones, as well as might help the creators of extensions to create them in a more generic way and add their local information in their profiles instead.

Grahame Grieve (Apr 05 2019 at 12:23):

StructureDefinition.jurisdiction

Lloyd McKenzie (Apr 05 2019 at 14:39):

If you're seeing a need for an extension that spans multiple countries, that's a good argument for proposing it as a "standard" extension and submitting it for publication by HL7 using the 'propose a change' link.
