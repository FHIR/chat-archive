Topic: ✔ Query for FHIR Resource using ValueSet?

Håkan MacLean (Feb 14 2022 at 10:00):

Hello! Tried searching but couldn't find anything useful. So say we have a ValueSet that contains a couple of Codes. We have created a bunch of FHIR Observations with codes from this ValueSet. Is there a way for a client to ask a FHIR Server to "give me Observations with Codes from this ValueSet"? I.e. have the FHIR server expand the ValueSet? Or is it the clients job to first expand it, and then create separate queries for each code? I looked at this page, but it seems to only be about the expand operation and I want to use the expand operation as part of a FHIR Resource query.

P.S. I know there is the Observation.category field, which seems related to the purpose of our ValueSet, but since I can see our ValueSet evolve over time and the same code even belonging to different ValueSet depending on our use-cases, coding this into the Observation.category field feels like a brittle solution.

René Spronk (Feb 14 2022 at 10:49):

code:in=cononical-url-of-valueset - see the :in modifier as documented on the search page in the FHIR spec.
