Tim Berezny (Jun 14 2018 at 02:41):

The notes for endpoint.managingOrganization suggest that this is NOT the cloud application that hosts the endpoint (e.g., an eReferral system), but rather organization that manages/receives the data (in the case of an eReferral, the target recipient).

The challenge is, eReferral applications often have a shared endpoint that applies across many organizations, thus cannot indicate the managingOrganization as the target healthcare service provider organization.

How to deal with the endpoint.managingOrganization element in the scenario of shared endpoints?
