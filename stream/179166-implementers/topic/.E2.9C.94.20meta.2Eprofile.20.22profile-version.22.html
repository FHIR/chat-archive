Topic: ✔ meta.profile "profile-version"

Christoph Müller (Aug 26 2021 at 12:52):

Dear all, is there yet a best practice for passing a profiles "version", that one resource claims to be conformant to? E.g. for a patient-ressource we have meta as follows:

"meta": {
"profile": [
"https://www.medizininformatik-initiative.de/fhir/core/modul-person/StructureDefinition/Patient"
]
},

As hospitals may not be able to keep up with the release of a new version (say, they are using a version 1.0.8 and now the current version is 2.0.0), they might want to post a Patient-resource saying "hey, i am version 1.0.8". Appending an "/1.0.8" results inn an invalid profile-url as it seems. We came up with the following and would like to ask, if that looks like a good idea to you :

"meta": {
"profile": [
"https://www.medizininformatik-initiative.de/fhir/core/modul-labor/StructureDefinition/ObservationLab"
],
"tag": [
{
"system": "https://www.medizininformatik-initiative.de/fhir/core/modul-labor/StructureDefinition/ObservationLab",
"version": "1.0"
}
]
}

Chris Moesel (Aug 26 2021 at 13:06):

@Christoph Müller - Meta.profile is a list of canonical. One of the primary differences between canonical and uri (or url) is that it can hold a version number. You do this by adding a pipe (|) and then the version at the end.

So, you should be able to do this:

"meta": {
  "profile": [
    "https://www.medizininformatik-initiative.de/fhir/core/modul-person/StructureDefinition/Patient|1.0.8"
  ]
},
