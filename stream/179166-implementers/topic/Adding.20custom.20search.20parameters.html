Topic: Adding custom search parameters

Richard Kavanagh (Jan 28 2020 at 13:43):

We have a need to add some custom search functionality and as such are looking to define some SearchParameters.

When naming the search I'm looking at the third bullet on http://hl7.org/fhir/STU3/search.html#conformance which suggests that search names are prefixed with a "-".

Is this what people are doing? I'm not seeing this at http://www.hl7.org/fhir/us/core/SearchParameter-us-core-race.html , http://build.fhir.org/ig/hl7au/au-fhir-pd/SearchParameter-au-searchparameter-receivingfacility-universal-id.html or http://fhir.org/guides/argonaut/r2/SearchParameter-ethnicity.html

Lloyd McKenzie (Jan 28 2020 at 15:35):

Yes, we definitely haven't adhered to that in producing HL7-published IGs. But maybe we don't have to because we can theoretically ensure we don't collide with ourselves? In practice, collisions are dealt with by the server defining what code it will use for a given search parameter, so the "start with '-' bit isn't critical.

Richard Kavanagh (Jan 28 2020 at 16:15):

OK - I think we will take the same approach. The "-" would be problematic if that was also a search parameter as you could end up with a double "-".
