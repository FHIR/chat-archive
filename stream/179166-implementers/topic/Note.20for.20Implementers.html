Grahame Grieve (Feb 23 2018 at 22:34):

I am creating a new facility in the build for the build to depend on external values sets and code systems. Content from IHE and DICOM is moving out of the build and will be replaced with references to their web sites. But the build still needs the content internally, so it will still have copies of the resources. In order to assist implementers, all the resources that are handled this way will be placed in a file external-resources.xml/json, in case you want access to them when generating code or initialising terminology servers. Note that this file will not be accessible from the web pages, but will be in examples.zip etc
