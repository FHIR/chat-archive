Joanna Gaskill (Jun 09 2017 at 15:07):

The encounter type codeset says these four values are the entire codeset but this doesn't seem nearly comprehensive enough to represent the types of encounters people may have. I'm looking to use this to define admission types for inpatient admissions, ie maternity, nwborn, emergent, elective, etc. (essentially the data in an HL7 PV1.4 segment). Is that a valid use for this element and if so, do I just define my own codeset? https://www.hl7.org/fhir/valueset-encounter-type.html

Michelle (Moseman) Miller (Jun 09 2017 at 15:35):

The V2 mapping tab (https://www.hl7.org/fhir/encounter-mappings.html#v2) does show Encounter.type mapped to PV1.4
The Encounter.type binding strength is only example (https://www.hl7.org/fhir/terminologies.html#example), so you don't have to use it.
A subset of your example codes might also fit with Encounter.priority, which has elective, urgent.
