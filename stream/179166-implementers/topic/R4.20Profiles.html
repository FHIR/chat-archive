hi Neha
I can't view your uploaded image for some reason.
A structure definition is FHIR's full original definition of a resource (like the source code for it). A schema (json or xml/xsd) is another way to represent that structure, in a less complete way, but that more common industry tooling can deal with. The schemas are created from the structure definitions. Only FHIR specific tooling can deal with structure definitions.
Did you find the FHIR downloads page that is referenced above your post?
