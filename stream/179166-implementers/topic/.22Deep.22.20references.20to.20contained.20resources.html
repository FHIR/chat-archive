http://hl7.org/fhir/references.html#canonical-fragments includes an example canonical value of http://fhir.acme.com/Questionnaire/example%7C1.0#vs1 and makes it clear that:

Canonical references may include a fragment when the reference is to a contained resource. When the target of a canonical reference is a contained resource, the canonical reference will have a fragment as part of the URL:

... but can a Reference.reference also play this trick? For example, can I have a DiagnosticReport.subject like {"reference": "Observation/123#pat"} (a hypothetical contained Patient inside an Observation)?

I see the following on http://hl7.org/fhir/references.html#contained that this #pat reference would seem to violate:

References to contained resources are never resolved outside the container resource

... but then again, the #vs1 example also seems to violate this rule, so I'm not sure I'm interpreting the prohibition on "never resolved outside the container" correctly :-)
