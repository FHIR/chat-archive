Andrew Tropin (Jan 17 2019 at 10:14):

I found usage of POST method with combination of ?mode query parameter very strange:
https://www.hl7.org/fhir/resource-operation-validate.html

Why not just mimic original urls for operations with /$validate suffix?

POST /Patient/$validate
PUT /Patient/id-here/$validate
DELETE /Patient/id-here/$validate

This variant seems much more natural, easier to implement and have less corner cases to handle than

POST /Patient/$validate?mode=create
POST /Patient/id-here/$validate?mode=update
POST /Patient/id-here/$validate?mode=delete

I propose to switch to more consistent and intuitive approach of calling $validate operation.

Grahame Grieve (Jan 17 2019 at 13:41):

those changes violate the http specification. DELETE is not a post query

Andrew Tropin (Jan 21 2019 at 16:22):

@Grahame Grieve Didn't get the idea. Why using DELETE /Patient/id-here/$validate instead of POST /Patient/id-here/$validate?mode=delete violates http spec?

Lloyd McKenzie (Jan 21 2019 at 16:24):
