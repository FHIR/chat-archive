Tristan Wilson (Jan 10 2019 at 20:20):

John Moehrke (Jan 10 2019 at 21:01):

You are allowed to extend when you feel the existing vocabulary in the defined valueset is not appropriate(definition of Extensible). However when you create a new vocabulary you need to define it as a vocabulary value in a system. Sure you can use "C" in YOUR-SYSTEM... Managing a vocabulary is not trivial. However that would be your vocabulary system and thus not known to those that don't know your vocabulary system.. hence the good idea to use a defined vocabulary value when it fits. So, is it really different?
Then note that RelatedPerson.relationship has a much more complete list of possible relationships; but these are not marital relationships, which is what Patient.maritalStatus is defined to hold.
There are other marital status values known in the HL7 v3 vocabulary http://build.fhir.org/v3/MaritalStatus/vs.html
which brings up the good practice to look around at vocabulary standards for the value you are interested in.

Robert McClure (Jan 14 2019 at 22:12):

@Simone Heckmann I'll chime in. Common Law Marriage is definitely a type of marriage and I don't understand why it was left out of the V3 code system. That is a mistake in my opinion. The FHIR MartialStatus code system is the one used in the value set cited above, is extensibly bound to Patient.MaritalStatus and is the code system we need to update. PA owns the value set - which has the definition "all the codes in the code system" (Vocabulary owns the code system but that is less important here), and given that PA owns the resource that needs this I am willing to work with a PA harmonization request to add the code "Common law" to the code system. René Spronk (Jan 15 2019 at 07:55):

One of the problems with this concept is that it really only exists in "common law" countries (loosely defined as Great Britain and its former colonies), i.e. its of little to no relevance in all countries with "napoleontic law". As such the concept would not be within the 80% if that were to be the main ground for deciding whether or not to include it. The definition seems to differ even between jurisdictions that have such a concept, that's probably why it should be left to a jurisdiction to define their own code in addition to the predefined HL7 concept codes.
