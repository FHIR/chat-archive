Topic: Medication Statement - timing code

Kevin O'Shea (Mar 30 2017 at 02:10):

The definition of MedicationStatement.dosage.timing.code says that it is "understood to be a complete statement of whatever is specified in the structured timing data". What is the intended granularity of this "complete statement"? There are several fields in the timing structure so is it intended to be a broad statement about the timing or should it include all of the specific details in the other fields?

Lloyd McKenzie (Mar 30 2017 at 18:20):

The definition in STU3 is: "
A code for the timing schedule. Some codes such as BID are ubiquitous, but many institutions define their own additional codes. If a code is provided, the code is understood to be a complete statement of whatever is specified in the structured timing data, and either the code or the data may be used to interpret the Timing, with the exception that .repeat.bounds still applies over the code (and is not contained in the code)." So, from that, I'd say that it covers everything in the Timing element except .repeat.bounds
