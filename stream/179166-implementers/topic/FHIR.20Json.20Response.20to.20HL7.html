I need to convert FHIR json response to HL7 EDI format? Is there an API to do it?

René Spronk (Jan 03 2019 at 14:39):

The HL7 EDI format (a delimiter based format) is typically used for the HL7 v2 standard, the HL7 FHIR standard doesn't have an EDI expression. In case you're looking to translate HL7 v2 into FHIR or vice versa, there's a specific stream on that topic on this chat forum.

Lloyd McKenzie (Jan 03 2019 at 15:46):

The stream is #v2 to FHIR. There isn't a single solution because v2 implementations tend to be so diverse. However, there are strategies that might prove useful. You can also look at the v2 mappings provided in the spec (e.g. http://hl7.org/fhir/observation-mappings.html#v2) and the guidance on v2 conversion provided in the spec itself: http://hl7.org/fhir/comparison-v2.html
