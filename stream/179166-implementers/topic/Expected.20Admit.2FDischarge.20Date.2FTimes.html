FYI, I asked for this before, per GF#7873, but the Patient Admin response (comment on the tracker) was to use the existing Encounter.period:

The period that is allocated to the Encounter may be used to describe future/expected end dates.
The period on the status History is explicitly actuals, so future values should not be present in this period.

I would still support separation of estimated (or expected) dates vs actual dates.
