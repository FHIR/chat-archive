Topic: Provenance versioning vs agents vs multiple

Brendan Keeler (Oct 13 2021 at 13:23):

When does one version Provenance ? When does one create a Provenance with multiple agents? When does one create a second Provenance that points to the same resource?

Feels like three different routes to the same goal

Lloyd McKenzie (Oct 13 2021 at 14:57):

Versioning Provenance would be unusual - as it would mean you'd somehow messed up the original record. Because it's a record of a single system operation, it shouldn't change. Multiple agents would be things like "who's the author, who's the supervisor, who's the data enterer, what was the data entry terminal, etc. There should ideally be a separate Provenance instance for each version of the resource.
