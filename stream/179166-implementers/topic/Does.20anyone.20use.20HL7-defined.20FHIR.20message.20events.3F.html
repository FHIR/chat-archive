Lloyd McKenzie (Feb 11 2018 at 21:39):

GF#3384 calls on FHIR to replace the existing Message Events with properly defined MessageDefinitions. However, most of the existing Message Events are essentially copied and pasted from other spreadsheets without work groups even realizing they were there. And even where the work group did create them, there hasn't been a lot of thought put into them. If we're going to actually define MessageDefinitions, we'd prefer to do so only where there's actually an interest in implementing them so that we can gain feedback on the designs.

So: If you're making use of any of the HL7 defined FHIR messaging events, let us know by posting here. We'll yank all the ones no one's using and create proper definitions for the ones people are using (based on feedback from those who are actually implementing...)

Richard Kavanagh (Jul 12 2018 at 08:32):

Appreciate it's a little late, but in NHS (England) we use our own message events and make use of the MessageDefinition resource. We make use of extensions within it due to the way in which we create our "profiles" for messages and to ease consumption from other parties (whi may not be as FHIR capable).

Our initial set are published here https://fhir.nhs.uk/MessageDefinition

These have largely been created using tooling here http://clarotech.co.uk/products/tool-fhir-message-tool/
