Hi, we're doing some testing of our Argonaut Sprint 2 server, and a question regarding the /metadata route has come up. Say we have a root_url of http://www.acme.com, and a path of /fhir. Should /metadata be at http://www.acme.com/metadata or http://www.acme.com/fhir/metadata? We implemented it as the former, because I read the Argonaut Sprint 2 spec as to mean that the /metadata route should be at a consistent location relative to the domain; where as the FHIR spec seems to indicate that it should be relative to the [base] url which includes the API path.
