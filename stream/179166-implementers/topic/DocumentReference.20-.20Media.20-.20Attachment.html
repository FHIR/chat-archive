Hans Buitendijk (May 20 2019 at 20:31):

At the recent Montreal workgroup meeting we discussed how to proceed with DocumentReference, Attachment data type and Media.

The conclusion was that we believed that it would make sense for the scope of Media to be moved into DocumentReference, while certain attributes of Media would be included in the Attachment data type, and then replace references to Media with DocumentReference if not already referenced.

Before finalizing that approach though, we agreed that we would create a separate branch to enable everybody to review/validate, so mid/late June at the latest, we would finalize that direction and apply it to the latest build (or not).

The branch to review is: http://build.fhir.org/branches/moehrke-DocRef-Media-Merge/

The current target is that we review any feedback and attempt to make that final decision to update the build during the June 18, 2019, 2-3pm EDT OO on FHIR call.
