Topic: Product pattern

Lloyd McKenzie (Jul 02 2019 at 02:43):

As part of it's mandate of "encourage consistency across resources", the FHIR-I workflow project has begun work on developing a 'product' pattern by looking at some of the more stable 'thing' resources and identifying areas of consistency across them - which could result in a pattern of the same nature as the event, request and definition patterns. Jose shared the work that OO has already done on resource relationships, but we didn't see anything that had yet been done at the element level, so hopefully there's no overlap in effort. We wanted to share what we're doing so there can be collaboration between the two groups. This Google sheet shows where we've gotten so far: https://docs.google.com/spreadsheets/d/1_g4Z1FeTRUFJMFCa2Y_d55afZ6U2E7n8WUM3XH7bUQI
