Robert McClure (Jan 27 2022 at 23:56):

@Brendan Keeler That is not the best solution because while that FTP site collects useful NCIT concept collections (in this case sets that happen to be used by NCPDP) in one place, the content is all inside NCIT at what we need is a way to identify the NCIt "value sets" consistently. FDA uses this extensively (which is why NCPDP does too)

First note that all these codes are drawn from the NCI Thesaurus code system so we have an HL7 url identifier for them. All code sets in the NCIt. like these, are represented in NCIt by creating a "head code" that is the name of the (value) set and then all the members have a relation pointing to that code of "Concept_In_Subset". You can see an example of this with the first entry in the download: NCIt Subset code C89507 - NCPDP DEASchedule Terminology that is the same for all the included members in the set (columns C and D). You can also see this by going to the NCIt for the head code C89507and looking at the "relationships" tab.

So, what we need to do is work with the NCI to clarify how to best represent NCIt subsets as HL7 valuesets, particularly the canonical url we should use. I knew this day would come :grinning: @Carol Macumber @Reuben Daniels
