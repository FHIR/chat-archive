Topic: Behaviour of CodeSystem operations with supplements

Michael Lawley (Jan 29 2021 at 05:51):

While it is prohibited from using a CodeSystem supplement's URI as the value of a system and thus type-level invocation of $lookup, $validate-code, etc are precluded, nothing is said about instance-level invocation of CodeSystem operations on a supplement.

So, should it be legal to call /CodeSystem/my-supplement/$lookup?code=123 and if so, then what should be returned -- just the supplement-specific details, or the union of the CodeSystem and the supplement's details (designations and/or properties)?

Grahame Grieve (Feb 04 2021 at 20:58):

I don't know.

Grahame Grieve (Feb 04 2021 at 23:22):

I've started working on code system supplement validation. As for next release, the validator will call it an error when

a/ you reference a supplement URL in Coding.system
b/ The CodeSystem supplement defines a concept that is not in the base code system
