Topic: Snomed Question

Grahame Grieve (Jan 12 2019 at 23:35):

is there a quick way to find out which snomed version a concept is defined in? e.g. 733104004

Grahame Grieve (Jan 12 2019 at 23:35):

I can't seem to easily fin when it was defined?

Rob Hausam (Jan 13 2019 at 13:14):

I'm not sure if there is a tool that easily exposes that (there could be that I'm not recalling exactly). You can get it from the sct2_Concept file in the full release by looking for the earliest effectiveTime date for the concept.

Peter Jordan (Jan 13 2019 at 15:23):

Yes, effectiveTime is a column in all RF_2 Release Files - so easily accessible if you persist those in a relational DB :) I believe that syntax to query RF_2 attributes will be exposed in the upcoming SCT Query Language.

Michael Lawley (Jan 14 2019 at 06:39):

Do you mean the first time it was defined?
For that you need access to the FULL RF2 Concepts file; the effectiveTime in the SNAPSHOT will only tell when it was last changed

Michael Lawley (Jan 14 2019 at 06:42):

and for this concept (733104004 |Swab from buccal mucosa (specimen)|), it is 20170731

Grahame Grieve (Jan 14 2019 at 10:37):

so there's no way that I can find out without getting the RF2 source?

Michael Lawley (Jan 14 2019 at 10:56):

Creation time/date of a code is not a standard property of the code, no. Furthermore, many people only ever have the SNAPSHOT RF2 and that doesn't contain this information; it is only available via the FULL RF2 file.

Rob Hausam (Jan 14 2019 at 17:21):

Yes, what Michael said is what I was meaning by "full release".

Patrick McLaughlin (NLM) (Apr 17 2019 at 17:49):

We include the effective time in the UMLS; it is an attribute of the preferred term. You can search by 'code' in the UTS Metathesaurus browser: https://uts.nlm.nih.gov/metathesaurus.html
