Grahame Grieve (Jul 20 2018 at 03:32):

I have applied GF#17350, which rationalised the multiple extensions we had for ordinal value down to a single extension. One consequence of this change was to mean a change to the definition, which had previously said that ordinal values could only be defined in code systems. We have lots of uses of ordinal values that are context specific - so I changed the definition to say that the value has to be defined in context (see build.fhir.org/extension-iso21090-ordinal-value.html)

Grahame Grieve (Jul 20 2018 at 03:35):
