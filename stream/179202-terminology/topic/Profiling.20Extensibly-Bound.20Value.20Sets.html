Mark Kramer (Jan 30 2020 at 18:26):

Observation.interpretation is bound extensibly to ObservationInterpretation. It so happens that none of the values in that value set fit are suitable interpretations of Karnofsky Performance Status (whose values are specified in LOINC LL4986-7). So my questions are:
1) Would it be acceptable to bind Observation.interpretation in a profile to a value set that includes non-overlapping values drawn from LL4986-7? (I expect yes, since the original binding is extensible).
2) Would it be acceptable to exclude some or all of the inapplicable codes from ObservationInterpretation value set from the value set in the profile? (I expect this would be yes, as well, because you can remove codes from value sets even when there is a required binding.)
3) If 1 and 2 are acceptable, does it follow that one can bind Observation.interpretation to an entirely different value set, under the proviso that the terms in the original and replacement value set have no overlapping meaning?

Grahame Grieve (Jan 30 2020 at 18:52):

yes to all 3
