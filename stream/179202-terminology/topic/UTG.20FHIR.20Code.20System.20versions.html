Topic: UTG FHIR Code System versions

Grahame Grieve (Aug 18 2020 at 23:39):

The code systems that are currently in FHIR main spec that have canonical URLs are being removed from UTG. I've looked through them, and the following code systems have had changes made to them since the UTG ones were copied:

  • CodeSystem/audit-event-type
  • CodeSystem/condition-clinical
  • CodeSystem/extra-security-role-type
  • CodeSystem/subscription-channel-type

I will update them in UTG directly, and there after, changes will be made via the UTG process.

Grahame Grieve (Aug 18 2020 at 23:42):

this brings me to the as yet unresolved problem of what the versions of those code systems should be in UTG. They are currently marked 4.2.0, which is the version of the FHIR core spec when they were copied from it. I don't think that this is a very useful version, and most of them are relatively simple example value sets. So I propose that I reset them all. Simple ones without any substantial investment, I'll make them 0.2.0, and ones where the community has invested in them will go to 1.0.0. (changes will be logged in history record). Thereafter, they should be subject to standard UTG versioning policy.

Is anyone unhappy with this change? if so, why?
