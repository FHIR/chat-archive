Michael Lawley (Oct 17 2021 at 22:11):

This section: http://build.fhir.org/terminology-service.html#validation says:

Every code system has an implicit value set that is "all the concepts defined in the code system" ... However, for some code systems, they are not known.

Then:

Clients can refer to these implicit value sets by providing the URI for the code system itself.

This creates a bunch of ambiguity:

  • Is this claim limited to $validate-code only, or to any place where an implicit ValueSet URI might be used?
  • Is this instead an awkward reference to $validate-code also being defined on CodeSystem?
