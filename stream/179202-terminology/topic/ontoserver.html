Dion McMurtrie (Apr 10 2018 at 13:42):

Check out https://medserve.online or for the browser built on top of it using the Medication based FHIR API look at https://browse.medserve.online
It is a rendering of AMT as FHIR medication resources to make getting at the content of AMT easier and provides a search API which is pretty powerful. There's a postman collection to explain what it does and how to use it https://documenter.getpostman.com/view/2091243/medserve/77k3MAR

Any questions please let me know, happy to discuss it. We'll probably talk about it at the NCTS connectathon next week on the 17th if you're going to be there.

Dion McMurtrie (Apr 10 2018 at 22:26):

You could go that way @Peter Jordan , but personally I don't see it as an either/or scenario. AMT as a SNOMED CT extension provides logical definition and classification of the concepts which I then render out as Medication resources as a way of accessing the content. I see value in both the FHIR rendering and the SNOMED CT extension forms and production process for AMT.

Look forward to discussing it more next week and where it may all go.
