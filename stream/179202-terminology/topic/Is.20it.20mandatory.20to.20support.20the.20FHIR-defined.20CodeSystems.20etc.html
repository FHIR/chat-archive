Topic: Is it mandatory to support the FHIR-defined CodeSystems etc

Michael Lawley (Nov 20 2016 at 01:08):

The FHIR spec defines a number of CodeSystems and ValueSets (eg the Internal ValueSets here http://build.fhir.org/terminologies-valuesets.html)

Is it mandatory to include support for these in a fresh-out-of-the-box terminology service?

I ask this because a number of public test suites (eg that of ProjectCrucible) assume certain of these already exist as text fixtures rather than creating them if required.

Grahame Grieve (Nov 20 2016 at 05:24):

it's not mandatory. That's an erroneous assumption on the part of the test writers

Grahame Grieve (Nov 20 2016 at 05:26):

Note that we're going to define a new service: ' Conformance Server' - this is a service that can do validation reliably, and these will also need to be a terminology server, and these *will* need to host the FHIR defined value sets. But this will not be a requirement of a terminology service
