Maria Papaioannou (Feb 22 2022 at 08:27):

Hello,

We are designing a prototype for the national integrated EHR system of Cyprus and we would like to use, where appropriate, the eHDSI MVC value sets to serve the requirements regarding the EU cross-border health services (Patient Summary and ePrescription).

I found that some of them are common with IPS value sets (e.g. http://hl7.org/fhir/uv/ips/STU1/ValueSet-medicine-doseform.html), some other IPS value sets are supersets of the corresponding eHDSI MVC value sets (e.g. https://hl7.org/fhir/uv/ips/ValueSet-procedures-uv-ips.html) while some others are totally different.

I also mapped some of the eHDSI MVC value sets with existing value sets defined by the FHIR project and HL7 (e.g. https://www.hl7.org/fhir/valueset-ucum-units.html , https://www.hl7.org/fhir/v3/substanceAdminSubstitution/vs.html, etc.).

Still, these are not defined explicitly by eHDSI or for any eHDSI project and, in addition, a number of the eHDSI MVC value sets are totally missing in FHIR format.

In this context, I would like to ask whether there is any ongoing work involving the construction of these value sets in FHIR?
