Topic: UMLS code systems

Thomas Moser (Nov 06 2017 at 15:42):

I am currently working with UMLS (Unified Medical Language System). This comprises a big collection of concepts of many different code systems (about 200 code systems), see here: https://www.nlm.nih.gov/research/umls/sourcereleasedocs/index.html for a list.

I need URLs for all of these code systems. Few of the code systems are already listed here: https://www.hl7.org/fhir/terminologies-systems.html
but most of them are not. For example, "Multum MediSource Lexicon" is a code system contained in UMLS and has the abbreviation "MMSL" in the UMLS content. Using this, MMSL|4862 is a coding for "Ibuprofen".

Since it is not realistic to get URLs for all of these 200 code systems from the respective content owners, it would be great if FHIR could set URLs for these systems. Since FHIR already has defined
http://www.nlm.nih.gov/research/umls/rxnorm
for RxNorm, which is also contained in UMLS, I think it would be very nice to have the similar URLs for all the other code systems contained in UMLS like the following
http://www.nlm.nih.gov/research/umls/<SAB>
where <SAB> is the abbreviation of the code system as provided by UMLS. The above example for "Multum MediSource Lexicon" would then be
http://www.nlm.nih.gov/research/umls/mmsl
(in small letters to be consistent with rxnorm).
Would you agree to this proposal?
