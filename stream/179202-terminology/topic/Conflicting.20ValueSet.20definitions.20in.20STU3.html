Topic: Conflicting ValueSet definitions in STU3

Michael Lawley (Sep 11 2019 at 01:23):

The code system with URI http://hl7.org/fhir/participant-type (see http://hl7.org/fhir/STU3/codesystem-encounter-participant-type.html) defines the value set with URI http://hl7.org/fhir/ValueSet/encounter-participant-type as its "all codes" ValueSet (and thus there are two members).
However, there is also an explicitly defined value set with the same URI http://hl7.org/fhir/ValueSet/encounter-participant-type (see http://hl7.org/fhir/STU3/valueset-encounter-participant-type.html) that has a very different definition.
Note the corresponding R4 code system does not define an "all codes" ValueSet.

Is this something that can/should be fixed as a STU3 technical correction?
