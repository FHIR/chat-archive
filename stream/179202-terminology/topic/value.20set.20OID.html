I'm trying to resolve https://jira.hl7.org/browse/FHIR-26406 and I'm not sure how value set OID assignment works. I agree that the OID for the value set (http://hl7.org/fhir/R4/valueset-immunization-status.html) doesn't come up when I go to the HL7 OID registry site, but I checked a few other value sets and don't find any of them. Can anyone help me understand how the OID is assigned to a value set? Should these be in the OID registry?

Robert McClure (Mar 10 2021 at 19:24):

There is no requirement that all OIDs for value sets (or code systems) used in HL7 artifacts be listed in the OID registry. When the source of truth for a value set or code system is an HL7 FHIR server, then that publication should be considered the source of truth for all the elements. Hence any terminology artifact published at hl7.org/fhir or at terminology.fhir.org that has an OID listed, that listing is the presumed source of truth and does not require another listing at the HL7 OID registry. @Ted Klein
