Topic: Convention for setting Questionnaire.url

Daniel Vreeman (Jul 30 2019 at 20:26):

Here at Regenstrief we're working on publishing Questionnaire resources for panels and forms content we publish in LOINC, e.g. PHQ-9, PRAPARE, etc. My inclination is that we set a convention for ones we publish to have the Questionnaire.url use a format like "http://loinc.org/q/{LoincNum for panel parent term}". So for the PHQ-9, it'd then be "http://loinc.org/q/44249-1". This is similar to how we use "http://loinc.org/vs/LL358-3" as the canonical url for the ValueSet for the LOINC Answer List LL358-3. Thoughts about using "q" in that pattern?

Grahame Grieve (Jul 30 2019 at 20:27):

we'd rather /Questionnnaire/ I think, than /q/ but /q/ is more consistent with your usual approach. And it's a style question
