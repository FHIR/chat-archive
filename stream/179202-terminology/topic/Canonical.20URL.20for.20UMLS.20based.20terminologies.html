Topic: Canonical URL for UMLS based terminologies

Robert McClure (Oct 09 2019 at 14:57):

Was just asked by @Bob Dolin what the canonical url is for MED-RT. It's not on https://www.hl7.org/fhir/terminologies-systems.html
https://www.nlm.nih.gov/research/umls/sourcereleasedocs/current/MED-RT resolves at NLM to https://www.nlm.nih.gov/research/umls/sourcereleasedocs/current/MED-RT/index.html from which source files and an explaination can be found. Seems like this would be the current choice and while MED-RT is distributed via UMLS, the url http://www.nlm.nih.gov/research/umls/medrt does not resolve at all. This raises an issue for RxNorm and other UMLS-based code systems which seem to have a different url approach. Bottom line is that, like so many other organizations, it seems NLM has not been approaching the use of urls with the rigor demanded by FHIR. I will try to have a discussion with NLM about this so I'll ask that we hold off on final decisions pending that discussion.
