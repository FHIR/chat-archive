Jay Lyle (Mar 16 2018 at 12:19):

We have data that has some description identifiers mixed in with concept identifiers. The argument about whether to use them to look up and send the appropriate concept identifiers is over; the 'do it' side won. Follow-up: has anyone considered sending the description id as well? We can put the description text in text, but what about the identifier? Is it a code?

Rob Hausam (Mar 16 2018 at 12:32):

SNOMED CT description IDs are not considered to be codes (as in appropriate for populating Coding.code). But, if you need to send them you can use the sctdescid extension on Coding (http://hl7.org/fhir/2018Jan/extension-coding-sctdescid.html) for that.
