Michael Lawley (Nov 15 2021 at 11:02):

Hi @Jens Villadsen I had a quick look at that CSV file and discovered that it is > 28 000 lines long. Snapper is not designed to handle CodeSystems / resources this big; it is intended as a "simple authoring tool".
Additionally, it only understands TAB and COMMA separated files, while this appears to be a SEMICOLON separated file.

For cases like this, I would normally use Snapper to author the CodeSystem metadata then copy the resulting JSON and use it with a tool like jq to generate the CodeSystem.concept part.
