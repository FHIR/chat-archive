Patrick Werner (Jan 13 2019 at 18:12):

Is there an existing (non SCT) ValueSet for ValueSet for Positive/Negative/Unknown?

Joel Schneider (Jan 13 2019 at 21:00):

LOINC has answer lists like this one:
https://r.details.loinc.org/AnswerList/LL1436-6.html

However, generalized positive/negative/etc. LOINC answer lists like that one typically contain a SNOMED CT 3rd party copyright notice.

LOINC also defines specialized answer lists which are apparently not bound to SNOMED CT. E.g.
https://r.details.loinc.org/AnswerList/LL2364-9.html
https://r.details.loinc.org/AnswerList/LL1853-2.html

Not sure if that helps.

Daniel Vreeman (Jan 16 2019 at 02:39):

You can use the LOINC Answer Lists without the associated SNOMED CT mappings (even if they are there) as standalone valuesets. There are quite a few variants on Pos/Neg in LOINC answer lists: Pos/Neg/{Equivocal|Inconclusive|Indeterminate|Invalid|Undetermined} but I don't see Unknown. Probably b/c LOINC has a separate policy that says users can add a null flavor to any LOINC answer list (b/c we don't want to proliferate them based on whether they are there in the list or not).

P.S. You might be pleased to know that you can get all the LOINC Answer Lists as FHIR ValueSets. E.g. https://fhir.loinc.org/ValueSet/?url=http://loinc.org/vs/LL360-9. Read more here: https://loinc.org/fhir

However, I see that there's a problem with some of the ValueSet expansions that have redundant LA codes listed. Will see if we can get that fixed.
