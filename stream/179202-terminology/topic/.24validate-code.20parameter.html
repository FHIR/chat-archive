Topic: $validate-code parameter

Yunwei Wang (Sep 06 2017 at 20:11):

In DSTU2, there is an identifier parameter for $validate-code which allows me to use different identifier, such as OID, instead of valueset ID. In STU3, that parameter is replaced by url. How can I continue to use OID other than define a new parameter in my own operation definition?

Lloyd McKenzie (Sep 06 2017 at 20:55):

urn:oid:[whatever your OID was] might work, though don't know if that's the intention.

Yunwei Wang (Sep 06 2017 at 20:56):

So I need to use url=urn:oid:[oid code]. Is that right?

Lloyd McKenzie (Sep 07 2017 at 03:43):

That would convey the OID as a legitimate URI. Whether the intenation is that the $validate-code operation will work with that, someone else will need to speak to.

Jim Steel (Sep 07 2017 at 06:10):

My understanding is that the intention of the original 'identifier' parameter was the same as the current meaning of the current 'url' parameter, i.e. it was not intended to allow any of the resource's identifiers.

On the other hand, as far as I know, there is nothing to stop using an oid in the 'url' property of your resource, probably using the form that Lloyd mentions.

Grahame Grieve (Sep 07 2017 at 13:23):

yes Jim is correct
