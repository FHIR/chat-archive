For anyone who's interested, there's a FHIR code system resource for ICD-10 vietnam as http://www.healthintersections.com.au/icd-10-vn.json. Note that I'm not sure how useful this is; it's certainly bigger than I would recommend using a CodeSystem resource for, and I'm probably not going to use an explicit code system in my server

Jim Steel (Nov 30 2017 at 06:23):

One approach that we have been using for large FHIR CodeSystem resources like this has been to use the content attribute as not-present to indicate that although it is possible to create/update it as a FHIR resource, it is not possible to read the resource content (the concept block) afterwards. The content is still usable in expand/lookup/subsumes/etc requests, though.
