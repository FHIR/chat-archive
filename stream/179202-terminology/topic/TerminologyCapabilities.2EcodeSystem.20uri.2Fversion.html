Topic: TerminologyCapabilities.codeSystem uri/version

John Timm (Feb 24 2021 at 14:41):

If TerminologyCapabilities.codeSystem.uri element is of type canonical (meaning it could include a version suffix <url>|<version>), then why is there also a TerminologyCapabilities.codeSystem.version.code element? Should the version be stored in one place removing the possibility of error/ambiguity?

Rob Hausam (Feb 24 2021 at 17:27):

I'll try to look at that and think more about what the answer should be.

Peter Jordan (Feb 24 2021 at 18:16):

Simple answer is that the current structure allows for multiple versions of a Code System to be defined. Furthermore, not all external Code Systems support appending versions to their identifiers.

Grahame Grieve (Feb 24 2021 at 20:17):

in this case it would not be appropriate to append a version to the TerminologyCapabilities.codeSystem.uri, but I overlooked thinking about the fact that it could be legal
