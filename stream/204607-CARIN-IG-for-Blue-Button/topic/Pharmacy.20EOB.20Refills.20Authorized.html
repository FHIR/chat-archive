I apologize if this was already submitted in JIRA. I was struggling to get the right filters to see if this has already been sent in.

I with a PBM and one item that has come up as a data element that would be useful for members is to see - along with which refill number a claim was for - but also how many refills have been authorized by the prescriber. The Pharmacy EOB Profile includes the refill number (00 for first fill, 01 for the first refill etc.). But there isn't an element or CPCDS data mapping for the number of refills authorized.

The use case we have is to alert a patient that a prescription must be renewed of if they had their last fill.

Is this something that could be added to the Pharmacy EOB profile? Or would we have to populate a reference to a MedicationRequest Resource (which has an aligned element 'numberOfRepeatsAllowed') and _include that resource with the results? I worry about one more resource which would likely repeat the same data as what is on the EOB (patient, medication, prescriber, substitution allowed) just to add one more element. But I agree that it is an attribute more of prescription than of claim.

Thanks for weighing in and let me know if I should submit a JIRA for this to be considered! Appreciate the help!
