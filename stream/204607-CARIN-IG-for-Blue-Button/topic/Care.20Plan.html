Henry Perretta (Jan 13 2021 at 16:36):

The Care Plan profile in the US Core IG does not allow for the direct use of pdf metadata (to map to this US Core IG profile). Other USCDI data sets, such as RAD narrative, allow for the use of pdf data via the DocumentReference profile. Is it allowed in the BB IG to use the US Core Document Reference profile for Care Plan data (in lieu of the Care Plan Profile)? Or could we extend the Care Plan profile to allow for the use of Care Plan pdf data?

Josh Lamb (Jan 13 2021 at 16:58):

Unstructured data is not currently required to be communicated to FHIR. We are only converting data that we can map at a field level to FHIR. The PDF would not be an interoperable format, and the contents of the PDF would vary.

Use of CarePlan (or anything outside of the resources within CARIN BB) would be outside of the scope of the STU1 guide.

Clinical concepts, like CarePlan, will be mapped to USCore or to DaVinci PDex.

John Moehrke (Jan 13 2021 at 17:06):
