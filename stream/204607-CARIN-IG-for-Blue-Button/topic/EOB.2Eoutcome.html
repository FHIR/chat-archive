Corey Spears (Feb 12 2021 at 18:39):

In the context of the CMS Patient Access API for which the CARIN for Blue Button IG is used, the requirement is that adjudicated claims be made available. While the IG states that the expected code supplied would be "complete", it does not restrict the use of other outcome codes. This means it is possible to also provide claims that have a different outcome.

Corey Spears (Feb 19 2021 at 15:35):

After following up with the CARIN BB team, the intent for STU1 was to keep this open with guidance of the usual case (outcome = complete), but after some deliberation, there is an intent to restrict this to completed claims only in STU2.
