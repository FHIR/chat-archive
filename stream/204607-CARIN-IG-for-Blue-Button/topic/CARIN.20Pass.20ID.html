Josh Lamb (Feb 04 2021 at 02:52):

What is the advantage of the PASS ID over a health plan member id? Once CAQH is up we should define code systems for our member identifiers (perhaps our FHIR URL or public portal URL). A universal patient ID does not need to come from a single code system. I can even imagine an information (ID matching) network between trusted payers. We have a code system for a member ID we agreed to use in CARIN IG. Maybe a combination of Organization.identifier:payerid and Member ID.

From an execution perspective, if we can get someone to log into two portals we can create a link. The more portals they add the more links we can create. The clinical data that includes a member id can now be linked as well.

Basically, I think that any identity proofed ID can work. Is this an incorrect assessment?
