Topic: Representing Patient emergency contact in IPS

Morten Ernebjerg (Dec 11 2020 at 08:00):

Hi all :wave: I am looking at storing the emergency contact in an IPS-conformant Patient instance. For R4 in general, there is a standard way to do this (see this thread): add a contact and set contact.relationship.coding.code equal to "C" in http://terminology.hl7.org/CodeSystem/v2-0131 (part of the extensibly bound ValueSet).

However, the IPS patient profile explicitly overwrites the extensible binding in the base spec with a required binding on contact.relationship to the value set http://hl7.org/fhir/uv/ips/ValueSet/personal-relationship-uv-ips. This, in turn, only contains terms designating family/social relations (roommate, mother etc.), but no code for emergency contact. Is there another way to represent an emergency contact? - it seems an important concept to capture (certainly is for us).
