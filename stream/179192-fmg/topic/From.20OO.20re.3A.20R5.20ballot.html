Josh Mandel (Sep 23 2021 at 15:14):

Report from OO (thanks Hans!) is they are looking at:

  • taking Device normative (certain elements), but unclear if this is ready in the Jan timeframe. Would like to do more connectathon testing to get there.

  • maybe taking Specimen normative (certain elements), but unclear if this is ready in the Jan timeframe. Would like to do more connectathon testing to get there.

  • some changes to Observation in R5 to introduce richer support for specimens + devices (currently backed by normative elements)

  • increasing maturity of NutritionOrder

... all of these would benefit from more connectathon testing; none is a sure thing as of today.

Josh Mandel (Sep 23 2021 at 16:00):

Also for discussion: we're seeing the pain of normative resources with 0..1 elements that really should have been 0..*. Perhaps as new resources are designed and as existing resources climb up the maturity scale, we should be asking editors to bias toward allowing 0..* by default, unless they can robustly reject the possibility that they will ever want more than one element in a slot.
