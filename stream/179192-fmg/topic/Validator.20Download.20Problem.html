Grahame Grieve (Feb 01 2019 at 01:50):

So in response to feedback from the implementers, James and I have moved the FHIR validator so that it's released through Maven.

So, to be clear:

  • in the past, the validator was built as part of the spec build, and you downloaded the correct file from http://build.fhir.org/validator.zip
  • the file was always called validator.zip or org.hl7.fhir.validator.jar (once you unzipped it)
  • we didn't archive past versions of the validator

Now:

  • the validator is built by maven and released through standard maven release frameworks
  • the release is versioned - e.g. the version of the release is explicitly in the jar
  • you download it (at the moment) by going to https://oss.sonatype.org/content/repositories/snapshots/ca/uhn/hapi/fhir/org.hl7.fhir.validation.cli/, find the latest version, and download the standalone jar (James and I are working on options for simplifying those instructions)
