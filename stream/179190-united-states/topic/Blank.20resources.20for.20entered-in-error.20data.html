Topic: Blank resources for entered-in-error data

Eric Lehner (Feb 09 2021 at 14:01):

The guidance supplied in https://www.hl7.org/fhir/us/core/general-guidance.html#representing-deleted-information states that:

If the status is entered-in-error:
for patient viewing systems the content of resource SHOULD be removed. In other words a blank resource.

What's meant here in terms of a blank resource? Is there a particular type of resource that should be used in this case?
If not, and we should instead use the specific resource type being searched/requested, what's the expectation of the data being returned in terms of being valid for that resource? Ex. for Procedure, is it valid in the case of entered-in-error data to fill in only the ID and Status, or do all required fields (code, subject, etc.) need to be filled with a data-absent-reason such as "masked"?

Michele Mottini (Feb 09 2021 at 14:36):

You should use the specific resource type being searched/requested. I'd say leaving everything out except the status should be OK - but don't do custom mappings for that, if there is underlying data that gets mapped that's fine

Eric Haas (Feb 10 2021 at 00:46):

updated the guidance in the ballot version. https://build.fhir.org/ig/HL7/US-Core/general-guidance.html#representing-entered-in-error-and-deleted-information
