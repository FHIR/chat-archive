Topic: resource references to a different patient

David Hay (Sep 25 2016 at 02:20):

for the genomics folk - I've just uploaded a update that allows you to refer to a resource for another patient (needed for familyMemberHistory). When you select a reference datatype there's a new link 'find {{x}} resource for another patient. Selecting it displays a dialog where you can select the patient and the resource. A quick check shows that it seems to work with extensions of type reference (as it should) - but not exhaustively checked...

David Hay (Sep 25 2016 at 02:24):

Other changes in this update:
- if you click 'save' when creating a resource, then subsequent 'saves' will update the resource (rather than creating a new one) until you close the editor.
- you can select a patient by id rather than name (useful when you already know the id - useless otherwise...)

David Hay (Sep 25 2016 at 02:26):

let me know if there are issues with any of these changes...
