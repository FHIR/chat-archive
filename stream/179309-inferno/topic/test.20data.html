John Moehrke (May 01 2020 at 17:04):

Noticing the test data... is there a dataset for DSTU2?

Robert Scanlon (May 01 2020 at 18:07):

We've only focused on producing test data for US Core v3.1 up to this point, which is R4. We use Synthea to generate realistic patient clinical data, pick out a couple of representative patients that demonstrate all MUST SUPPORT requirements of US Core v3.1, and fill in any gaps (there aren't many).

Robert Scanlon (May 01 2020 at 18:11):

I believe Synthea can output to DSTU2 still, so you could use raw Synthea FHIR DSTU2 output if you want that same basic set of data, but our scripts for culling the data set to something good for testing would need some effort to port to DSTU2 (and you would need some definition of MUST SUPPORT, from Argon Data Query IG for example).

Robert Scanlon (May 01 2020 at 18:14):

Crucible is capable of generating unrealistic / garbage data sets for DSTU2/STU3/R4 that are technically conformant (well, almost, some of the invariant constraints may be a little off), I can point you to the relevant resources for that if you are interested. I'd like to get something like that into Inferno as we continue to broaden beyond our initial USCDI focus.
