Nandan Sharma (Oct 27 2021 at 06:41):

Hello All,
We are in the process of implementing of AllergyIntolerance Profile, the cardinality of the reaction.manifestation is 1..* and it is also considered as the USCDI element as "reaction" so considering cardinality and USCDI this would be mandatory element for Inferno Program Edition Test (USCAI-07) to pass under AllergyIntolerance i.e. it cannot be skipped or cannot be replaced by DAR.
Is our understanding correct about this element..? Looking forward to reply for better understanding. Thanks.

Yunwei Wang (Oct 27 2021 at 14:23):

In US Core v3.1.1, AllergyIntolerance.reaction is 0..* and AllergyIntolerance.reaction.manifestation is 1..*. That means if an AllergyIntolerance instance does NOT have reaction then it does not have to have reaction.manifestation.

In case an AllergyIntolerance does have reaction but the manifestation is missing, US Core v3.1.1 does require DAR to be used for an mandatory element in the Missing Data

For non-coded data elements, use the DataAbsentReason Extension in the data type
Use the code unknown - The value is expected to exist but is not known.

Nandan Sharma (Oct 28 2021 at 15:36):

It means reaction.manifestation can be replaced with DAR as per Missing Data guidelines but AllergyIntolerance.reaction will be required and mandatory element as it is considered under USCDI v1 though it is marked as 0..* is US Core..?? Please guide. Thanks.

Yunwei Wang (Oct 28 2021 at 16:22):

No. AllergyIntolerance.reaction is not required.
