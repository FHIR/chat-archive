Topic: Mandatory write and $docref Operation

James Dinsmore (Apr 19 2021 at 17:16):

At the bottom of the DocumentReference US Core profile page (https://www.hl7.org/fhir/us/core/StructureDefinition-us-core-documentreference.html) it mentions a mandatory write capability (POST [base]/DocumentReference) and mandatory operation (GET [base]/DocumentReference/$docref?patient=[id]) but neither of these are currently part of the Inferno test tool. Will either of these be added to Inferno? If not are either of these required to pass G10?

Robert Scanlon (Apr 19 2021 at 20:43):

Neither of these are required by g10. The rule excludes any write capabilities, so writing DocumentReferences is outside the scope of testing. I've been told $docref is out of scope because it is not needed to access USCDI information. I don't have any offical reference that states that is the case, so this is inferred based on the fact that US Core requirements are bounded by USCDI for certification.

Robert Scanlon (Apr 19 2021 at 20:46):

I do not expect these to be added to the certification tests ("Inferno Program Edition"), or at least until/unless ONC expands the criteria. However, I would like to add them to our non-certification US Core tests when we have a chance, because I think it would be useful to be able to test all of US Core (and not just the ONC-required subset), if you wanted.

James Dinsmore (Apr 19 2021 at 21:38):
