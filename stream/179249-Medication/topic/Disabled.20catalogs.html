Patricia Almeyda Polanco (Jun 04 2020 at 20:03):

We are using this IG https://build.fhir.org/ig/HL7/fhir-order-catalog/ to create our medication catalogs. We have a catalog for each pharmacy. We need something like an status to mark these catalogs as disable when we stop working with a pharmacy, but catalog.status bindings is required and non of the options seems to be right for this case.

Melva Peters (Jun 04 2020 at 20:24):

What about using Composition.ValidityPeriod which would allow you to change the end date when the catalog is no longer valid? https://build.fhir.org/ig/HL7/fhir-order-catalog/StructureDefinition-CatalogHeader-definitions.html#Composition.extension:ValidityPeriod

The Pharmacy Work Group is starting to get involved in this work for defining Medication Catalogs - my understanding is that much of the work so far has focused on Lab Uses Cases. We would definitely welcome your feedback in the requirements for Medication Catalogs.
