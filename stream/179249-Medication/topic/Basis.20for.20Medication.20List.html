Hugh Glover (Jan 05 2021 at 21:04):

This is on the Vulcan/RWD stream but I wanted it to have a wider audience:

*US Core recommends that an “active” medication list can be obtained using only the MedicationRequest resource. However the reference implementation for this RWD track is looking at various test servers and we can see that different approaches have been followed on different servers. Some indeed follow US core while others make much greater use of MedicationStatement. It would be very helpful to have Vendor opinions on the adequacy of US Core recommendations for collecting concomitant medication information for subjects on a clinical trial.
Thanks - Hugh*

Melva Peters (Jan 06 2021 at 00:06):

The Pharmacy Work Group has an active project to develop best practices and implementation guidance on creating Medication Lists for different contexts or use cases. We include all resources that may be appropriate for the use case - MedicationRequest, MedicationDispense, MedicationUsage (aka MedicationStatement) and MedicationAdministration. We have recommended to US Core that they consider including MedicationUsage back in in a future release and support both resources.
